UAO_CustomizingComponent
부모: UActorComponent
역할
플레이어 캐릭터에 부착되어 외형 커스터마이징을 관리하는 컴포넌트

초기화
BeginPlay에서 PlayerState의 커스터마이징 데이터 로드 (재시도 포함)
지원 캐릭터
ECharacterMesh: Elsa, Anka, Bruce, Cameron (4종)
UActorComponent를 상속하며, PlayerCharacter에 부착되어 캐릭터 외형(메시, 헤어, 의상)을 런타임에 변경합니다. Mutable의 UCustomizableObjectInstance를 캐릭터 종류별로 생성하여 관리하고, FCustomizingData 구조체로 현재 외형 상태를 추적합니다. BeginPlay에서 PlayerState로부터 저장된 커스터마이징 데이터를 로드하며, PlayerState가 아직 없으면 다음 Tick에서 재시도합니다. 서버 RPC를 통해 클라이언트의 변경 요청을 서버로 전달하고, ReplicatedUsing으로 다른 클라이언트에 동기화합니다.

UAO_PlayerCharacter_AttributeDefaults
부모: UDataAsset
역할
플레이어 어트리뷰트의 기본값을 정의하는 데이터 에셋
용도
PlayerCharacter.InitializeAttributes()에서 읽어 AttributeSet 초기값 설정
특징
순수 데이터 컨테이너. 로직 없음, cpp는 include만 포함
카테고리
Health, Stamina, Speed 3가지 섹션
UDataAsset을 상속하는 순수 데이터 에셋입니다. 에디터에서 블루프린트/인스턴스로 생성하여 캐릭터의 초기 체력, 스태미나, 이동속도 값을 설정합니다. PlayerCharacter의 InitializeAttributes()가 이 에셋에서 값을 읽어 AttributeSet에 적용하므로, 밸런스 조정 시 코드 수정 없이 에셋만 변경하면 됩니다.

MaxHealth float 100.0 최대 체력 상한
Stamina float 100.0 초기 스태미나
MaxStamina float 100.0 최대 스태미나 상한
WalkSpeed float 200.0 걷기 이동속도
RunSpeed float 500.0 달리기 이동속도
SprintSpeed float 800.0 전력 질주 이동속도
순수 데이터 에셋이므로 함수가 없습니다. cpp 파일은 헤더 include만 포함합니다.

UAO_AbilityTask_WaitForInteractableTraceHit
부모: UAbilityTask
역할
플레이어 시야 내 상호작용 오브젝트를 주기적으로 추적하는 GAS 태스크
감지 방식
카메라 방향 SphereCast (기본 반경 30, 범위 100, 0.1초 주기)
출력
InteractableChanged 델리게이트로 감지된 상호작용 정보 배열 전달
시각 피드백
CustomDepth + StencilValue로 오브젝트 하이라이트 on/off
UAbilityTask를 상속하며, 상호작용 어빌리티가 활성화되어 있는 동안 주기적으로 카메라 방향 SphereCast를 수행합니다. 감지된 오브젝트가 IAO_Interface_Interactable을 구현하거나 InteractableComponent를 보유하면 상호작용 후보로 등록합니다. 이전 프레임과 비교하여 변화가 있으면 하이라이트를 갱신하고 InteractableChanged 델리게이트를 브로드캐스트합니다. 사망/납치/무적 태그가 있으면 트레이스를 차단하고 기존 하이라이트를 제거합니다. AimWithPlayerController로 카메라 시점 기반 정밀 타겟팅을 수행하며, ClipCameraRayToAbilityRange로 구체 범위 내로 레이를 제한합니다.
변수명타입기본값설명
InteractionQuery FAO_InteractionQuery - 상호작용 요청 정보. RequestingAvatar, RequestingController 포함
StartLocation FGameplayAbilityTargetingLocationInfo - 트레이스 시작 위치 정보
