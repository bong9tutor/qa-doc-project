UAO_GameInstance
부모: UGameInstance
역할
게임 인스턴스 — 맵 이동 중에도 살아남는 전역 데이터 컨테이너
생존 범위
앱 시작 → 앱 종료 (맵 로드 · 언로드와 무관하게 유지)
실행 위치
서버 · 클라이언트 각각 독립 인스턴스 존재 (복제 없음)
주요 책임
스테이지 인덱스, 연료 초기값, 부활 횟수, 호스트 정보, 게임 통계 보존
UGameInstance를 상속하며, 언리얼 엔진이 앱 실행 시 단 하나만 생성합니다. 맵 전환(ServerTravel) 시 GameMode · GameState · Actor는 모두 파괴되지만 GameInstance는 파괴되지 않으므로, 스테이지 간에 유지해야 할 데이터 (연료량, 스테이지 인덱스, 부활 횟수, 팀 사망 횟수)를 저장하는 용도로 사용합니다. 생성자에서 SharedTrainFuel을 43.0f (에셋 로드 실패 대비 기본값)으로 설정하고, Init() 오버라이드에서 FuelDataAsset으로부터 실제 연료값을 로드하여 덮어씁니다. AO_OnlineSessionSubsystem과 AO_Passive_WorldSubsystem도 이 인스턴스의 서브시스템으로 생명주기를 공유합니다. 호스트 판별은 PlayerState의 FUniqueNetIdRepl에서 문자열을 추출하여 LobbyHostNetIdStr과 비교하는 방식입니다.
변수명타입지정자설명
SharedTrainFuel float VisibleAnywhereBPReadOnly 스테이지 · 휴게공간 사이를 오갈 때 보존할 기차 연료 수치. 생성자에서 43.0f로 초기화되며, Init()에서 FuelDataAsset->InitialFuel 값으로 덮어쓰입니다. ResetRun() 시 GetInitialFuelValue()로 재설정됩니다.
CurrentStageIndex int32 VisibleAnywhereBPReadOnly 현재 진행 중인 스테이지 번호 (0부터 시작). AO_MapRoutes::GetStageMapName()에 이 값을 전달하여 맵 이름(FName)을 조회합니다. TryAdvanceStageIndex()로 1씩 증가합니다.
LobbyHostNetIdStr FString VisibleAnywhereBPReadOnly 로비 호스트 플레이어의 Steam UniqueNetId를 ToString()한 문자열. SetLobbyHostFromPlayerState()에서 FUniqueNetIdRepl → GetUniqueNetId() → ToString() 체인으로 추출합니다. 빈 문자열이면 호스트 미등록 상태입니다.
InitialSharedReviveCount int32 EditAnywhereBPReadOnly 런 시작 시 팀에 주어지는 공유 부활 횟수의 기본값. 생성자에서 2로 설정됩니다. EditAnywhere이므로 에디터에서 조정 가능합니다.
SharedReviveCount int32 VisibleAnywhereBPReadOnly 현재 남은 팀 공유 부활 횟수. 생성자에서 InitialSharedReviveCount(=2)로 초기화됩니다. GameState의 AddSharedReviveCount() / TryConsumeSharedReviveCount()가 이 값을 GI에 동기화합니다.
GameStartTime / GameEndTime float EditAnywhereBPReadWrite 게임 시작 · 종료 타임스탬프. GameState의 BeginPlay()에서 GI로부터 읽어오고, SetGameStartTime()/SetGameEndTime()에서 GI에 다시 기록합니다. 결과 화면에서 클리어 시간 계산에 사용됩니다.
TeamDeathCount int32 EditAnywhereBPReadWrite 팀 전체 사망 횟수 누적값 (통계용). 생성자에서 0으로 초기화되고 ResetRun()에서도 0으로 리셋됩니다. GameState::AddTeamDeathCount()가 GI에 동기화합니다.
FuelDataAsset TObjectPtr<UAO_FuelData> EditAnywhereBPReadOnly 기차 초기 연료량을 정의하는 데이터 에셋(protected). GetInitialFuel()과 GetInitialFuelValue()가 이 에셋의 InitialFuel 필드를 참조하며, null이면 폴백값(SharedTrainFuel 또는 43.0f)을 반환합니다.
Init() Override ▶
하는 일
엔진이 GameInstance를 생성한 직후 호출하는 초기화 함수입니다. Super::Init()을 먼저 호출한 뒤, GetInitialFuel()을 통해 FuelDataAsset에서 InitialFuel 값을 읽어 SharedTrainFuel에 대입합니다.
내부 구현
GetInitialFuel()은 FuelDataAsset이 유효하면 에셋의 InitialFuel을, null이면 이미 생성자에서 설정된 SharedTrainFuel(43.0f)을 그대로 반환합니다.
호출 시점
앱 시작 시 엔진이 자동 호출합니다.

하는 일

내부 구현

호출 시점

UAO_OnlineSessionSubsystem
부모: UGameInstanceSubsystem
역할
Steam Online Subsystem 세션 API를 래핑하는 서브시스템
생존 범위
GameInstance와 동일 (앱 전체 생존)
실행 위치
주로 클라이언트에서 호출, 실제 세션은 Steam 서버 경유
주요 책임
호스트 · 찾기 · 입장 · 파기 + 음성채팅 제어 + 네트워크 실패 처리
UGameInstanceSubsystem을 상속하므로 GameInstance가 생성될 때 자동으로 인스턴스화됩니다. Steam OSS의 IOnlineSessionPtr를 통해 세션을 생성(Host) · 검색(Find) · 입장(Join) · 파기(Destroy)합니다. 세션 설정 키(SERVER_NAME, HAS_PASSWORD, PASSWORD_MD5, CURRENT_PLAYERS, IN_GAME)를 AO_SessionKeys 네임스페이스에 FName 상수로 정의합니다. Initialize()에서 Steam 초대 수락 델리게이트(OnSessionUserInviteAccepted)와 GEngine의 네트워크 실패 델리게이트(HandleNetworkFailure)를 바인딩하고, Deinitialize()에서 해제합니다. 또한 VoiceInterface->Shutdown()으로 음성 세션을 정리합니다. 방 이름은 Base64로 인코딩/디코딩하여 UTF-8 한글이 세션 설정에서 안전하게 전달되도록 처리합니다. 비밀번호는 FMD5::HashAnsiString으로 해시하여 PASSWORD_MD5 키에 저장합니다. bOpInProgress 플래그로 Host/Join/Destroy 동시 호출을 방지하는 재진입 가드 패턴을 사용합니다.
변수명타입지정자설명
OnFindSessionsCompleteEvent FAO_OnFindSessionsComplete BlueprintAssignable 세션 검색이 완료되었을 때 브로드캐스트되는 다이나믹 멀티캐스트 델리게이트 (bool bSuccessful 파라미터). UI Widget이 바인딩해 방 목록을 갱신합니다.
LastSearch TSharedPtr<FOnlineSessionSearch> (private) FindSessions()에서 생성되는 검색 요청 객체. MaxSearchResults, bIsLanQuery, SEARCH_LOBBIES 쿼리 설정을 포함합니다.
LastSearchResults TArray<FOnlineSessionSearchResult> (private) 검색 완료 콜백에서 중복 · 잘못된 세션을 필터링한 뒤 캐시되는 결과 배열. TSet<FString>으로 세션 ID 중복 제거 후, SEARCH_LOBBIES 태그, SERVER_NAME 존재 여부, OwningUserName 비어있지 않은지 검증합니다.
bFinding / bOpInProgress bool (private) bFinding: 검색 진행 여부 (FindSessions 시작 시 true, 완료/취소 시 false). bOpInProgress: Host/Join/Destroy 작업 재진입 방지 플래그. HandleNetworkFailure()에서 둘 다 false로 리셋됩니다.
Cached* (재호스트 캐시) int32, bool, FString (private) CachedNumPublicConnections(기본 4), bCachedIsLAN, CachedRoomName, CachedPassword. 기존 세션 파괴 후 재호스트(bPendingRehost=true)를 위해 파라미터를 임시 저장합니다.
세션 키 상수
(AO_SessionKeys::)		 FName namespace 상수 KEY_SERVER_NAME, KEY_HAS_PASSWORD, KEY_PASSWORD_MD5, KEY_CURRENT_PLAYERS, KEY_IN_GAME. 세션 설정의 키-값 딕셔너리에서 사용합니다. EOnlineDataAdvertisementType::ViaOnlineServiceAndPing으로 광고됩니다.
bPendingReturnToMenu bool (private) 호스트가 DestroyCurrentSession() 호출 시 true로 설정. OnDestroyThenRecreateSession 콜백에서 이 플래그를 확인하고 메인 메뉴로 이동합니다.

내부 구현

APlayerState를 상속하며, 각 플레이어의 개별 상태를 보관합니다. GameState와 달리 PlayerState는 각 컨트롤러에 소속되며, 맵 이동(ServerTravel) 시 CopyProperties()가 호출되어 다음 맵의 새 PlayerState에 데이터를 복사합니다. 이 메커니즘으로 인벤토리(PersistentInventory), 체력(PersistentHealth), 커스터마이징(CharacterCustomizingData), 사망 횟수(DeathCount)가 스테이지 간에 유지됩니다. GetLifetimeReplicatedProps()에서 총 8개 변수를 DOREPLIFETIME으로 등록합니다: bLobbyIsReady, LobbyJoinOrder, bIsLobbyHost, bIsAlive, CharacterCustomizingData, PersistentInventory, DeathCount, bHasPersistentHealth, PersistentHealth. 생성자에서 CharacterCustomizingData의 기본값을 Elsa 메시, Hair01, Glacier 의상으로 설정합니다. BeginPlay()에서 InitVoiceChat()을 호출하여 매 레벨 이동마다 음성 채팅을 재초기화합니다.
변수명타입지정자설명
bLobbyIsReady bool ReplicatedUsing=OnRep_LobbyIsReady 로비 레디 상태. 생성자에서 false로 초기화. 변경 시 OnRep_LobbyIsReady가 OnLobbyReadyChanged 델리게이트를 브로드캐스트하고 RefreshLobbyReadyBoard()를 호출합니다.
LobbyJoinOrder int32 ReplicatedUsing=OnRep_LobbyJoinOrder 로비 입장 순서 (0부터 시작, -1은 미할당). OnRep에서 RefreshLobbyReadyBoard()를 호출하여 보드를 재빌드합니다.
bIsLobbyHost bool ReplicatedUsing=OnRep_IsLobbyHost 이 플레이어가 세션 호스트인지 여부. 서버에서 GameInstance 기반으로 설정됩니다. OnRep에서 RefreshLobbyReadyBoard()를 호출합니다.
bIsAlive bool ReplicatedUsing=OnRep_IsAlive 플레이어 생존 여부. 기본값 true. OnRep에서 OSS->UpdateVoiceMember(this) 호출, 로컬 컨트롤러인 경우 자동 부활 카운트다운(5초) 시작/중지 로직을 실행합니다. 사망 시 공유 부활칩이 1개 이상이면 bPendingAutoRespawn=true + StartRespawnCountdown(5.0f).
DeathCount int32 ReplicatedUsing=OnRep_DeathCountVisibleAnywhereBPReadOnly 이 플레이어의 누적 사망 횟수. OnRep에서 UAO_DelegateManager->OnStatisticsUpdated를 브로드캐스트합니다. CopyProperties()에서 로비 맵이면 0으로 초기화, 그 외에는 이어받습니다.
CharacterCustomizingData FCustomizingData EditDefaultsOnlyBPReadOnlyReplicated 캐릭터 외형 커스터마이징 데이터. 기본값: CharacterMeshType=Elsa, HairStyle=Hair01, ClothType=Glacier. ServerRPC로 서버에서 설정한 뒤 모든 클라이언트로 복제됩니다.
PersistentInventory TArray<FInventorySlot> Replicated 맵 이동 시 보존할 인벤토리 슬롯 배열. SaveInventoryBeforeTravel()에서 Inv->Slots를 직접 복사합니다. CopyProperties()에서 bInventoryShouldPersist 플래그에 따라 다음 PS에 복사하거나 Empty()합니다.
bHasPersistentHealth / PersistentHealth bool / float Replicated 맵 이동 시 보존할 체력값. SaveHealthBeforeTravel()에서 bHasPersistentHealth=true + PersistentHealth=InHealth. CopyProperties()에서 다음 PS에 복사됩니다. ResetStateHealth()로 초기화(false, 0.0f)할 수 있습니다.
bInsideTravelSafeZone / bInventoryShouldPersist / bIsTraveling bool VisibleAnywhere / (UPROPERTY) / (일반) 안전 구역 내 위치 여부, 인벤토리 보존 필요 여부, 이동 중 여부를 나타내는 상태 플래그들. SetSafeZoneState()로 bInsideTravelSafeZone을 설정합니다.
