게임 프레임워크 클래스
세션 생명주기·네트워크 상태·플레이어 데이터를 담당하는 4개 핵심 클래스

UAO_GameInstance
부모: UGameInstance
역할
게임 인스턴스 — 맵 이동 중에도 살아남는 전역 데이터 컨테이너
생존 범위
앱 시작 → 앱 종료 (맵 로드·언로드와 무관하게 유지)
실행 위치
서버·클라이언트 각각 독립 인스턴스 존재 (복제 없음)
주요 책임
스테이지 인덱스, 연료 초기값, 부활 횟수, 게임 통계 보존
UGameInstance를 상속하며, 언리얼 엔진이 앱 실행 시 단 하나만 생성합니다. 맵 전환(ServerTravel) 시 GameMode·GameState·Actor는 모두 파괴되지만 GameInstance는 파괴되지 않으므로, 스테이지 간에 유지해야 할 데이터 (연료량, 스테이지 인덱스, 부활 횟수)를 저장하는 용도로 사용합니다. AO_OnlineSessionSubsystem도 이 인스턴스 안에서 생명주기를 공유합니다.
변수명타입지정자설명
SharedTrainFuel float VisibleAnywhereBPReadOnly 스테이지·휴게공간 사이를 오갈 때 보존할 기차 연료 수치. 서버에서 기록하고 다음 맵 시작 시 GAS AttributeSet에 적용됩니다.
CurrentStageIndex int32 VisibleAnywhereBPReadOnly 현재 진행 중인 스테이지 번호 (0부터 시작). GetCurrentStageMap()이 이 값으로 맵 이름을 조회합니다.
LobbyHostNetIdStr FString VisibleAnywhereBPReadOnly 로비 호스트 플레이어의 Steam UniqueNetId 문자열. 호스트 권한 판별에 사용합니다.
InitialSharedReviveCount int32 EditAnywhereBPReadOnly 런 시작 시 팀에 주어지는 공유 부활 횟수의 기본값. EditAnywhere이므로 에디터에서 직접 조정할 수 있습니다.
SharedReviveCount int32 VisibleAnywhereBPReadOnly 현재 남은 팀 공유 부활 횟수. GameState의 TryConsumeSharedReviveCount()가 이 값을 감소시킵니다.
GameStartTime / GameEndTime float EditAnywhereBPReadWrite 게임 시작·종료 타임스탬프. 결과 화면에서 클리어 시간 계산에 사용됩니다.
TeamDeathCount int32 EditAnywhereBPReadWrite 팀 전체 사망 횟수 누적값 (통계용).
FuelDataAsset UAO_FuelData* EditAnywhereBPReadOnly 기차 초기 연료량을 정의하는 데이터 에셋. GetInitialFuel()이 이 에셋을 참조합니다.

UAO_OnlineSessionSubsystem
부모: UGameInstanceSubsystem
역할
Steam Online Subsystem 세션 API를 래핑하는 서브시스템
생존 범위
GameInstance와 동일 (앱 전체 생존)
실행 위치
주로 클라이언트에서 호출, 실제 세션은 Steam 서버 경유
주요 책임
호스트·찾기·입장·파기 + 음성채팅 제어
UGameInstanceSubsystem을 상속하므로 GameInstance가 생성될 때 자동으로 인스턴스화됩니다. Steam OSS의 IOnlineSessionPtr를 통해 세션을 생성(Host)·검색(Find)·입장(Join)·파기(Destroy)합니다. 세션 설정 키(SERVER_NAME, HAS_PASSWORD, CURRENT_PLAYERS, IN_GAME 등)를 이용해 로비 UI에 방 목록을 표시하고 비밀번호 검증·스테이지 진입 차단을 처리합니다. 또한 언리얼의 IOnlineVoice 인터페이스를 통해 플레이어별 음소거/해제를 관리합니다.

bFinding / bOpInProgress bool (private) 검색 진행 여부와 Host/Join/Destroy 작업 재진입 방지 플래그. 중복 호출 시 작업을 건너뜁니다.

AAO_GameState
부모: AGameState
역할
현재 레벨의 게임 진행 상태를 모든 클라이언트에 복제
생존 범위
현재 레벨 동안만 존재 (맵 전환 시 파괴)
실행 위치
서버에서 권한 있는 상태 변경, 클라이언트는 복제본 수신
주요 책임
공유 부활 횟수, 스테이지 성패, 클리어, 힌트, 통계
AGameState를 상속하며, 모든 플레이어가 공유해야 하는 게임 진행 상태를 보관합니다. 서버에서 SetStageFailed() / SetGameClear() 같은 Authority 전용 함수로 상태를 변경하면, ReplicatedUsing 변수가 자동으로 클라이언트로 전파되고 OnRep_* 함수가 실행되어 각 클라이언트의 UI 델리게이트(OnStageFailed, OnGameCleared 등)를 브로드캐스트합니다.
변수명타입지정자설명

bIsStageFailed bool ReplicatedUsingBPReadOnly 스테이지 실패 여부. true가 되면 OnRep가 OnStageFailed 델리게이트를 브로드캐스트합니다.
bIsGameCleared bool ReplicatedUsingBPReadOnly 게임 클리어 여부. 마지막 스테이지 완료 시 서버가 true로 설정합니다.
RunResetTrigger int32 ReplicatedUsing 패시브 초기화 신호. 서버가 Authority_NotifyGlobalReset()에서 값을 증가시키면 모든 클라이언트의 OnRep_RunResetTrigger가 실행됩니다.
bHint1 / bHint2 / bHint3 bool Replicated 선발대 흔적(힌트) 3종 발견 여부 플래그. FindHint(num)으로 해당 번호를 true로 설정합니다.
CurrentFindHintNum int32 ReplicatedUsingBPReadWrite 현재까지 발견된 힌트 수. OnRep_HintCount가 OnHintCountChanged 델리게이트를 발동합니다.

AAO_PlayerState
부모: APlayerState
역할
플레이어 1인의 상태를 서버·클라이언트에 복제 유지
생존 범위
CopyProperties로 맵 이동 후에도 데이터 이어받기 가능
실행 위치
서버 Authority + 모든 클라이언트에 복제본 존재
주요 책임
로비 레디, 입장 순서, 생존 여부, 인벤토리/체력 맵 이동 보존
APlayerState를 상속하며, 각 플레이어의 개별 상태를 보관합니다. GameState와 달리 PlayerState는 각 컨트롤러에 소속되며, 맵 이동(ServerTravel) 시 CopyProperties()가 호출되어 다음 맵의 새 PlayerState에 데이터를 복사합니다. 이 메커니즘으로 인벤토리(PersistentInventory)와 체력(PersistentHealth)이 스테이지 간에 유지됩니다. 로비에서는 bLobbyIsReady와 LobbyJoinOrder로 레디 보드 UI를 구동합니다.
변수명타입지정자설명
bLobbyIsReady bool ReplicatedUsing=OnRep_LobbyIsReady 로비 레디 상태. 변경 시 OnRep_LobbyIsReady가 실행되어 OnLobbyReadyChanged 델리게이트를 브로드캐스트합니다.
LobbyJoinOrder int32 ReplicatedUsing=OnRep_LobbyJoinOrder 로비 입장 순서 (0부터 시작). 첫 번째 입장자(0번)가 호스트로 취급됩니다.
bIsLobbyHost bool ReplicatedUsing=OnRep_IsLobbyHost 이 플레이어가 세션 호스트인지 여부. 서버에서 GameInstance 기반으로 설정됩니다.
bIsAlive bool ReplicatedUsing=OnRep_IsAlive 플레이어 생존 여부. 사망 시 false로 설정되고, 음성채팅 뮤트 처리에도 사용됩니다.
DeathCount int32 ReplicatedUsingVisibleAnywhereBPReadOnly 이 플레이어의 누적 사망 횟수 (통계용).
CharacterCustomizingData FCustomizingData EditDefaultsOnlyBPReadOnlyReplicated 캐릭터 외형 커스터마이징 데이터. ServerRPC_SetCharacterCustomizingData로 서버에서 설정한 뒤 모든 클라이언트로 복제됩니다.
PersistentInventory TArray<FInventorySlot> Replicated 맵 이동 시 보존할 인벤토리 슬롯 배열. SaveInventoryBeforeTravel()이 이동 직전 기록하고, 다음 맵에서 InventoryComponent로 복원합니다.
bHasPersistentHealth / PersistentHealth bool / float Replicated 맵 이동 시 보존할 체력값. SaveHealthBeforeTravel()로 기록하고 다음 맵의 AttributeSet에 적용합니다.
