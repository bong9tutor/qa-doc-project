UAO_InventoryComponent
부모: UActorComponent
역할
플레이어 캐릭터에 부착되는 인벤토리 컴포넌트 — 슬롯 기반 아이템 관리
부모 클래스
UActorComponent
실행 위치
서버에서 아이템 픽업·사용·드롭 처리, 클라이언트에는 Slots/SelectedSlotIndex 복제
주요 책임
5슬롯 인벤토리 관리, 아이템 픽업/드롭/사용, 입력 바인딩, 리스너 알림
플레이어 캐릭터에 부착되어 5개 슬롯으로 구성된 인벤토리를 관리합니다. 슬롯 배열(Slots)과 선택 인덱스(SelectedSlotIndex)는 ReplicatedUsing으로 복제되어 클라이언트와 동기화됩니다. Enhanced Input을 통해 슬롯 선택, 아이템 사용(좌클릭), 아이템 드롭(우클릭) 입력을 바인딩합니다.

소모품(Consumable) 사용 GAS 파이프라인: UseInventoryItem_Server에서 ItemDataTable의 PassiveAmount를 조회 → ASC에서 AddHealthClass GameplayEffect Spec 생성 → SetSetByCallerMagnitude("Data.Item", PassiveAmount) 설정 → ApplyGameplayEffectSpecToSelf로 체력 회복 적용 → ClearSlot()으로 슬롯 비움.

무기(Weapon) 사용: 캐릭터 위치에서 전방 10,000 유닛으로 LineTraceSingleByChannel(ECC_Visibility) → ApplyPointDamage(1.0f)로 대미지 적용 → DrawDebugLine으로 시각화.

네트워크 복제 흐름: 생성자에서 SetIsReplicatedByDefault(true). GetLifetimeReplicatedProps에서 DOREPLIFETIME으로 Slots, SelectedSlotIndex, ItemDataTable을 복제 등록. Slots는 OnRep_Slots에서 NotifyListeners + OnInventoryUpdated.Broadcast, SelectedSlotIndex는 OnRep_SelectedIndex에서 NotifyListeners를 호출하여 클라이언트 UI를 동기화합니다.

FInventorySlot 구조체에 ItemID, Quantity, FuelAmount, ItemType 정보를 저장합니다. IAO_InventoryListener 인터페이스를 구현한 리스너에게 슬롯 변경과 선택 변경을 통보합니다. 사망 시 CharDead()로 모든 아이템을 월드에 드롭하며, PlayerState에 인벤토리를 영속 저장할 수 있습니다.

CanInventoryAction 검사: Owner의 ASC에서 "Status.Action.Inspecting" GameplayTag가 있으면 false를 반환하여 검사 중 인벤토리 조작을 차단합니다. 다만 이 함수가 OnLeftClick/OnRightClick에서 호출되지 않아 실제로는 차단이 작동하지 않을 수 있습니다.

FindEmptySlotIndex() const ▶
구현 상세 (.cpp)
• 주의: for 루프가 i = 1부터 시작합니다 (i = 0이 아님). 슬롯 0은 항상 건너뜁니다.
• Slots[i].ItemID == "empty"인 첫 번째 인덱스를 반환
• 빈 슬롯이 없으면 INDEX_NONE(-1) 반환
분석
인덱스 0번 슬롯은 빈 슬롯 검색에서 제외됩니다. SelectedSlotIndex 기본값도 1이므로 0번 슬롯은 특수 용도(예: 장비 슬롯)로 의도된 것으로 보이지만, CharDead()에서는 i = 0부터 드롭하므로 0번에도 아이템이 존재할 수 있습니다. PickupItem에서 빈 슬롯이 없을 때 SelectedSlotIndex 위치에 교체하는데, 이때만 0번 슬롯에 아이템이 들어갈 수 있습니다.

UseInventoryItem_Server() Server, Reliable ▶
구현 상세 (.cpp: _Implementation)
• IsValidSlotIndex(SelectedSlotIndex) && ItemID != "empty" 검증
• Consumable 분기:

  - ASC->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get())
  - Multicast_PlayInventorySound(2) → ClearSlot()
• Weapon 분기:
  - Start = Owner 위치, End = Start + ForwardVector * 10,000
  - FCollisionQueryParams에 Owner를 IgnoredActor로 추가
  - LineTraceSingleByChannel(ECC_Visibility) 실행
  - 적중 시 ApplyPointDamage(HitActor, 1.0f, Direction, Hit, OwnerController, Owner, UDamageType::StaticClass())
  - DrawDebugLine(Red, 1초 지속)
주의 사항
• Weapon 사용 시 아이템이 소모되지 않습니다 (ClearSlot 미호출)
• ItemDataTable이 null일 때 FindRow 호출로 크래시 가능
• Consumable 사용 시 AddHealthClass가 null이면 MakeOutgoingSpec가 실패하지만 SpecHandle.IsValid() 체크로 방어됨
• Fuel, Passive, RevivalChip, None 타입은 UseInventoryItem에서 아무 처리도 하지 않습니다

아이템 액터에 부착되어 AI가 해당 아이템을 줍거나 놓을 수 있게 합니다. TryPickup()은 물리 시뮬레이션을 비활성화하고 지정된 소켓에 아이템을 부착하며, TryDrop()은 부착을 해제하고 물리를 다시 활성화합니다.

물리 제어 상세: ConfigurePhysics()는 최초 호출 시 Owner의 RootComponent를 UPrimitiveComponent로 캐스트하여 TargetPhysicsComp에 캐싱합니다. 이후 bEnablePhysics=true이면 SimulatePhysics(true) + QueryAndPhysics, false이면 SimulatePhysics(false) + NoCollision을 설정합니다. NoCollision은 부착 상태에서 다른 물체와 충돌하지 않도록 합니다.

쿨다운 시스템: CooldownExpireTime은 World->GetTimeSeconds() + Duration으로 설정됩니다. TryDrop에서 5초, ItemCarryComponent의 DropItem에서 DropCooldownTime(기본 20초)으로 재설정됩니다. 후자가 더 길므로 SetIgnoreCooldown(DropCooldownTime)이 5초를 덮어씁니다.

bIsPickedUp은 ReplicatedUsing=OnRep_IsPickedUp으로 복제되어 클라이언트에서 ConfigurePhysics(!bIsPickedUp)를 호출하여 물리 상태를 동기화합니다.

UAO_AddFuel_GameplayAbility
부모: UGameplayAbility (GAS)
역할

트리거 방식

주요 책임

GAS의 UGameplayAbility를 상속하여 기차 연료 충전을 구현합니다. 생성자에서 AbilityTriggers에 FAbilityTriggerData를 추가하여 "Event.Interaction.AddFuel" 태그를 GameplayEvent 소스로 등록합니다. ASC->HandleGameplayEvent() 호출 시 이 태그가 매칭되면 자동으로 ActivateAbility가 호출됩니다.

ActivateAbility 상세 흐름: IsNetAuthority 확인 + CommitAbility(리소스 확보) → ASC/AddEnergyEffectClass 유효성 검증 → TriggerEventData->EventMagnitude에서 FuelAmount 추출 → FuelAmount ≤ 0이면 취소 → MakeOutgoingSpec으로 GE Spec 생성 → SetByCallerMagnitude("Data.EnergyAmount", FuelAmount) → ApplyGameplayEffectSpecToSelf → EndAbility(false, false).

PendingAmount 변수: Transient로 선언되어 직렬화되지 않습니다. 코드에서 사용되지 않으며 (실제 값은 EventMagnitude에서 가져옴), 향후 확장 또는 디버깅 목적으로 존재할 수 있습니다.
변수명타입지정자설명
AddEnergyEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly 연료 증가에 사용할 GameplayEffect 클래스. 에디터에서 SetByCallerMagnitude를 사용하는 Instant/Periodic GE를 지정. "Data.EnergyAmount" 태그로 충전량을 전달.
PendingAmount float (Transient, 기본값 0) 코드에서 직접 사용되지 않음. ActivateAbility에서 TriggerEventData->EventMagnitude를 직접 사용.

GAS의 UGameplayAbility를 상속하여 기차 연료 누수(감소)를 구현합니다. AddFuel과 달리 이벤트 기반이 아니라 TryActivateAbility로 명시적으로 활성화됩니다.

PendingAmount 값 충돌: 헤더에서 PendingAmount = -10.f로 선언하지만, 생성자(.cpp)에서 -5.f로 재설정합니다. 실제 사용값은 -5입니다. 에디터에서 EditDefaultsOnly로 편집 가능하므로 Blueprint에서 다시 변경될 수 있습니다.

ActiveGEHandle 패턴: ActivateAbility에서 ApplyGameplayEffectSpecToSelf의 반환값을 ActiveGEHandle에 보관합니다. EndAbility에서 이 핸들로 RemoveActiveGameplayEffect를 호출하여 효과를 명시적으로 제거합니다. DeleteEnergyEffectClass가 Infinite Duration GE인 경우, 어빌리티 종료 시 효과도 함께 제거되어야 하므로 이 패턴이 필요합니다.

UI 브로드캐스트: ActivateAbility 내에서 GE 적용 후 FuelAttributeSet에서 현재 Fuel을 읽어 AAO_Train의 OnFuelChangedDelegate를 직접 Broadcast합니다. 이는 GAS의 AttributeValueChange 델리게이트와 별도로 수동 알림을 추가한 것입니다.
