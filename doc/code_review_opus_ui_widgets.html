UAO_UserWidget
부모: UUserWidget
역할
프로젝트 전체 위젯의 공통 기반 클래스로, DataTable 기반 UI 사운드 재생, ESC 닫기 이벤트 인터페이스, 호버 불투명도 기본값을 제공합니다.
부모 클래스
UUserWidget (엔진 기본 위젯)
설계 의도
WBP_Base_Modal 등 중간 블루프린트 계층을 통해 Open/Close 애니메이션을 하위 WBP에 일괄 적용하는 확장 포인트
구조체
FAO_SoundRow (FTableRowBase) — SoundAsset, VolumeMultiplier, PitchMultiplier, Description
프로젝트 내 모든 커스텀 위젯의 최상위 부모로 설계된 클래스입니다. WBP_Base_Modal 등 중간 블루프린트 계층을 통해 Open/Close 애니메이션 등 공통 행동을 하위 WBP에 일괄 적용할 수 있습니다.

PlayUISoundFromDataTable은 DataTable에서 FAO_SoundRow를 RowName으로 검색하여 PlaySound2D(IsUISound=true)로 2D 사운드를 재생합니다. SoundDataTable null 체크, Row 미발견 시 화면 디버그 메시지(GEngine->AddOnScreenDebugMessage) 출력을 포함합니다.

OnEscapeCloseRequested는 BlueprintImplementableEvent로, C++에서는 선언만 제공하고 블루프린트에서 ESC 키 닫기 동작을 구현합니다. 이 이벤트는 하위 위젯에서 일관된 ESC 키 처리 인터페이스 역할을 합니다.
위젯 수명주기 참고: 이 클래스 자체는 NativeOnInitialized/NativeConstruct/NativeDestruct 오버라이드가 없습니다. 하위 클래스(ConfirmQuit, ConfirmReturn 등)가 이 수명주기 메서드를 오버라이드하며, Super 호출을 통해 UUserWidget의 기본 동작이 보장됩니다.
설계 관찰: ConfirmQuit/ConfirmReturn/Spectate/Health/Stamina/NameTag만 이 클래스를 상속합니다. 반면 HostDialog/PasswordDialog/MainMenu/LobbyList/LobbyEntry/PauseMenu/ReadyBoard/ReadyBoardRow는 UUserWidget을 직접 상속하여, PlayUISoundFromDataTable이나 HoverOpacity 등 공통 기능을 사용하지 못합니다. 의도적 설계인지 확인이 필요합니다.

리팩토링 이력
기존 BindWidget + UButton 직접 바인딩이 WBP Application Btn 방식으로 교체되어 주석 처리됨
게임 종료(Quit) 전 사용자에게 확인을 요청하는 모달 팝업입니다. 기존 UButton 직접 바인딩(BindWidget) 코드는 WBP Application Btn 방식으로 교체되어 주석 처리된 상태입니다. 현재는 블루프린트에서 HandleClicked_Confirm/Cancel을 BlueprintCallable로 호출합니다.

ESC 키는 NativeOnPreviewKeyDown에서 UIActionKeySubsystem의 IsUICloseKey 검사를 통해 Cancel과 동일하게 동작합니다. 키 이벤트 처리 체인: GetOwningPlayer() → GetGameInstance() → GetSubsystem<UAO_UIActionKeySubsystem>() → IsUICloseKey(Key). 어느 단계에서든 null이면 Super::NativeOnPreviewKeyDown으로 폴백합니다.
위젯 수명주기: NativeOnInitialized에서 Super만 호출합니다 (기존 버튼 바인딩 코드는 주석 처리). NativeOnPreviewKeyDown이 키 입력 인터셉트의 핵심이며, ESC 감지 시 HandleClicked_Cancel을 호출하고 FReply::Handled()를 반환하여 이벤트 전파를 차단합니다.

역할
방 이름/비밀번호를 입력받아 세션을 생성(HostSessionAuto)하는 모달 팝업 다이얼로그
부모 클래스
UUserWidget (UAO_UserWidget 아님에 주의)
입력 모드
UIOnly 모드, 방 이름 텍스트박스에 초기 포커스, 마우스 커서 표시
유효성 검사
방 이름: 1~10자, 제어문자(<0x20) 금지. 비밀번호: 최대 4자 (실시간 자동 절삭)
세션 호스팅을 위한 마우스 중심 모달 팝업입니다. NativeConstruct에서 Btn_Backdrop, Txt_RoomName, Txt_Password의 이벤트를 바인딩하고 ApplyModalInput으로 UIOnly 입력 모드를 설정합니다.

방 이름 유효성: IsValidRoomName은 빈 문자열, 10자 초과, 제어문자(U+0020 미만)를 거부합니다. GetTrimmedRoomName은 좌우 공백을 TrimStartAndEnd로 제거합니다.

비밀번호: OnPasswordChanged에서 4자 초과 입력 시 LeftInline(4)으로 실시간 잘라냅니다. GetTrimmedPassword에서도 동일한 4자 제한을 적용합니다. TrimStartAndEnd + LeftInline 이중 처리.

세션 생성: OnClicked_Create에서 유효성 검증 후 OnlineSessionSubsystem->HostSessionAuto(4, RoomName, Password)를 호출합니다. 최대 인원이 4로 하드코딩되어 있습니다.

Enter 키: OnRoomNameCommitted에서 ETextCommit::OnEnter 감지 시 OnClicked_Create()를 직접 호출하여 Enter 키로도 방 생성이 가능합니다.
NativeConstruct 중복 바인딩 위험: Btn_Backdrop에는 IsAlreadyBound 체크가 있지만, Txt_RoomName과 Txt_Password의 OnTextChanged/OnTextCommitted에는 중복 바인딩 방지가 없습니다. NativeConstruct가 여러 번 호출되는 시나리오(예: SetVisibility 토글)에서 델리게이트 다중 등록이 발생할 수 있습니다.

SetSpectatingPlayerName, BeforeSpectatingCharacterChanged, AfterSpectatingCharacterChanged (BlueprintImplementableEvent)
관전 모드에서 관전 대상 캐릭터의 정보를 표시합니다. SetSpectatingCharacter가 핵심 함수로, 관전 대상 변경 시 Before/After 이벤트를 발행하고 HealthWidget/StaminaWidget을 새 캐릭터에 리바인딩합니다.
P0 크래시 위험: SetSpectatingCharacter에서 checkf(InCharacter, TEXT("InCharacter is nullptr"))를 사용합니다. checkf는 조건이 false일 때 Development/Debug 빌드에서 크래시(assertion failure)를 발생시키고, Shipping 빌드에서는 무시됩니다. 그러나 이후 코드에서 InCharacter를 역참조하므로 Shipping에서도 nullptr 크래시가 발생합니다. nullptr InCharacter에 대한 우아한 처리(early return 등)가 필요합니다.

또한 BeforeSpectatingCharacterChanged(SpectatingCharacter)가 checkf 이전에 호출되므로, 이전 SpectatingCharacter가 nullptr(첫 호출 시)일 때 BlueprintImplementableEvent에 nullptr이 전달됩니다. BP에서 이를 처리하지 않으면 문제가 될 수 있습니다.
