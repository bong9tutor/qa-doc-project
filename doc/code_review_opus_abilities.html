UAO_GameplayAbility_Death
부모: UGameplayAbility
역할
플레이어 사망 처리 어빌리티 — 사망 몽타주 재생, 이동 차단, 래그돌 전환까지 관리
GAS 카테고리
Ability.State.Death · InstancedPerActor
실행 위치
ServerOnly — 서버에서만 활성화되어 권한 있는 상태에서 사망 로직 실행
주요 책임
HitReact 취소, 사망 사운드 재생, Death/Block 태그 적용, 래그돌 이벤트 대기 및 전환
체력이 0이 되면 서버에서 활성화되는 사망 전용 어빌리티입니다. 생성자에서 InstancingPolicy를 InstancedPerActor, NetExecutionPolicy를 ServerOnly로 설정하고, Ability.State.Death 태그를 AssetTag로 등록합니다. 활성화 시 CommitAbility → IsNetAuthority 검사 → Status.Death 중복 확인 순서로 진행합니다. 진행 중인 HitReact 어빌리티를 Ability.State.HitReact 태그로 CancelAbilities() 호출하여 취소하고, GameplayCue.Player.Death를 ExecuteGameplayCue로 실행하여 사망 사운드를 재생합니다. UAbilityTask_WaitGameplayEvent 태스크를 생성하여 RagdollEventTag에 해당하는 이벤트를 대기하며, 동시에 DeathSpectateComponent의 MulticastRPC_PlayDeathMontage()로 모든 클라이언트에서 사망 몽타주를 재생합니다. 서버 권한으로 CharacterMovement의 StopMovementImmediately() → DisableMovement()를 호출하고, DeathTagEffectClass와 BlockAbilitiesEffectClass를 MakeOutgoingGameplayEffectSpec → ApplyGameplayEffectSpecToOwner 체인으로 적용합니다. DeathMontage->GetPlayLength()를 폴백 시간으로 사용하는 타이머를 설정하여, 래그돌 이벤트를 수신하지 못하더라도 FinalizeDeath()가 반드시 호출되도록 보장합니다. FinalizeDeath()는 bDeathFinalizeCalled 플래그로 이중 호출을 방지하고, ASC->CurrentMontageStop(0.05f)으로 몽타주를 즉시 정지한 뒤 DeathSpectateComponent->MulticastRPC_EnterRagdoll()로 래그돌 상태로 전환합니다.

BlockAbilitiesEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly BPReadOnly 사망 중 다른 어빌리티 활성화를 차단하는 GameplayEffect 클래스. DeathTagEffectClass와 동일한 적용 방식을 사용합니다.

UAO_GameplayAbility_HitReact
부모: UGameplayAbility
역할
피격 리액션 전용 어빌리티 — 피격 방향에 따른 몽타주 재생과 무적/차단 효과 적용
GAS 카테고리
Ability.State.HitReact · InstancedPerActor · Event Trigger(Event.Combat.HitReact)
실행 위치
ServerInitiated — 서버에서 시작하여 클라이언트로 복제, 양쪽에서 몽타주 재생
주요 책임
피격 방향 계산(DotProduct), 방향별 몽타주 선택/재생, 무적+차단 GE 적용/제거, 이동 비활성화/복원
Event.Combat.HitReact 이벤트를 AbilityTrigger로 발동하는 피격 리액션 어빌리티입니다. 생성자에서 InstancedPerActor, ServerInitiated를 설정하고, FAbilityTriggerData에 Event.Combat.HitReact 태그를 GameplayEvent 소스로 등록합니다. Ability.State.HitReact를 AssetTag로, Status.Death를 ActivationBlockedTag로 설정합니다. 활성화 시 TriggerEventData에서 이벤트 태그(예: Event.Combat.HitReact.Heavy)를 BaseTag로 추출하고, GetHitDirectionSuffix()로 Front/Back/Left/Right 방향을 결정합니다. 방향 접미사를 BaseTag에 FString::Printf로 결합하여 FinalTag(예: Event.Combat.HitReact.Heavy.Front)를 생성하고, HitReactMontageMap에서 FinalTag → BaseTag → DefaultHitReactMontage 순서로 3단계 폴백으로 몽타주를 선택합니다. PlayMontageAndWait 태스크로 재생하며, OnCompleted/OnBlendOut에 OnMontageCompleted, OnCancelled/OnInterrupted에 OnMontageCancelled를 바인딩합니다. 서버(HasAuthority)에서만 CharacterMovement->DisableMovement()로 이동을 차단하고, InvulnerableEffectClass와 BlockAbilitiesEffectClass를 적용하여 핸들을 저장합니다. EndAbility에서 서버 권한으로 MOVE_Walking 복원, 핸들로 효과를 제거 후 Invalidate합니다.

ActivateAbility() ▶
내부 흐름
1. CommitAbility() — 실패 시 EndAbility(bWasCancelled=true).
2. checkf(DefaultHitReactMontage) — null이면 크래시합니다.
3. TriggerEventData에서 BaseTag = EventTag 추출.
4. GetHitDirectionSuffix(TriggerEventData, ActorInfo) 호출 → FString DirectionSuffix.
5. BaseTag가 유효하면 FString::Printf("%s.%s", BaseTag, DirectionSuffix)로 FinalTag 생성 → RequestGameplayTag()로 변환.
6. 몽타주 선택 3단계: HitReactMontageMap.Find(FinalTag) → HitReactMontageMap.Find(BaseTag) → DefaultHitReactMontage.
7. UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy() 태스크 생성.
8. OnCompleted/OnBlendOut → OnMontageCompleted, OnCancelled/OnInterrupted → OnMontageCancelled 바인딩 후 ReadyForActivation().
9. HasAuthority(&ActivationInfo) 블록 내: Character->GetCharacterMovement()->DisableMovement().
10. InvulnerableEffectClass가 유효하면 MakeOutgoingGameplayEffectSpec → ApplyGameplayEffectSpecToOwner → InvulnerableEffectHandle에 저장.
11. BlockAbilitiesEffectClass가 유효하면 동일 패턴으로 적용 → BlockAbilitiesEffectHandle에 저장.
네트워크 타이밍
ServerInitiated이므로 서버에서 활성화가 시작되고 클라이언트에 복제됩니다. 몽타주 재생 태스크는 양쪽에서 실행되어 클라이언트에서도 시각적 피격 리액션이 표시됩니다. GE 적용과 이동 비활성화는 서버에서만 수행됩니다.

GetHitDirectionSuffix() ▶
시그니처
FString GetHitDirectionSuffix(const FGameplayEventData* TriggerEventData, const FGameplayAbilityActorInfo* ActorInfo) const — 반환값: FString("Front"/"Back"/"Left"/"Right")
내부 흐름
1. TriggerEventData->Instigator를 InstigatorActor로, ActorInfo->AvatarActor를 TargetActor로 추출합니다. 어느 쪽이든 null이면 "Front"를 반환합니다.

3. TargetActor->GetActorForwardVector()를 Forward로, Z=0 평면화 후 Normalize합니다.

5. ForwardDot = DotProduct(Forward, ToInstigator), RightDot = DotProduct(Right, ToInstigator).

상호작용 실행 어빌리티 — 홀딩 관리, 몽타주 재생, AnimNotify 처리, Finalize 이벤트 전송
GAS 카테고리
Ability.Action.AbilityInteract.Execute · InstancedPerActor · Event Trigger · ReplicateYes
실행 위치
ServerInitiated · ReplicateYes — 서버에서 시작, 클라이언트로 복제. Server RPC 2개 포함
주요 책임
Duration 홀딩 UI/타이머, Toggle/OneShot 분기, 몽타주 멀티캐스트, DecayHold 전용 로직, Server RPC 동기화
상호작용 키 입력 시 GameplayEvent로 발동되는 실행 어빌리티입니다. UAO_GameplayAbility_Interact_Info를 상속하여 InitializeAbility()로 타겟 액터의 InteractionInfo를 초기화합니다. 생성자에서 Manual, InstancedPerActor, ServerInitiated, ReplicateYes를 설정하고, CDO(RF_ClassDefaultObject)에서 AbilityTag, ActivationOwnedTag(Status.Action.AbilityInteract), AbilityTrigger(Execute 태그, GameplayEvent 소스)를 등록합니다. 활성화 시 CommitAbility → Super::ActivateAbility → InitializeAbility(TargetActor)를 수행합니다. ASC->GenericGameplayEventCallbacks에 Finalize 태그 리스너를 서버/클라이언트 분기로 등록합니다. DecayHoldElement이면 입력 해제 태스크만 생성하고 반환합니다. Duration=0이면 즉시 ExecuteInteraction(), 양수이면 이동 중지 → Progress UI → HoldMontage → WaitForInvalidInteraction(각도/거리) + WaitInteractionInputRelease 태스크 → DurationTimer를 설정합니다. ExecuteInteraction에서 PuzzleElement/BaseInteractable의 Toggle 여부에 따라 Active/Deactivate 몽타주를 선택하고, InteractionComponent->MulticastPlayInteractionMontage()로 재생합니다. bWaitForAnimationNotify가 false이면 즉시 SendFinalizeEvent()를 호출합니다. SendFinalizeEvent에서 ASC->TriggerAbilityFromGameplayEvent(AbilityToGrant)로 결과 어빌리티를 활성화합니다. ServerNotifyDecayHoldInputReleased(Server, Reliable)과 ServerNotifyAnimationEvent(Server, Reliable) 2개의 Server RPC로 클라이언트-서버 동기화를 처리합니다.
변수명타입지정자설명
AcceptanceAngle float EditDefaultsOnly 홀딩 중 허용되는 최대 시야각(도). 초과 시 OnInvalidInteraction()이 호출되어 취소합니다. 기본값 120.0f.
AcceptanceDistance float EditDefaultsOnly 홀딩 중 허용되는 최대 거리. 초과 시 취소됩니다. 기본값 100.0f.
DurationTimerHandle FTimerHandle (private) 홀딩 완료 시점을 관리하는 타이머 핸들. TWeakObjectPtr 기반 람다로 OnDurationEnded()를 호출합니다.
MontageTimerHandle FTimerHandle (private) 몽타주 재생 완료 후 어빌리티 종료를 위한 타이머 핸들. MontageLength + 0.5f로 설정됩니다.
bHoldingPhaseCompleted bool (private) 홀딩 단계 완료 여부 플래그. ActivateAbility에서 false로 리셋, OnDurationEnded에서 true로 설정됩니다. InputReleased에서 이 값이 false이면 취소합니다.

내부 흐름

ActivateAbility() ▶
내부 흐름
1. CommitAbility() → Super::ActivateAbility() → bHoldingPhaseCompleted = false.
2. TriggerEventData->Target으로 InitializeAbility() — 실패 시 종료.
3. ASC->GenericGameplayEventCallbacks에 Finalize 태그 리스너 등록: 서버이면 OnAnimNotifyReceived, 클라이언트이면 OnLocalAnimNotifyReceived.
4. WorldInteractable->OnInteractActiveStarted() 호출.
5. DecayHoldElement 분기: WaitInteractionInputRelease 태스크만 생성 후 return.
6. Duration <= 0 분기: bHoldingPhaseCompleted=true → ExecuteInteraction().
7. Duration > 0 분기: StopMovementImmediately() → Progress UI → HoldMontage → WaitForInvalidInteraction(AcceptanceAngle, AcceptanceDistance) → WaitInteractionInputRelease → DurationTimer(WeakLambda).
네트워크 타이밍
ServerInitiated + ReplicateYes이므로 서버에서 활성화가 시작되고 클라이언트에 복제됩니다. 클라이언트에서 AnimNotify를 감지하면 ServerNotifyAnimationEvent RPC로 서버에 전달하고, 서버에서 실제 Finalize 처리를 수행합니다.
EndAbility() ▶
내부 흐름
1. DecayHoldElement이면 MulticastSetMovementForActor(true) — 움직임 복구.
2. WorldInteractable->OnInteractActiveEnded() — 홀딩 종료 알림.
3. DurationTimerHandle/MontageTimerHandle ClearTimer.
4. ASC->GenericGameplayEventCallbacks에서 Finalize 리스너 RemoveAll(this).
5. bHoldingPhaseCompleted = false.
6. InteractionWidgetController에 Notice(bShouldRefresh=false, bSwitchActive=true) 브로드캐스트.
7. Super::EndAbility().

UAO_GameplayAbility_Inspect_Enter
부모: UAO_InteractionGameplayAbility
역할
검사 모드 진입 어빌리티 — 상호작용 대상의 InspectableComponent를 통해 검사 모드에 진입
GAS 카테고리
Interaction/Inspection · InstancedPerActor · ServerOnly
실행 위치
ServerOnly — 서버에서만 실행되어 잠금 상태를 권한 있게 관리, 멀티플레이어 충돌 방지
주요 책임
서버 권한 확인(check()), 다른 플레이어 잠금 확인, InspectableComponent 잠금 설정, InspectionComponent로 검사 모드 진입
GA_Interact_Execute에서 SendFinalizeEvent() 시 발동되는 검사 모드 진입 어빌리티입니다. 에디터에서 검사가 필요한 액터의 AbilityToGrant로 이 클래스를 설정합니다. 생성자에서 InstancedPerActor, ServerOnly를 설정하고, Status.Action.Inspecting을 ActivationOwnedTag로, Ability.Action.AbilityInteract를 BlockAbilitiesWithTag로 추가합니다. 활성화 시 check(ActorInfo->OwnerActor->HasAuthority())로 서버 권한을 확인합니다(실패 시 크래시). CommitAbility → Super::ActivateAbility를 수행합니다. TriggerEventData->Target에서 InspectableComponent를 찾아 IsLockedByOtherPlayer(Instigator)로 다른 플레이어가 이미 사용 중인지 확인합니다. 사용 가능하면 SetInspectionLocked(true, Instigator)로 잠금 처리합니다. Instigator에서 InspectionComponent를 찾아 SetInspectEnterHandle(Handle)로 어빌리티 핸들을 저장하고, EnterInspectionMode(TargetActor)를 호출하여 검사 모드에 진입합니다. 어빌리티가 활성화된 동안 Status.Action.Inspecting 태그가 유지되어 점프 등 다른 어빌리티가 차단됩니다. EndAbility에서는 Super::EndAbility()만 호출하며, 실제 검사 모드 종료와 잠금 해제는 InspectionComponent 측에서 처리합니다.
변수명타입지정자설명
이 클래스는 자체 멤버 변수를 선언하지 않습니다. 부모 UAO_InteractionGameplayAbility의 ActivationPolicy를 사용합니다.
UAO_GameplayAbility_Inspect_Enter() ▶
내부 흐름
1. InstancedPerActor, ServerOnly 설정.
2. AO_InteractionTags::Status_Action_Inspecting을 ActivationOwnedTag에 추가 — 활성화 동안 Inspecting 태그가 부여됩니다.
3. AO_InteractionTags::Ability_Action_AbilityInteract를 BlockAbilitiesWithTag에 추가 — 검사 중 상호작용 어빌리티를 차단합니다.
ActivateAbility() ServerOnly ▶
시그니처
void ActivateAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo, const FGameplayEventData* TriggerEventData)
내부 흐름
1. check(ActorInfo->OwnerActor->HasAuthority()) — 서버 권한 확인, 실패 시 check 크래시.
2. CommitAbility() — 실패 시 EndAbility(bWasCancelled=true).
3. Super::ActivateAbility() 호출.
4. TriggerEventData가 null이거나 Target이 null이면 EndAbility(bWasCancelled=true).
5. TargetActor = const_cast<AActor*>(TriggerEventData->Target.Get()), Instigator = const_cast<AActor*>(TriggerEventData->Instigator.Get()).
6. TargetActor->FindComponentByClass<UAO_InspectableComponent>() — null이면 종료.
7. InspectableComp->IsLockedByOtherPlayer(Instigator) — 다른 플레이어가 잠금 중이면 종료.
8. Instigator->FindComponentByClass<UAO_InspectionComponent>() — null이면 종료.
9. InspectableComp->SetInspectionLocked(true, Instigator) — 이 플레이어가 잠금합니다.
10. InspectionComp->SetInspectEnterHandle(Handle) — 어빌리티 핸들을 저장하여 나중에 EndAbility를 호출할 수 있도록 합니다.
11. InspectionComp->EnterInspectionMode(TargetActor) — 실제 검사 모드에 진입합니다.
호출 시점
GA_Interact_Execute의 SendFinalizeEvent()가 TriggerAbilityFromGameplayEvent()로 이 어빌리티를 트리거합니다.
EndAbility() ▶
내부 흐름
Super::EndAbility()만 호출합니다. 검사 모드 종료와 InspectableComponent 잠금 해제는 InspectionComponent 측에서 SetInspectEnterHandle을 통해 이 어빌리티의 EndAbility를 호출하는 방식으로 처리합니다.
