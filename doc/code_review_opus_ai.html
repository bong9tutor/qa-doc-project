GAS 리플리케이션
EGameplayEffectReplicationMode::Mixed — 소유자에게 전체, 타인에게는 최소 복제
ACharacter와 IAbilitySystemInterface를 다중 상속하여 이동 · 충돌 · GAS를 모두 지원합니다. 생성자에서 AbilitySystemComponent, AIAttributeSet, AIMemoryComponent 세 가지 서브오브젝트를 생성하며, ASC는 SetIsReplicated(true)와 Mixed 리플리케이션 모드로 설정됩니다.

BeginPlay에서 서버/클라이언트를 구분하여 CharacterMovementComponent의 네트워크 스무딩 파라미터를 세밀하게 조정합니다. 서버에서는 bRunPhysicsWithNoController=true로 컨트롤러 없이도 물리 실행이 가능하고, 클라이언트에서는 물리 연산을 최소화하여 성능을 절약합니다. 또한 클라이언트에서는 CapsuleComponent의 Pawn 채널 충돌을 Ignore로 설정하여 서버-클라이언트 간 위치 차이에 의한 밀림 현상을 방지합니다.

기절(Stun)은 GAS 태그 시스템(Status.Debuff.Stunned)으로 상태를 추적하며, 기절 몽타주는 NetMulticast Reliable RPC로 모든 클라이언트에 동기화합니다. TestStun/TestStunEnd는 GAS 이벤트 시스템(Event.AI.Stunned)과 태그 기반 어빌리티 취소(Ability.State.Stunned)를 활용합니다.

InitializeAbilitySystem() → BindDefaultAbilities() → BindDefaultEffects() ▶
하는 일
GAS 시스템을 초기화하고 기본 어빌리티/이펙트를 일괄 등록합니다. 서버에서만 호출됩니다.
InitializeAbilitySystem 상세
ASC->InitAbilityActorInfo(this, this) — OwnerActor와 AvatarActor를 모두 자기 자신으로 설정합니다. 플레이어와 달리 AI는 PlayerState를 사용하지 않으므로 Owner와 Avatar가 동일합니다.
BindDefaultAbilities 상세
DefaultAbilities 배열을 순회하며 각 클래스가 유효한지 확인 후, FGameplayAbilitySpec(AbilityClass)를 생성하여 GiveAbility로 등록합니다. Level 파라미터 없이 기본 생성자를 사용하므로 Level 1로 부여됩니다.
BindDefaultEffects 상세
DefaultEffects 배열을 순회하며 MakeEffectContext()로 컨텍스트를 생성합니다. Context.AddInstigator(this, this)로 가해자를 자기 자신으로 설정한 뒤, MakeOutgoingSpec(EffectClass, 1.f, Context)으로 Level 1 스펙을 생성하고 ApplyGameplayEffectSpecToSelf로 자기 자신에게 적용합니다. 주로 초기 체력 설정, 기본 버프 등에 사용됩니다.
IsStunned() const BlueprintCallable ▶
하는 일
현재 기절 상태인지를 GAS 태그로 확인합니다.
상세 구현
FGameplayTag::RequestGameplayTag(FName("Status.Debuff.Stunned"))로 태그를 요청한 뒤, ASC->HasMatchingGameplayTag(StunnedTag)로 현재 태그 컨테이너에 해당 태그가 존재하는지 확인합니다. ASC가 null이면 ensureMsgf로 경고를 출력하고 false를 반환합니다.
StateTree 연동
StateTree Condition에서 이 함수를 호출하여 기절 중 공격/이동 행동을 억제합니다.
OnStunBegin() / OnStunEnd() BlueprintCallable ▶
하는 일
기절 상태 진입 · 종료 처리를 수행하는 공개 진입점입니다. 내부에서 HandleStunBegin/End()를 호출합니다.
HandleStunBegin 상세 구현
HasAuthority() 확인 후 서버에서만 GetCharacterMovement()->StopMovementImmediately()를 호출합니다. 이동 속도를 0으로 만들어 AI가 기절 중 이동하지 못하도록 합니다. 기절 태그 추가와 몽타주 재생은 GAS 어빌리티(AO_GA_AI_Stun)에서 별도로 처리합니다.
HandleStunEnd 상세 구현
기본 구현은 비어 있으며, 파생 클래스에서 오버라이드하여 종별 기절 해제 로직(이동 재개, 이펙트 제거 등)을 추가합니다.
호출 시점
GAS의 AO_GA_AI_Stun 어빌리티가 시작 시 OnStunBegin(), 종료 시 OnStunEnd()를 호출합니다.
Multicast_PlayStunMontage(Montage, PlayRate) NetMulticast, Reliable ▶
하는 일
기절 애니메이션 몽타주를 서버와 모든 클라이언트에서 동시에 재생합니다.
상세 구현
Montage가 null이면 즉시 반환합니다. GetMesh()->GetAnimInstance()로 AnimInstance를 가져오고, Montage_IsPlaying(Montage)로 이미 재생 중인지 확인합니다. 이미 재생 중이면 중복 재생을 방지하고, 아닌 경우에만 Montage_Play(Montage, PlayRate)를 호출합니다. 재생 시작 후 CurrentStunMontage에 참조를 저장하여 이후 중지에 사용합니다.
NetMulticast, Reliable 의미
NetMulticast: 서버에서 호출하면 서버 자신 + 모든 연결된 클라이언트에서 실행됩니다.
Reliable: 패킷 손실 시 재전송을 보장합니다. 기절 애니메이션은 시각적으로 중요하므로 반드시 재생되어야 합니다.
Multicast_StopStunMontage(BlendOutTime) NetMulticast, Reliable ▶
하는 일
CurrentStunMontage를 BlendOutTime(기본 0.25초) 동안 부드럽게 종료합니다.
상세 구현
CurrentStunMontage가 null이면 즉시 반환합니다. AnimInstance를 가져온 뒤 Montage_IsPlaying(CurrentStunMontage)로 재생 중인지 확인합니다. 재생 중이면 Montage_Stop(BlendOutTime, CurrentStunMontage)으로 블렌드 아웃하며 중지합니다. 마지막으로 CurrentStunMontage를 nullptr로 초기화합니다.
GetCurrentAttackConfig() BlueprintNativeEvent ▶
하는 일
현재 사용할 공격 설정(FEnemyAttackConfig)을 반환합니다.
기본 구현
_Implementation 함수는 빈 FEnemyAttackConfig()를 반환합니다. 즉, 기반 클래스에서는 기본값(Damage=30, KnockbackStrength=500, AttackRadius=150 등)을 가진 설정을 반환하며, 파생 클래스의 Blueprint에서 오버라이드하여 고유한 공격 파라미터를 반환합니다.
BlueprintNativeEvent 의미
C++ 기본 구현(_Implementation 접미사 함수)을 제공하면서, Blueprint에서도 오버라이드가 가능합니다.
StateTree 연동
공격 Task에서 GetCurrentAttackConfig()를 호출하여 AttackRadius, AttackDistance, Damage를 동적으로 가져옵니다.
TestStun() / TestStunEnd() CallInEditor ▶
하는 일
에디터 디테일 패널에서 버튼으로 기절 시작/종료를 테스트합니다.
TestStun 상세 구현
FGameplayTag::RequestGameplayTag(FName("Event.AI.Stunned"))로 이벤트 태그를 생성합니다. FGameplayEventData를 구성하여 Instigator와 Target을 모두 자기 자신으로 설정합니다. ASC->HandleGameplayEvent(StunEventTag, &EventData)로 GAS 이벤트를 발행하면, 이 이벤트를 대기하고 있는 AO_GA_AI_Stun 어빌리티가 자동으로 활성화됩니다.
TestStunEnd 상세 구현
Ability.State.Stunned 태그를 FGameplayTagContainer에 추가한 뒤, ASC->CancelAbilities(&StunAbilityTags)로 해당 태그를 가진 모든 활성 어빌리티를 취소합니다. 이로써 기절 어빌리티의 EndAbility가 호출되고 기절 상태가 해제됩니다.

HeardMemoryDuration = 10초 (EditDefaultsOnly)
UActorComponent를 상속하며, AAO_AICharacterBase에 CreateDefaultSubobject로 부착됩니다. 플레이어별 기억 데이터(FAO_PlayerMemoryData)를 TMap에 저장하고, 키로 TWeakObjectPtr<AAO_PlayerCharacter>를 사용하여 플레이어 파괴 시에도 댕글링 포인터가 발생하지 않습니다.

시각 기억과 청각 기억은 별도의 유효성 검증 로직을 사용합니다. 시각 기억은 IsMemoryValid(Player, MaxAge)에서 LostTime 기준으로 경과 시간을 계산하고, 청각 기억은 GetLastHeardLocation() 내부에서 HeardMemoryDuration과 비교하여 만료 시 ZeroVector를 반환합니다. 모든 조회 함수에서 null Player에 대한 ensureMsgf 검증이 포함되어 있어 디버깅 시 문제 원인을 추적할 수 있습니다.

Crab AI는 GetLastLostLocation/GetRecentLostLocations로 플레이어가 마지막에 있던 위치를 피하고, 선공형 AI(Stalker 등)는 GetLastKnownLocation으로 플레이어를 추적합니다.
FAO_PlayerMemoryData (USTRUCT, BlueprintType)
LastKnownLocation FVector (기본값 ZeroVector) — 시야 내에 있을 때 매 호출마다 갱신되는 최신 확인 위치. UpdatePlayerLocation()이 기록합니다.
LastSeenTime float (기본값 0) — 마지막으로 플레이어를 시야에서 확인한 게임 시간. GetWorld()->GetTimeSeconds()로 기록됩니다.
LastLostLocation FVector (기본값 ZeroVector) — 플레이어가 시야에서 사라진 순간의 위치. SetLastKnownLocation()이 시야 소실 시점에 기록합니다.
LostTime float (기본값 0) — 시야에서 사라진 시간. IsMemoryValid()에서 (CurrentTime - LostTime)으로 경과 시간을 계산합니다.
bIsInSight bool (기본값 false) — 현재 이 플레이어가 AI 시야 내에 있는지 여부. UpdatePlayerLocation에서 true, SetLastKnownLocation에서 false로 전환됩니다.
변수명타입지정자설명
PlayerMemories TMap<TWeakObjectPtr<AAO_PlayerCharacter>, FAO_PlayerMemoryData> (UPROPERTY) 플레이어 포인터 → 기억 데이터 매핑. FindOrAdd로 자동 생성되므로 초기화가 필요 없습니다. TWeakObjectPtr을 키로 사용하여 GC 안전성을 보장합니다.

IsMemoryValid(Player, MaxAge) BlueprintCallable ▶
하는 일
해당 플레이어의 기억이 MaxAge 초 이내에 갱신된 것인지 확인합니다.
상세 구현

AActor를 상속하며 레벨에 직접 배치합니다. BeginPlay에서 HasAuthority() 또는 IsPlayInEditor()를 확인하여 서버/에디터에서만 스폰 로직을 시작합니다. SpawnInterval(±SpawnIntervalRandomDeviation) 간격의 비반복 타이머를 설정하여 TrySpawnAI()를 호출하고, 각 함수 끝에서 다음 타이머를 재등록하는 체인 방식으로 동작합니다.

EQS 쿼리 결과에서 점수 기반 정렬 후 상위 25%(최소 1개, 최대 5개)를 후보군으로 선정하고 이 중 랜덤으로 하나를 선택하여 같은 위치에 반복 스폰되는 것을 방지합니다. 선택된 위치는 NavigationSystemV1로 NavMesh에 프로젝션하고, 해당 몬스터 클래스의 CDO에서 CapsuleComponent 높이를 읽어 Z축을 보정하여 바닥 끼임을 방지합니다.

ExecuteSpawn에서 스폰 후 Controller가 없으면 캐릭터의 AIControllerClass를 확인하여 자동으로 AIController를 스폰하고 Possess합니다. AIControllerClass가 미설정이면 기본 AAIController를 사용합니다.

IntensiveVolumes의 각 볼륨은 별도 타이머로 자체 스폰 간격을 가지며, 볼륨 내 MaxMonstersInArea, AllowedMonsterClasses 등 독립 설정을 지원합니다. NavArea 기반 집중 스폰도 지원하지만, CheckIntensiveNavAreas의 NavModifierVolume 탐색 로직은 미완성 상태입니다.

③ 상위 25% (최소 1개, 최대 5개)를 후보군으로 선정합니다.
④ FMath::RandRange(0, TopCount-1)로 후보군 중 하나를 랜덤 선택합니다.
이 방식은 항상 최고 점수 위치에 스폰하는 것을 방지하면서도, 품질이 낮은 위치를 배제합니다.
NavMesh 프로젝션 및 높이 보정
NavigationSystemV1->ProjectPointToNavigation(SpawnLocation, ProjectedLocation, FVector(500, 500, 500))으로 NavMesh 위의 유효 위치로 보정합니다. 성공 시 선택된 MonsterClass의 CDO(Class Default Object)에서 CapsuleComponent의 ScaledCapsuleHalfHeight를 읽어 SpawnLocation.Z += CapsuleHalfHeight + 5.0f로 Z축을 보정하여 바닥에 끼이는 것을 방지합니다.
몬스터 선택
MonsterClasses[FMath::RandRange(0, Num-1)]로 랜덤 선택합니다.
ExecuteSpawn(SpawnLocation, MonsterClass) — 스폰 · Controller 생성 ▶
하는 일
AI를 실제로 월드에 스폰하고, Controller가 없으면 자동 생성하여 Possess합니다.
상세 구현
스폰 파라미터: SpawnCollisionHandlingOverride = AdjustIfPossibleButAlwaysSpawn (위치 보정 시도 후 무조건 스폰), Owner = this.
SpawnRotation = ZeroRotator (방향 지정 없음).
SpawnActor<AAO_AICharacterBase>(MonsterClass, SpawnLocation, SpawnRotation, SpawnParams)로 스폰합니다.

스폰 성공 후:
① SpawnedMonsters.Add(SpawnedMonster) — 추적 목록에 추가
② OnDestroyed.AddDynamic(this, OnAIDestroyed) — 파괴 델리게이트 바인딩
③ GetController()가 null이면 Controller 자동 생성: AIControllerClass가 설정되어 있으면 해당 클래스를, 미설정이면 AAIController::StaticClass()를 사용합니다. SpawnActor<AController>로 Controller를 생성(AlwaysSpawn)한 뒤 NewController->Possess(SpawnedMonster)를 호출합니다.

OnAIDestroyed(DestroyedActor) ▶
하는 일
스폰된 AI가 파괴될 때 호출되는 델리게이트 콜백입니다.
상세 구현
