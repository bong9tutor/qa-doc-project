AAO_AIControllerBase
부모: AAIController
역할
모든 AI Controller의 공통 베이스 — Perception, StateTree 통합, 팀 시스템
부모 클래스
AAIController (IGenericTeamAgentInterface 포함)
실행 위치
서버에서만 실행 (HasAuthority 체크). 클라이언트에서는 BeginPlay/OnPossess가 조기 리턴
StateTree 통합
UStateTreeAIComponent를 생성자에서 생성. OnPossess에서 DefaultStateTree 할당 후 StartLogic() 호출
Perception 구성
Sight + Hearing 두 감각. DominantSense = Sight. 모든 Affiliation 감지 (Enemy/Friendly/Neutral)
NavMesh 필터링
IsPlayerOnNavMesh(저비용, ProjectPoint) + CanReachPlayer(고비용, FindPathSync) 이원화 체계
AAIController를 상속하여 시야(Sight)와 청각(Hearing) 두 가지 AIPerception 감각을 설정합니다. 생성자에서 UAIPerceptionComponent와 UStateTreeAIComponent를 CreateDefaultSubobject로 생성하고 SetPerceptionComponent로 등록합니다.

BeginPlay 흐름: HasAuthority() 확인 후 SetupPerceptionSystem()을 호출하여 SightConfig(SightRadius=1500, LoseSightRadius=2000, 시야각 90도)와 HearingConfig(HearingRange=2000)를 NewObject로 생성하여 ConfigureSense에 전달합니다. DominantSense를 Sight로 설정한 뒤 RequestStimuliListenerUpdate()를 호출합니다. OnTargetPerceptionUpdated 델리게이트를 바인딩하여 모든 감지 이벤트를 수신합니다.

OnPossess 흐름: Super::OnPossess 후 HasAuthority 체크, StateTreeComponent에 DefaultStateTree를 SetStateTree로 할당하고 StartLogic()으로 행동 트리를 가동합니다. OnUnPossess에서는 StopLogic("UnPossessed")로 정지 후 Super::OnUnPossess를 호출합니다.

OnTargetPerceptionUpdated 내부 흐름: 시야 감지 시 플레이어 캐스팅 → PlayerState에서 GetIsAlive() 확인 → bCanTargetDeadPlayer 체크 → AISubsystem::IsPlayerBeingKidnapped() 납치 상태 확인 → WasSuccessfullySensed() 분기로 OnPlayerDetected/OnPlayerLost 호출. 플레이어가 아닌 액터는 OnActorDetected/OnActorLost로 분배합니다. 청각 감지 시 OnNoiseHeard를 호출합니다.

GetPlayersInSight/GetNearestPlayerInSight/HasPlayerInSight는 PlayersInSight 배열을 순회하며 죽은 플레이어(bCanTargetDeadPlayer 기반), 납치 중 플레이어(AISubsystem 질의), NavMesh 도달 불가능 플레이어(bFilterUnreachablePlayers 기반)를 3중 필터링합니다. GetNearestPlayerInSight는 FVector::DistSquared로 최근접 플레이어를 선택합니다.

IGenericTeamAgentInterface를 통해 팀 시스템을 구현하며, AI는 TeamId=1, 플레이어 캐릭터는 Hostile, 같은 팀 AI는 Friendly, 그 외는 Neutral을 반환합니다.

UAO_AISubsystem

역할
AI 상태 관리 서브시스템 — 아이템 예약, 드롭 쿨다운, 납치 상태 관리
부모 클래스
UWorldSubsystem (월드 단위 싱글턴, ShouldCreateSubsystem에서 IsGameWorld() 확인)
타이머 정리 주기
5초마다 CreateWeakLambda로 3개 Cleanup 함수 호출 (만료 예약, 드롭 쿨다운, 납치 쿨다운)
유틸리티
GetAllPlayerLocations (GetAllActorsOfClass), FindLocationFarthestFromPlayers (16방향 NavMesh 샘플링), GetAllStalkerLocations
UWorldSubsystem을 상속하며 게임 월드에서만 생성됩니다. Initialize에서 World->GetTimerManager().SetTimer로 5초 루핑 타이머를 등록하여 CleanupExpiredReservations, CleanupExpiredDropCooldowns, CleanupExpiredKidnapCooldowns를 주기적으로 실행합니다. Deinitialize에서 타이머 해제 후 모든 TMap을 Empty()로 정리합니다.

아이템 예약: TryReserveItem은 ensureMsgf로 null 검증 후 기존 예약자 확인(다른 AI가 예약했으면 false). 동일 Reserver는 재예약 허용. ReservationTimeout=30초 후 CleanupExpiredReservations에서 자동 해제됩니다.

드롭 쿨다운: MarkItemAsRecentlyDropped로 CooldownDuration(기본 20초) 동안 같은 아이템 재픽업 방지. IsItemRecentlyDropped에서 (CurrentTime - DropTime) < CooldownDuration으로 판정합니다.

납치 예약: TryReservePlayerForKidnap은 ① IsPlayerBeingKidnapped 확인 → ② IsPlayerRecentlyKidnapped 쿨다운 확인 → ③ Double-check 후 KidnappedPlayers.Add. ReleasePlayerFromKidnap + MarkPlayerAsRecentlyKidnapped로 해제 시 쿨다운 적용.

FindLocationFarthestFromPlayers: 16방향 등간격 샘플링으로 Origin + 방향 * SearchRadius 위치를 ProjectPointToNavigation으로 검증, 각 위치에서 모든 플레이어까지 최소 거리를 계산하여 Maximin 전략으로 가장 안전한 위치를 반환합니다.

클램프 정책
PreAttributeChange에서 MovementSpeed를 [0, MaxMovementSpeed] 범위로 클램프
UAttributeSet을 상속하며 MovementSpeed(기본 300)와 MaxMovementSpeed(기본 600) 두 가지 어트리뷰트를 정의합니다. 생성자에서 InitMovementSpeed(300.f), InitMaxMovementSpeed(600.f)로 초기화합니다.

GetLifetimeReplicatedProps에서 DOREPLIFETIME_CONDITION_NOTIFY 매크로로 두 어트리뷰트 모두 COND_None(항상), REPNOTIFY_Always(항상 통지)로 등록합니다. OnRep_MovementSpeed/OnRep_MaxMovementSpeed에서 GAMEPLAYATTRIBUTE_REPNOTIFY 매크로로 GAS 리플리케이션 시스템에 변경을 통지합니다.

PreAttributeChange에서 MovementSpeed가 변경될 때 [0, GetMaxMovementSpeed()] 범위로 FMath::Clamp합니다. MaxMovementSpeed 변경은 별도 클램프 없이 허용됩니다.

MovementSpeed FGameplayAttributeData BPReadOnlyReplicated 현재 이동 속도 (기본 300). ReplicatedUsing = OnRep_MovementSpeed. PreAttributeChange에서 [0, MaxMovementSpeed] 클램프.
