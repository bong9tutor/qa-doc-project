// 세션 탐색
void UAO_OnlineSessionSubsystem::FindSessions(int32 MaxSearchResults)
{
    LastSessionSearch = MakeShareable(new FOnlineSessionSearch());
    LastSessionSearch->MaxSearchResults = MaxSearchResults;
    LastSessionSearch->bIsLanQuery = false;
    LastSessionSearch->QuerySettings.Set(
        SEARCH_PRESENCE, true, EOnlineComparisonOp::Equals);
    SessionInterface->FindSessions(*UserId, LastSessionSearch.ToSharedRef());
}

Seamless Travel 라이프사이클
ServerTravel 호출 시 내부에서 일어나는 일
bUseSeamlessTravel = true (기본값)
AAO_GameMode_InGameBase 생성자에서 bUseSeamlessTravel = true를 설정합니다. 이 플래그가 켜져 있으면 ServerTravel 시:
  • PlayerController 보존 — PC가 파괴되지 않고 새 맵으로 이전됩니다.
  • PlayerState 보존 — CopyProperties()가 호출되어 기존 PS 데이터를 새 PS로 복사합니다.
  • TransitionMap 경유 — 엔진이 내부적으로 LV_Transition 맵을 거쳐 새 맵을 비동기 로드합니다.
  • HandleSeamlessTravelPlayer() — 새 맵의 GameMode에서 각 이동된 Controller에 대해 호출됩니다.
bUseSeamlessTravel이 false이면 모든 PlayerController와 PlayerState가 파괴됩니다. 클라이언트가 새 맵에 재접속해야 하므로 세션 데이터가 유실됩니다.

OnRep 콜백 타이밍
Seamless Travel 후 클라이언트 측 OnRep_* 콜백은 새 맵의 BeginPlay 이후에 발화합니다. 이 사이 짧은 시간 동안 Replicated 프로퍼티가 아직 동기화되지 않은 상태일 수 있으므로, AAO_GameState::AddPlayerState()에서 0.1초 타이머 딜레이를 사용하는 패턴이 존재합니다.
QA 포인트: Travel 직후 OnRep이 아직 호출되지 않은 시점에서 UI를 갱신하면 null 참조나 stale 데이터가 표시될 수 있습니다. 이를 재현하려면 고지연 환경에서 테스트하세요.
동기화된 Travel (Voice Crash 방지)
직접 ServerTravel()을 호출하면 Voice 시스템이 정리되기 전에 맵 전환이 시작되어 크래시가 발생합니다. 이를 방지하기 위해 RequestSynchronizedServerTravel() 패턴을 사용합니다: 
// 1. 서버가 모든 PC에게 Travel 준비 요청
for (PC : AllPlayerControllers)
    PC->Client_PrepareForTravel(URL);  // Client RPC

// 2. 각 클라이언트가 VoiceChat 정리 후 서버에 완료 통보
PC->NotifyPlayerCleanupCompleteForTravel(PC);

// 3. 모든 플레이어 완료 확인 후 실제 Travel 실행
if (CleanupCompletePlayers.Num() >= TotalPlayers)
    World->ServerTravel(PendingTravelURL);
데이터 영속성
ServerTravel을 거쳐도 유지되는 데이터와 그 메커니즘
데이터 저장 위치 영속 메커니즘 초기화 시점
PersistentInventory AAO_PlayerState (Replicated) SaveInventoryBeforeTravel() → CopyProperties() SafeZone 밖이면 삭제
PersistentHealth AAO_PlayerState (Replicated) SaveHealthBeforeTravel() → CopyProperties() ResetStateHealth()
SharedReviveCount UAO_GameInstance GameInstance는 Travel 간 영속 → GameState로 동기화 ResetRun()
SharedTrainFuel UAO_GameInstance GameInstance에 직접 보관 (서버 기준 값) ResetRun() → FuelDataAsset 초기값
CurrentStageIndex UAO_GameInstance GameInstance에 직접 보관 (0, 1, 2 ...) ResetRun() → 0
CharacterCustomizingData AAO_PlayerState (Replicated) CopyProperties()에서 복사 세션 종료 시
DeathCount AAO_PlayerState (Replicated) CopyProperties() — 로비 맵이면 0으로 초기화 로비 진입 시
LobbyHostNetIdStr UAO_GameInstance GameInstance에 직접 보관 ResetSessionData()

QA 포인트: 스테이지에서 SafeZone 밖에 있는 상태로 맵 전환이 발생하면 인벤토리가 의도적으로 삭제됩니다. Rest Room에서는 위치와 관계없이 항상 유지됩니다.
스테이지 진행 패턴
승리 시 TryAdvanceStageIndex()가 호출되어 CurrentStageIndex를 1 증가시킵니다. AO_MapRoutes::GetStageMapName(CurrentStageIndex)로 다음 맵 이름을 결정합니다. 
// GameInstance — 스테이지 진행
bool TryAdvanceStageIndex()
{
    if (CurrentStageIndex + 1 < StageCount) {
        ++CurrentStageIndex;   // 0→1→2
        return true;
    }
    return false;  // 마지막 스테이지 → 더 이상 없음
}
세션 라운드 사이클

전체 흐름 다이어그램

KEY_IN_GAME — 중간 입장 차단
로비에서 스테이지로 이동 시 SetSessionInGame(true)가 호출되어 Steam 세션 설정을 갱신합니다: 
// 세션을 "인게임" 상태로 설정
void SetSessionInGame(bool bInGame)
{
    Settings->Set(KEY_IN_GAME, bInGame, ViaOnlineServiceAndPing);
    Settings->bAllowJoinInProgress  = !bInGame;
    Settings->bAllowJoinViaPresence = !bInGame;
    Session->UpdateSession(NAME_GameSession, *Settings);
}
KEY_IN_GAME = true일 때 세션 검색 결과에 "진행 중"으로 표시되며, bAllowJoinInProgress와 bAllowJoinViaPresence가 모두 false로 설정되어 새로운 플레이어의 참가가 차단됩니다.
게임 실패로 로비 복귀 시 RollbackSessionInGameFlag()에서 SetSessionInGame(false)를 호출합니다. 이 호출이 누락되면 로비에서도 참가가 차단되는 버그가 발생할 수 있습니다.
RestRoom에서 다음 스테이지 선택
HandleRestExitRequest()가 호출되면:
  1. TryAdvanceStageIndex()로 다음 스테이지 인덱스 증가 시도
  2. 성공 시 → GetCurrentStageMap()으로 맵 이름 획득 → RequestSynchronizedServerTravel()
  3. 실패 시 (마지막 스테이지 이후) → ResetRun() + 로비로 복귀
맵 라우팅은 AO_MapRoutes 헬퍼가 담당합니다. GetStageMapName(index)로 인덱스에 대응하는 맵 경로를 반환합니다.
음성 채팅 초기화
Steam Voice 시스템의 시작/종료 및 플레이어 입장 처리
StartVoiceChat / StopVoiceChat 라이프사이클
UAO_OnlineSessionSubsystem에서 관리합니다. 
// 시작: Local Talker 등록 + 네트워크 전송 시작
void StartVoiceChat()
{
    VoiceInterface->RegisterLocalTalker(0);
    VoiceInterface->StartNetworkedVoice(0);
}

// 종료: 전송 중단 + 패킷 정리 + Remote 제거 + Unregister
void StopVoiceChat()
{
    VoiceInterface->StopNetworkedVoice(0);
    VoiceInterface->ClearVoicePackets();
    VoiceInterface->RemoveAllRemoteTalkers();
    for (i = 0..MAX_LOCAL_PLAYERS)
        VoiceInterface->UnregisterLocalTalker(i);
}
레벨 이동 시 AAO_PlayerState::BeginPlay() → InitVoiceChat()에서 사용자 설정에 따라 UnmuteAllRemoteTalker() 및 StartVoiceChat()을 재호출합니다.
플레이어 입장 시 0.1초 딜레이 패턴
AAO_GameState::AddPlayerState()에서 새 플레이어가 추가될 때, 즉시 UnmuteRemoteTalker()를 호출하면 내부 데이터(UniqueNetId 등)가 아직 준비되지 않아 null 참조가 발생합니다. 이를 방지하기 위해 0.1초 타이머 딜레이를 사용합니다: 
// GameState — 플레이어 입장 시 Unmute 처리
void AAO_GameState::AddPlayerState(APlayerState* PlayerState)
{
    Super::AddPlayerState(PlayerState);

    // 내부 데이터 복제 대기 (즉시 실행하면 null)
    FTimerDelegate UnmuteDelegate;
    UnmuteDelegate.BindUFunction(this,
        FName("UnmuteVoiceOnAddPlayerState"), PlayerState);
    GetWorldTimerManager().SetTimer(
        UnmuteVoiceTimerHandle,
        UnmuteDelegate,
        0.1f,    // 0.1초 딜레이
        false   // 반복 없음
    );
}
QA 포인트: 이 딜레이 값(0.1초)은 경험적 수치입니다. 고지연 네트워크에서는 부족할 수 있으며, 현재 TODO 주석에 "Event(Delegate) 방식으로 전환 필요"라고 표시되어 있습니다.
사망/부활 시 음성 제어
UpdateVoiceMember()를 통해 플레이어의 생존 상태에 따라 Mute/Unmute를 결정합니다:
  • 내가 살아있고 + 상대가 살아있음 → Unmute (양방향 소통)
  • 내가 살아있고 + 상대가 죽음 → Mute (죽은 플레이어 소리 차단)
  • 내가 죽어있음 → 모든 상대 Unmute (죽은 상태에서는 전부 청취 가능)
