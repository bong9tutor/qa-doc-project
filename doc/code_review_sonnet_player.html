목차

AAO_PlayerCharacter
부모: ACharacter · IAbilitySystemInterface · IAO_FoleyAudioBankInterface
역할
플레이어가 조종하는 캐릭터 — 입력 처리, 이동, GAS 어빌리티 실행
주요 인터페이스
IAbilitySystemInterface (GAS 연동), IAO_FoleyAudioBankInterface (발소리 등 Foley 사운드)
실행 위치
서버 + 모든 클라이언트. 입력은 로컬 클라이언트에서만, 이동·상태는 복제
GAS 초기화 시점
PossessedBy(서버) + OnRep_PlayerState(클라이언트)에서 각각 수행
ACharacter를 상속하며, GAS(Gameplay Ability System)를 통해 근접 공격·달리기·점프·트래버설·상호작용·검사·사망 등 모든 캐릭터 능력을 관리합니다. 입력은 Enhanced Input(IMC_Player)으로 받아 이동/시야 회전은 직접 처리하고, 어빌리티 입력(공격·스프린트 등)은 InputAbilities TMap을 통해 GAS에 전달합니다. 멀티플레이어에서 이동 속도(Gait)와 착지 정보(LandVelocity, bJustLanded)는 Replicated 변수로 동기화하여 다른 클라이언트 화면에서도 올바른 애니메이션이 재생되도록 합니다.

변수명타입지정자설명
Gait EGait ReplicatedUsing=OnRep_GaitEditAnywhere 현재 보행 상태(Walk/Run/Sprint). 복제되어 다른 클라이언트의 애니메이션 상태머신에 전달됩니다.
LandVelocity FVector ReplicatedBPReadOnly 착지 직전의 속도 벡터. 착지 애니메이션 블렌딩 세기 계산에 사용됩니다.
bJustLanded bool ReplicatedBPReadOnly 착지 직후 짧은 시간 true로 유지되는 플래그. 애니메이션 인스턴스가 착지 모션을 재생하는 트리거로 사용합니다.
CharacterInputState FCharacterInputState EditAnywhere 로컬 입력 상태(bWantsToSprint, bWantsToWalk). 클라이언트에서 변경 후 ServerRPC_SetInputState로 서버에 전달합니다.
DefaultAbilities TArray<TSubclassOf<UGameplayAbility>> EditDefaultsOnlyBPReadOnly 캐릭터 초기화 시 자동으로 부여(Grant)할 어빌리티 목록. 입력 없이 자동 실행되는 패시브·상시 어빌리티가 여기에 들어갑니다.
InputAbilities TMap<int32, TSubclassOf<UGameplayAbility>> EditDefaultsOnlyBPReadOnly 입력 ID → 어빌리티 클래스 매핑. 키를 누르면 HandleGameplayAbilityInputPressed(ID)가 해당 어빌리티를 실행합니다.
DefaultEffects TArray<TSubclassOf<UGameplayEffect>> EditDefaultsOnlyBPReadOnly 초기화 시 자동 적용할 게임플레이 이펙트 목록. 초기 체력·스태미나 수치 설정에 사용합니다.
IMC_Player / IA_Move 등 UInputMappingContext / UInputAction EditDefaultsOnlyBPReadOnly Enhanced Input 에셋 레퍼런스. IMC_Player가 매핑 컨텍스트(키 조합), 각 IA_*가 개별 입력 액션을 정의합니다.
PossessedBy(AController* NewController) Override ▶
하는 일
서버에서만 실행됩니다. 컨트롤러가 이 캐릭터를 점유(Possess)할 때 엔진이 호출합니다. GAS 초기화의 서버 측 진입점입니다.
흐름
① 부모 PossessedBy 호출 → ② AbilitySystemComponent에 PlayerState 주인 등록 → ③ InitializeAttributes() → ④ BindGameplayAbilities() → ⑤ BindGameplayEffects()
왜 서버에서만?
GAS의 어빌리티 부여(Grant)는 Authority(서버)에서만 수행해야 합니다. 클라이언트는 복제된 어빌리티 목록을 받아 사용합니다.
OnRep_PlayerState() Override ▶
하는 일
클라이언트에서 PlayerState 복제가 처음 수신될 때 엔진이 호출합니다. 클라이언트 측 GAS 초기화 진입점입니다.
흐름
① PlayerState 유효성 확인 → ② AbilitySystemComponent 소유자 등록 → ③ BindAttributeDelegates() → ④ UI 업데이트
PossessedBy와의 차이
PossessedBy는 서버 전용, OnRep_PlayerState는 클라이언트 전용. 둘이 짝을 이뤄 서버·클라이언트 양쪽에서 GAS를 초기화합니다.
SetupPlayerInputComponent() Override ▶
하는 일
Enhanced Input 컨텍스트(IMC_Player)를 PlayerController에 등록하고, 각 InputAction을 콜백 함수에 바인딩합니다.
바인딩 구조
IA_Move → Move() (매 프레임 이동벡터 적용)
IA_Look → Look() (마우스/스틱 시야 회전)
IA_Jump → StartJump() / TriggerJump()
IA_Sprint → HandleGameplayAbilityInputPressed/Released(ID)
(공격 등 GAS 어빌리티도 동일 패턴)
ServerRPC_SetInputState(bWantsToSprint, bWantsToWalk) Server, Reliable ▶
하는 일
클라이언트가 스프린트/걷기 입력을 서버에 전달합니다. 서버에서 CharacterInputState를 업데이트하고 SetCurrentGait()를 호출해 Gait를 변경합니다.
Server, Reliable 의미
클라이언트에서 호출하면 서버에서 실행됩니다. Reliable = 패킷 손실 시 재전송 보장. 이동 상태는 반드시 서버에서 권한 있게 처리해야 합니다.
흐름
① 클라이언트 입력 감지 → ② ServerRPC 호출 → ③ 서버에서 Gait 변경 → ④ Gait(ReplicatedUsing) 자동 복제 → ⑤ 모든 클라이언트 OnRep_Gait 실행 → ⑥ 애니메이션 갱신
StartSprint_GAS(bool bShouldSprint) ▶
하는 일
스프린트 시작/종료를 GAS에 알립니다. bWantsToSprint를 갱신하고 ServerRPC_SetInputState를 통해 서버에 전달합니다.
호출 시점
IA_Sprint 입력 액션 Pressed/Released 때 바인딩된 콜백에서 호출됩니다.
HandlePlayerDeath() ▶
하는 일
AttributeSet의 OnPlayerDeath 델리게이트에 바인딩된 함수로, Health가 0 이하가 되었을 때 호출됩니다.
흐름
① PlayerState.bIsAlive = false → ② DeathSpectateComponent 관전 모드 전환 → ③ 사망 어빌리티(GAS) 실행 → ④ GameState에 AddTeamDeathCount 통보
InitializeAttributes() / BindGameplayAbilities() / BindGameplayEffects() ▶
InitializeAttributes
AttributeSet을 ASC(AbilitySystemComponent)에 등록하고, AttributeDefaults 에셋에서 초기값을 읽어 적용합니다.
BindGameplayAbilities
DefaultAbilities와 InputAbilities 배열을 순회하며 각 어빌리티를 ASC에 Grant(부여)합니다. 이 작업은 서버(Authority)에서만 수행합니다.
BindGameplayEffects
DefaultEffects 배열의 게임플레이 이펙트를 자신에게 적용합니다. 주로 초기 체력/스태미나 수치 세팅에 사용합니다.
Landed() / OnJumped_Implementation() Override ▶
하는 일
Landed는 착지 시 LandVelocity를 기록하고 bJustLanded를 true로 설정한 뒤, 타이머로 짧은 시간 후 false로 초기화합니다. OnJumped는 점프 시 Foley 사운드를 재생합니다.
LandVelocity / bJustLanded가 Replicated인 이유
다른 플레이어의 화면에서도 올바른 착지 애니메이션이 재생되어야 하기 때문입니다.
방향대상 클래스관계 설명

보유UAO_InteractionComponent주변 인터랙터블 감지 및 GAS AbilityTask 실행
읽음AAO_PlayerState사망 시 bIsAlive 설정, 인벤토리 저장 요청

UAO_PlayerCharacter_AttributeSet
부모: UAttributeSet
역할
플레이어의 수치 스탯(체력·스태미나·속도)을 GAS 어트리뷰트로 관리
생존 범위
PlayerCharacter와 함께 생성·파괴
실행 위치
서버에서 Authority 연산, ReplicatedUsing으로 클라이언트에 전파
주요 책임
수치 클램핑(PreAttributeChange), 사망 감지, 스태미나 잠금
GAS의 UAttributeSet을 상속하며, FGameplayAttributeData 타입으로 각 스탯을 선언합니다. ATTRIBUTE_ACCESSORS_BASIC 매크로가 GetHealth(), SetHealth(), GetHealthAttribute() 같은 접근자 함수를 자동 생성합니다. 수치 변경은 항상 GameplayEffect를 통해 이루어지며, 직접 SetHealth()를 호출하는 것이 아니라 GAS 파이프라인을 통해 PreAttributeChange → PostGameplayEffectExecute 순서로 처리됩니다. 이 구조 덕분에 수치 검증, 클램핑, 사망 처리를 한 곳에서 관리할 수 있습니다.
변수명타입지정자설명
Health / MaxHealth FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_Health 현재 체력과 최대 체력. 서버에서 GameplayEffect로 변경되면 OnRep_Health가 클라이언트에서 실행되어 UI 델리게이트를 발동합니다.
Stamina / MaxStamina FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_Stamina 현재 스태미나와 최대 스태미나. 스프린트·점프 등의 어빌리티가 소비하고, 스태미나 회복 GameplayEffect가 채워줍니다.
WalkSpeed / RunSpeed / SprintSpeed FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_*Speed 3단계 이동속도. OnSpeedChanged 콜백에서 CharacterMovementComponent의 MaxWalkSpeed를 갱신합니다.
StaminaLockoutPercent float EditDefaultsOnlyBPReadOnly 스태미나 잠금 임계값(기본 0.25 = 25%). Stamina가 MaxStamina의 25% 이하로 떨어지면 스프린트를 강제 중단합니다.
OnPlayerDeath FOnPlayerDeath (Multicast Delegate) (멤버 델리게이트) Health가 0 이하가 되었을 때 PostGameplayEffectExecute에서 브로드캐스트합니다. PlayerCharacter의 HandlePlayerDeath()가 이 델리게이트에 바인딩됩니다.
PreAttributeChange(Attribute, NewValue) Override ▶
하는 일
어트리뷰트 값이 실제로 변경되기 직전에 호출됩니다. 주로 범위 클램핑을 수행합니다.
동작
Health의 경우: NewValue = FMath::Clamp(NewValue, 0.f, MaxHealth)
Stamina의 경우: NewValue = FMath::Clamp(NewValue, 0.f, MaxStamina)
Speed의 경우: NewValue = FMath::Max(0.f, NewValue)
주의
PreAttributeChange의 클램핑은 임시적입니다. 최종 값은 PostGameplayEffectExecute에서도 검증합니다.
PostGameplayEffectExecute(Data) Override ▶
하는 일
GameplayEffect가 어트리뷰트를 변경한 직후 서버에서 호출됩니다. 최종 값 검증과 사이드 이펙트(사망 처리, 스태미나 잠금) 실행 위치입니다.
흐름 (Health 변경 시)
① Health를 [0, MaxHealth] 범위로 클램핑 → ② Health == 0이면 OnPlayerDeath.Broadcast() → ③ PlayerCharacter.HandlePlayerDeath() 실행
흐름 (Stamina 변경 시)
① Stamina 클램핑 → ② Stamina < MaxStamina * StaminaLockoutPercent이면 HandleStaminaLockout() 호출
HandleStaminaLockout(Data) ▶
하는 일
스태미나가 임계값(StaminaLockoutPercent) 이하로 떨어졌을 때 스프린트를 강제 종료합니다.
흐름
ASC에 스프린트 어빌리티 취소 요청 → CharacterInputState.bWantsToSprint = false → 서버 복제
OnRep_Health / OnRep_Stamina 등 (7종) ReplicatedUsing ▶
하는 일
서버에서 어트리뷰트가 변경된 후 클라이언트가 복제 데이터를 수신했을 때 자동 호출됩니다. GAS의 표준 패턴으로, GAMEPLAYATTRIBUTE_REPNOTIFY 매크로를 호출하여 GAS 내부 예측(Prediction) 시스템과 동기화합니다.
파라미터 OldValue
이전 값을 받아 클라이언트가 변화량을 계산할 수 있습니다 (예: 피격 시 피해량 표시).
방향대상 클래스관계 설명

수정자UGameplayEffect (GAS)피해·치유·스태미나 변경 등 모든 수치 변경은 GameplayEffect를 통해 이루어짐
읽힘HUD / UI WidgetOnRep 콜백에서 브로드캐스트된 델리게이트를 구독해 체력바·스태미나바 UI 갱신
읽힘AAO_PlayerState맵 이동 전 SaveHealthBeforeTravel이 현재 Health 값을 읽어 보존
