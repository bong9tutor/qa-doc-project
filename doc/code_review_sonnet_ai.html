AAO_AICharacterBase
부모: ACharacter · IAbilitySystemInterface
역할
모든 적 AI 캐릭터의 공통 기반 클래스 (7종 파생)
파생 클래스
Bull, Crab, Insect, LavaMonster, Stalker, Troll, Werewolf
실행 위치
서버에서 AI 로직 실행. 몽타주는 NetMulticast로 모든 클라이언트에 전파
행동 구동
StateTree (종별 Conditions·Evaluators·Tasks 파일 분리)
ACharacter와 IAbilitySystemInterface를 상속하여 이동·충돌·GAS를 모두 지원합니다. 각 적 종류(Bull, Crab 등)는 이 클래스를 상속받아 고유한 StateTree와 어빌리티를 정의합니다. 공통 기능인 기절(Stun) 처리는 이 기반 클래스에서 구현하며, 기절 몽타주는 NetMulticast RPC로 서버→모든 클라이언트에 동기화합니다. 기억 시스템(MemoryComponent)과 공격 설정(GetCurrentAttackConfig)도 여기서 정의하여 StateTree Tasks가 일관된 인터페이스로 접근할 수 있도록 합니다.

StateTree 연동
수색 Task → GetLastKnownLocation, 회피 Task → GetLastLostLocation
UActorComponent를 상속하며, AAO_AICharacterBase에 부착되어 있습니다. 플레이어별 기억 데이터(FAO_PlayerMemoryData)를 TMap에 저장하고 MemoryDuration(기본 60초)이 지난 기억은 무효로 처리합니다. 청각 기억(LastHeardLocation)은 HeardMemoryDuration(기본 10초)의 더 짧은 유효 시간을 가집니다. Crab AI는 GetLastLostLocation으로 플레이어가 마지막에 있던 위치를 피하고, 선공형 AI(Stalker 등)는 GetLastKnownLocation으로 플레이어를 추적합니다.
FAO_PlayerMemoryData (USTRUCT, BlueprintType)
LastKnownLocation 시야 내에 있을 때 매 틱 업데이트되는 마지막 확인 위치. UpdatePlayerLocation()이 기록합니다.
LastSeenTime 마지막으로 플레이어를 시야에서 확인한 게임 시간(GetWorld()->GetTimeSeconds()). 기억 유효성 판단에 사용합니다.
LastLostLocation 플레이어가 시야에서 사라진 순간의 위치. SetLastKnownLocation()이 시야 소실 시점에 기록합니다.
LostTime 시야에서 사라진 시간. 회피 행동 시 얼마나 최근의 정보인지 판단합니다.
bIsInSight 현재 이 플레이어가 AI 시야 내에 있는지 여부. AIPerception 이벤트에서 true/false로 전환됩니다.
변수명타입지정자설명
PlayerMemories TMap<TWeakObjectPtr<AAO_PlayerCharacter>, FAO_PlayerMemoryData> (UPROPERTY) 플레이어 포인터 → 기억 데이터 매핑. TWeakObjectPtr을 키로 사용하여 플레이어가 파괴되어도 Dangling Pointer가 발생하지 않습니다.
MemoryDuration float EditDefaultsOnly 시각 기억 유효 시간 (기본 60초). IsMemoryValid()에서 LastSeenTime과 비교합니다.
LastHeardLocation FVector (UPROPERTY) 소리로 감지된 마지막 위치. 발소리·폭발 등의 청각 자극 위치를 기억합니다.
LastHeardTime float (UPROPERTY) 마지막 청각 감지 시간. ClearHeardLocation에서 HeardMemoryDuration 만료 확인에 사용합니다.
HeardMemoryDuration float EditDefaultsOnly 청각 기억 유효 시간 (기본 10초). 시각 기억(60초)보다 짧습니다.

AAO_Spawner_AIManager
부모: AActor
역할
레벨 내 AI 스폰 타이밍·위치·수량을 자동 관리
배치 방식
레벨에 액터로 배치, 에디터에서 설정 노출
실행 위치
서버에서만 AI 스폰 (클라이언트에는 복제된 캐릭터만 표시)
위치 결정
EQS (Environment Query System)로 플레이어 시야 밖 위치 선택
AActor를 상속하며 레벨에 직접 배치합니다. BeginPlay에서 SpawnInterval(±SpawnIntervalRandomDeviation) 간격의 타이머를 설정하여 주기적으로 TrySpawnAI()를 호출합니다. 스폰 위치는 EQS(SpawnLocationQuery)를 비동기 실행하여 플레이어 시야에서 벗어난 NavMesh 위치를 선택합니다. RestrictedVolumes에 등록된 금지 영역에서는 스폰하지 않으며, IntensiveVolumes에 등록된 집중 스폰 영역에서는 별도 타이머로 더 빠른 스폰을 수행합니다.
변수명타입지정자설명
MonsterClasses TArray<TSubclassOf<AAO_AICharacterBase>> EditAnywhereBPReadWrite 스폰할 몬스터 클래스 목록 (최대 7종). 배열 중 하나를 랜덤 선택하여 스폰합니다.
MaxMonstersInLevel int32 EditAnywhereBPReadWrite 레벨 내 동시 존재 가능한 최대 몬스터 수 (기본 10). GetCurrentSpawnedAICount()가 이 수치를 초과하면 스폰을 건너뜁니다.
SpawnInterval float EditAnywhereBPReadWrite 기본 스폰 시도 간격 (기본 5초). 실제 간격은 ±SpawnIntervalRandomDeviation의 랜덤 편차가 적용됩니다.
SpawnIntervalRandomDeviation float EditAnywhere 스폰 간격 랜덤 편차 (기본 ±2초). 스폰이 규칙적으로 느껴지지 않도록 불규칙성을 추가합니다.
SpawnLocationQuery UEnvQuery* EditAnywhereBPReadWrite 스폰 위치 결정에 사용할 EQS 에셋. 일반적으로 플레이어 시야각 밖, NavMesh 위, 최소 거리 이상의 조건을 설정합니다.
RestrictedVolumes TArray<AAO_Area_SpawnRestriction*> EditAnywhereBPReadWrite 스폰 금지 볼륨 목록. EQS 결과 위치가 이 볼륨 안에 있으면 스폰을 취소합니다.
IntensiveVolumes TArray<AAO_Area_SpawnIntensive*> EditAnywhereBPReadWrite 집중 스폰 볼륨 목록. 각 볼륨마다 IntensiveSpawnTimers에 별도 타이머가 할당되어 더 짧은 간격으로 스폰합니다.
SpawnedMonsters TArray<AAO_AICharacterBase*> (protected) 현재 살아있는 스폰 몬스터 목록. OnAIDestroyed 델리게이트로 사망 시 자동 제거됩니다.
CachedPlayers TArray<AAO_PlayerCharacter*> (protected) 현재 레벨의 플레이어 목록 캐시. EQS 쿼리 컨텍스트(플레이어 시야 계산)에 전달됩니다.
BeginPlay() — 타이머 등록 Override ▶
하는 일

서버 전용 이유
AI 스폰은 Authority(서버)에서만 수행해야 합니다. 클라이언트에서 이 함수를 실행하면 서버와 일치하지 않는 AI가 생성됩니다.
