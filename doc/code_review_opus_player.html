목차
AO_PlayerCharacter AO_PlayerCharacter_AttributeSet
AAO_PlayerCharacter
부모: ACharacter · IAbilitySystemInterface · IAO_FoleyAudioBankInterface
역할
플레이어가 조종하는 캐릭터 — Enhanced Input 처리, 3인칭 카메라, GAS 어빌리티 실행, 커스터마이징 외형
주요 인터페이스
IAbilitySystemInterface (GAS 연동 — GetAbilitySystemComponent 구현), IAO_FoleyAudioBankInterface (Foley 사운드 뱅크 제공)
실행 위치
서버 + 모든 클라이언트. bReplicates=true, ASC ReplicationMode=Mixed. 입력은 로컬 클라이언트에서만, Gait/LandVelocity/bJustLanded는 복제
GAS 초기화 시점
PossessedBy(서버) → InitAbilityActorInfo + InitializeAttributes + BindGameplayAbilities + BindGameplayEffects + BindAttributeDelegates. BeginPlay(클라이언트) → InitAbilityActorInfo + BindSpeedAttributeDelegates
캐릭터 이동 설정
bOrientRotationToMovement=true, RotationRate=(0, 540, 0). 컨트롤러 Yaw/Pitch/Roll 회전 사용 안 함 — 카메라만 컨트롤러 회전 추종
충돌 프로파일
캡슐 콜리전 프로파일 "Player". 웅크리기: MaxWalkSpeedCrouched=200, CrouchedHalfHeight=69
ACharacter를 상속하며, GAS(Gameplay Ability System)를 통해 근접 공격, 달리기, 점프, 트래버설, 상호작용, 검사, 사망 등 모든 캐릭터 능력을 관리합니다. 생성자에서 PrimaryActorTick.bCanEverTick=false로 설정되어 Tick 함수를 사용하지 않으며, 모든 상태 변화는 입력 이벤트와 GAS 콜백으로 구동됩니다. 입력은 Enhanced Input(IMC_Player)으로 받아 이동/시야 회전은 직접 처리하고, 어빌리티 입력(스프린트=InputID 1, 아웃라인=InputID 2)은 InputAbilities TMap을 통해 GAS에 전달합니다. 카메라는 SpringArm(TargetArmLength=400, bEnableCameraLag=true, CameraLagSpeed=10)에 부착된 3인칭 카메라로, CalcCamera를 오버라이드하여 로컬 컨트롤 시 Camera 컴포넌트를 사용하고, 비로컬 시 DeathSpectateComponent의 RepCameraView를 사용합니다. 멀티플레이어에서 이동 속도(Gait)와 착지 정보(LandVelocity, bJustLanded)는 Replicated 변수로 동기화하여 다른 클라이언트 화면에서도 올바른 애니메이션이 재생되도록 합니다. Mutable CustomizableObject 기반 외형 커스터마이징은 Body/Head 두 슬롯으로 분리되어 BaseSkeletalMesh에 부착되며, CustomizableSkeletalComponent가 네트워크 복제됩니다.

AAO_PlayerCharacter() [생성자] ▶
하는 일
모든 컴포넌트를 CreateDefaultSubobject로 생성하고 초기값을 설정합니다. Tick은 비활성화(bCanEverTick=false), bReplicates=true로 네트워크 복제를 활성화합니다.
이동 설정
bUseControllerRotationYaw/Pitch/Roll = false (캐릭터는 컨트롤러 회전을 따르지 않음). bOrientRotationToMovement=true (이동 방향으로 회전). RotationRate=(0, 540, 0). 캡슐 콜리전 프로파일 "Player".
카메라
SpringArm: bUsePawnControlRotation=true, TargetArmLength=400, bEnableCameraLag=true, CameraLagSpeed=10. Camera: bUsePawnControlRotation=false, SpringArm SocketName에 부착.
웅크리기
NavAgentProps.bCanCrouch=true, bCanWalkOffLedgesWhenCrouching=true, MaxWalkSpeedCrouched=200, CrouchedHalfHeight=69.
GAS
ASC: SetIsReplicated(true), ReplicationMode=Mixed. AttributeSet: CreateDefaultSubobject로 생성되어 ASC에 자동 등록.
커스터마이징
BaseSkeletalMesh → GetMesh()에 부착. Body/HeadSkeletalMesh → BaseSkeletalMesh에 부착. Body/HeadComponent: SetComponentName("Body"/"Head"), SetIsReplicated(true). CustomizingComponent: SetIsReplicated(true).
AI 지각
AIPerceptionStimuliSource: bAutoRegister=true. PostProcessComponent: RootComponent에 부착 (아웃라인 이펙트 용도).
PossessedBy(AController* NewController) Override ▶
하는 일
서버에서 컨트롤러가 이 캐릭터를 점유할 때 엔진이 호출합니다. GAS 초기화, 인벤토리 복원, UI 초기화의 서버 측 진입점입니다.
상세 흐름
1. Super::PossessedBy 호출
2. checkf로 ASC null 검증
3. AbilitySystemComponent->InitAbilityActorInfo(this, this) — Owner와 Avatar를 모두 자기 자신으로 설정
4. HasAuthority() 조건 → InitializeAttributes() → BindGameplayAbilities() → BindGameplayEffects() → BindAttributeDelegates()
5. SetTimerForNextTick으로 다음 틱에 UAO_Passive_WorldSubsystem->ReapplyAllPassives(PC) 호출 (부활 시 패시브 재적용)
6. InventoryComponent에서 PlayerState.PersistentInventory가 비어있지 않으면 ApplySlotsFromSave로 복원 후 Empty, NotifyListeners, RegisterToSubsystem 호출
7. NameplateComponent->HandlePlayerStateChanged
8. 로컬 컨트롤러일 때 InteractionComponent->InitializeInteractionUI(PC)
왜 서버에서만 GAS 초기화?
GAS의 어빌리티 부여(Grant)는 Authority에서만 수행해야 합니다. 클라이언트는 복제된 어빌리티 목록을 받아 사용합니다. 하지만 InitAbilityActorInfo는 서버/클라이언트 모두에서 호출합니다(클라이언트는 BeginPlay에서).
BeginPlay() Override ▶
하는 일
서버와 클라이언트 모두에서 실행됩니다. AI 지각 등록, 클라이언트 측 ASC 초기화, Enhanced Input 매핑 컨텍스트 등록, VOIP 타이머 시작을 수행합니다.
상세 흐름
1. AIPerceptionStimuliSource → RegisterForSense(Sight), RegisterForSense(Hearing), RegisterWithPerceptionSystem
2. !HasAuthority() 조건 → ASC->InitAbilityActorInfo(this, this) (클라이언트 측 Actor Info 초기화)
3. BindSpeedAttributeDelegates() (WalkSpeed/RunSpeed/SprintSpeed 변경 델리게이트 연결)
4. HasAuthority() && InteractableComponent → OnInteractionSuccess.AddDynamic(HandleInteractableComponentSuccess)
5. IsLocallyControlled() → AddMappingContext(IMC_Player, 0), bShowMouseCursor=false, SetInputMode(GameOnly)
6. 0.2초 간격 반복 타이머 → TryRegisterVoiceTalker() (PlayerState 유효할 때까지 재시도)
SetupPlayerInputComponent(UInputComponent*) Override ▶
하는 일
UEnhancedInputComponent로 캐스팅 후 각 InputAction을 콜백 함수에 바인딩합니다. 하위 컴포넌트(InteractionComponent, InspectionComponent, InventoryComp)에도 SetupInputBinding을 위임합니다.
바인딩 상세
IA_Move → Triggered → Move() (FVector2D 입력, 컨트롤러 Yaw 기준 이동)
IA_Look → Triggered → Look() (FVector2D 입력, AddControllerYaw/PitchInput)
IA_Jump → Started → StartJump() (Traversal 우선 시도, 실패 시 Jump 태그 활성화)
IA_Jump → Triggered → TriggerJump() (Traversal 지속 시도)
IA_Crouch → Started → HandleCrouch() (토글 방식: IsCrouched ? UnCrouch : Crouch)
IA_Walk → Started → HandleWalk() (bWantsToWalk 토글, 스프린트 중이면 무시)
IA_Sprint → Triggered → HandleGameplayAbilityInputPressed(1)
IA_Sprint → Completed → HandleGameplayAbilityInputReleased(1)
IA_Outline_Train → Started → HandleGameplayAbilityInputPressed(2)
컴포넌트 위임
InteractionComponent, InspectionComponent, InventoryComp 각각 SetupInputBinding(PlayerInputComponent)을 호출하여 자체 입력을 바인딩합니다.
OnRep_PlayerState() Override ▶
하는 일
클라이언트에서 PlayerState 복제가 수신될 때 호출됩니다. NameplateComponent->HandlePlayerStateChanged(GetPlayerState())를 호출하여 네임플레이트 UI를 갱신합니다.
참고
이 구현에서는 GAS 초기화가 포함되어 있지 않습니다. 클라이언트 측 ASC 초기화는 BeginPlay에서 수행됩니다.
OnRep_Controller() Override ▶
하는 일
클라이언트에서 Controller 복제가 수신될 때 호출됩니다. 로컬 컨트롤러일 때 InteractionComponent->InitializeInteractionUI(PC)를 호출하여 상호작용 UI를 초기화합니다.
PossessedBy와의 중복
PossessedBy에서도 동일한 UI 초기화를 수행합니다. OnRep_Controller는 클라이언트 측 안전장치로, Controller 복제 시점에 UI가 확실히 초기화되도록 보장합니다.
InitializeAttributes() ▶
하는 일
AttributeDefaults 데이터 에셋에서 초기값을 읽어 AttributeSet의 각 어트리뷰트를 Init* 함수로 설정합니다. 서버에서만 실행됩니다(!HasAuthority()이면 즉시 return).
초기화 순서
1. checkf로 AttributeDefaults, AttributeSet null 검증
2. InitMaxHealth(AttributeDefaults->MaxHealth)
3. InitStamina(AttributeDefaults->Stamina)
4. InitMaxStamina(AttributeDefaults->MaxStamina)
5. InitWalkSpeed / InitRunSpeed / InitSprintSpeed
6. PlayerState에서 bHasPersistentHealth가 true이면 PersistentHealth 값 사용 (0 이하면 50.f로 보정, 그 외 [0, MaxHealth] 클램프). false이면 AttributeDefaults->Health 사용
체력 영속 로직
레벨 전환 시 PlayerState에 저장된 체력을 복원합니다. 사망 상태(0 이하)에서 부활하면 50.f 고정값으로 복원됩니다.
BindGameplayAbilities() ▶
하는 일
DefaultAbilities와 InputAbilities를 순회하며 ASC에 GiveAbility로 부여합니다.
상세 구현
DefaultAbilities: FGameplayAbilitySpec(DefaultAbility)로 스펙 생성 → InputID 미지정 → GiveAbility. 태그 기반 활성화(TryActivateAbilitiesByTag) 어빌리티에 적합.
InputAbilities: FGameplayAbilitySpec(InputAbility.Value)로 생성 → AbilitySpec.InputID = InputAbility.Key 설정 → GiveAbility. FindAbilitySpecFromInputID(ID)로 검색/활성화 가능.
BindGameplayEffects() ▶
하는 일
DefaultEffects 배열의 게임플레이 이펙트를 자신에게 적용합니다.
상세 구현
각 DefaultEffect에 대해:
1. MakeEffectContext() → AddInstigator(this, this)
2. MakeOutgoingSpec(DefaultEffect, Level=1.f, Context)
3. Handle.IsValid() 검증 → ApplyGameplayEffectSpecToSelf(*Handle.Data.Get())
주로 스태미나 자동 회복(Infinite Duration), 초기 수치 설정 등에 사용됩니다.
BindAttributeDelegates() / BindSpeedAttributeDelegates() ▶
BindAttributeDelegates
PossessedBy(서버)에서 호출. AttributeSet을 UAO_PlayerCharacter_AttributeSet으로 캐스트 후 OnPlayerDeath 멀티캐스트 델리게이트에 HandlePlayerDeath를 바인딩합니다.
BindSpeedAttributeDelegates
BeginPlay(서버+클라이언트 모두)에서 호출. ASC의 GetGameplayAttributeValueChangeDelegate를 사용하여 WalkSpeed, RunSpeed, SprintSpeed 3개 어트리뷰트 변경 시 OnSpeedChanged 콜백을 연결합니다.
OnSpeedChanged 동작
SetCurrentGait()를 호출하여 현재 Gait에 맞는 속도를 MaxWalkSpeed에 반영하고, 클라이언트이면 ServerRPC_SetInputState로 서버에 입력 상태를 전달합니다.
ServerRPC_SetInputState(bWantsToSprint, bWantsToWalk) Server, Reliable ▶
하는 일
클라이언트가 스프린트/걷기 입력 상태를 서버에 전달합니다. 서버에서 CharacterInputState의 두 필드를 업데이트하고 SetCurrentGait()를 호출합니다.
Server, Reliable 의미
UFUNCTION(Server, Reliable) — 클라이언트에서 호출하면 서버에서만 _Implementation이 실행됩니다. Reliable = 패킷 손실 시 재전송 보장.
전체 복제 흐름
1. 클라이언트 입력 감지 (HandleWalk / StartSprint_GAS / OnSpeedChanged)
2. 로컬에서 CharacterInputState 변경 + SetCurrentGait() (로컬 즉시 반영)
3. !HasAuthority()이면 ServerRPC_SetInputState 호출
4. 서버에서 CharacterInputState 동기화 + SetCurrentGait() → Gait 변경

UAO_PlayerCharacter_AttributeSet
부모: UAttributeSet
역할
플레이어의 수치 스탯(체력/스태미나/3단계 이동속도)을 GAS 어트리뷰트로 관리. 수치 클램핑, 사망 감지, 스태미나 잠금을 한 곳에서 처리
생존 범위
PlayerCharacter 생성자에서 CreateDefaultSubobject로 생성되어 ASC에 자동 등록. PlayerCharacter와 함께 생성/파괴
실행 위치
서버에서 Authority 연산. 모든 7개 어트리뷰트가 DOREPLIFETIME_CONDITION_NOTIFY(COND_None, REPNOTIFY_Always)로 클라이언트에 항상 복제/통지
GAS 파이프라인
GameplayEffect 적용 → PreAttributeChange(임시 클램핑) → 값 반영 → PostGameplayEffectExecute(최종 검증 + 사이드 이펙트) → OnRep_*(클라이언트 동기화)
GAS의 UAttributeSet을 상속하며, FGameplayAttributeData 타입으로 각 스탯을 선언합니다. ATTRIBUTE_ACCESSORS_BASIC 매크로가 GetHealth(), SetHealth(), GetHealthAttribute(), InitHealth() 같은 접근자 함수를 자동 생성합니다. 수치 변경은 항상 GameplayEffect를 통해 이루어지며, 직접 SetHealth()를 호출하는 것이 아니라 GAS 파이프라인을 통해 PreAttributeChange → PostGameplayEffectExecute 순서로 처리됩니다. PreAttributeChange에서는 Health와 Stamina만 [0, Max] 범위로 클램핑합니다 (Speed 어트리뷰트는 클램핑하지 않음). PostGameplayEffectExecute에서는 Health 변경 시 사망 감지(0 이하 + DeathTag 미보유 + HasAuthority), Stamina 변경 시 HandleStaminaLockout을 수행합니다. 복제 설정은 COND_None + REPNOTIFY_Always를 사용하여, 동일한 값으로 재설정되더라도 항상 OnRep이 호출됩니다.
변수명타입지정자설명
Health FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_Health 현재 체력. PreAttributeChange에서 [0, MaxHealth]로 클램핑. PostGameplayEffectExecute에서 0 이하일 때 사망 처리(OnPlayerDeath.Broadcast). SetHealth로 최종 값을 다시 클램핑합니다.
MaxHealth FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_MaxHealth 최대 체력. Health 클램핑의 상한값으로 사용됩니다. InitializeAttributes에서 AttributeDefaults->MaxHealth로 초기화됩니다.
Stamina FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_Stamina 현재 스태미나. PreAttributeChange에서 [0, MaxStamina]로 클램핑. PostGameplayEffectExecute에서 클램핑 후 HandleStaminaLockout 호출. 스프린트/점프 어빌리티가 소비하고 Infinite GE가 회복합니다.
MaxStamina FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_MaxStamina 최대 스태미나. Stamina 클램핑의 상한값 및 StaminaLockout 임계값 계산(MaxStamina * StaminaLockoutPercent)에 사용됩니다.
WalkSpeed FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_WalkSpeed 걷기 이동속도. OnRep_Gait에서 Gait==Walk일 때 CharacterMovement->MaxWalkSpeed에 반영됩니다. OnSpeedChanged 콜백으로 변경 감지됩니다.
RunSpeed FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_RunSpeed 달리기 이동속도. Gait의 기본값(EGait::Run)에 대응하는 속도입니다. 게임 시작 시 이 값이 MaxWalkSpeed의 초기값이 됩니다.
SprintSpeed FGameplayAttributeData BPReadOnlyReplicatedUsing=OnRep_SprintSpeed 스프린트 이동속도. 스태미나 소비와 함께 사용됩니다. HandleStaminaLockout에서 스프린트 어빌리티가 취소되면 Gait가 Run으로 돌아갑니다.
StaminaLockoutPercent float EditDefaultsOnlyBPReadOnly 스태미나 잠금 해제 임계값. 기본값 0.25 (25%). 스태미나가 0에 도달하면 잠금 시작, MaxStamina * 0.25 이상으로 회복되면 잠금 해제. 히스테리시스 패턴으로 잦은 토글을 방지합니다.
OnPlayerDeath FOnPlayerDeath (DECLARE_MULTICAST_DELEGATE) (멤버 델리게이트) Health가 0 이하 + Status.Death 태그 미보유 + HasAuthority일 때 PostGameplayEffectExecute에서 Broadcast. PlayerCharacter의 HandlePlayerDeath()가 AddUObject로 바인딩됩니다.

스태미나 잠금(Lockout) 상태를 히스테리시스 패턴으로 관리합니다. 서버에서만 실행됩니다 (!OwningActor || !HasAuthority()이면 return).
상세 구현
1. Threshold = GetMaxStamina() * StaminaLockoutPercent (기본 25%)
2. LockoutTag = "Status.Lockout.Stamina"
3. 잠금 진입: NewStamina <= 0.f && LockoutTag 미보유 → ASC->AddLooseGameplayTag(LockoutTag) + "Ability.Movement.Sprint" 태그를 가진 어빌리티 CancelAbilities
4. 잠금 해제: LockoutTag 보유 && NewStamina >= Threshold → ASC->RemoveLooseGameplayTag(LockoutTag)
히스테리시스 설계
잠금은 0 도달 시에만 진입하고, 해제는 25%까지 회복되어야 합니다. 이렇게 진입/해제 기준을 다르게 하여 스태미나가 0 근처에서 반복적으로 스프린트 ON/OFF 되는 것을 방지합니다.
