UAO_InteractionComponent
부모: UActorComponent
역할
플레이어 캐릭터에 부착되어 상호작용 입력 처리, GAS 어빌리티 부여, 서버 RPC 호출을 총괄하는 컴포넌트
부모 클래스
UActorComponent
실행 위치
서버에서 어빌리티 부여 및 상호작용 실행, 클라이언트에서 입력 바인딩 및 UI 초기화
주요 책임
Enhanced Input 바인딩, Server RPC로 상호작용 트리거, 몽타주 멀티캐스트, 위젯 컨트롤러 생성
플레이어 캐릭터에 부착되는 상호작용 전용 컴포넌트입니다. BeginPlay에서 서버 권한이면 DefaultAbilities를 GAS에 부여하고, 로컬 플레이어이면 InteractionWidget과 WidgetController를 생성합니다. Enhanced Input의 Started/Completed 이벤트를 바인딩하여 상호작용 키 입력을 감지하고, ServerTriggerInteract RPC로 서버에 GameplayEvent를 전달합니다. MulticastPlayInteractionMontage를 통해 모션 워핑과 몽타주 재생을 모든 클라이언트에서 동기화합니다. bIsHoldingInteract는 Replicated로 선언되어 서버-클라이언트 간 홀딩 상태가 동기화됩니다.

UAO_InteractableComponent
부모: UActorComponent · IAO_Interface_Interactable
역할
상호작용 가능한 액터에 부착되어 상호작용 정보 제공, 메시 콜리전 설정, 이펙트 재생을 담당하는 컴포넌트
부모 클래스
UActorComponent, IAO_Interface_Interactable 인터페이스 구현
실행 위치
서버에서 NotifyInteractionSuccess 처리, 멀티캐스트로 이펙트 재생 동기화
주요 책임
InteractionInfo 반환, 콜리전 채널 설정, VFX/SFX 스폰, 소켓 기반 워핑 Transform 제공
상호작용 대상 액터에 부착하는 컴포넌트입니다. IAO_Interface_Interactable 인터페이스를 구현하여 GetInteractionInfo(), CanInteraction(), GetMeshComponents(), GetInteractionTransform()을 제공합니다. BeginPlay에서 Owner의 메시 컴포넌트를 수집하여 Interaction 콜리전 채널을 Block으로 설정합니다. 상호작용 성공 시 NotifyInteractionSuccess()가 호출되며, InteractionEffect 설정이 유효하면 MulticastPlayInteractionEffect를 통해 VFX와 SFX를 모든 클라이언트에서 재생합니다. VFX/SFX 모두 지연 스폰(Delay)을 지원하며, 타이머와 WeakLambda를 사용하여 안전하게 처리합니다.

UAO_InteractionWidget
부모: UUserWidget
역할
상호작용 UI를 표시하는 추상 위젯 클래스 (홀딩 프로그레스, 안내 텍스트 등)
부모 클래스
UUserWidget (Abstract, BlueprintType)
실행 위치
로컬 클라이언트에서만 생성 및 표시
주요 책임
WidgetController를 보관하고, 블루프린트에서 OnWidgetControllerSet 이벤트를 구현하여 UI를 갱신
Abstract로 선언된 UUserWidget 파생 클래스입니다. C++에서는 WidgetController 참조만 보관하고, 실제 UI 표현은 블루프린트 파생 클래스에서 OnWidgetControllerSet 이벤트를 구현하여 처리합니다. SetWidgetController()가 호출되면 컨트롤러를 저장한 뒤 OnWidgetControllerSet()을 호출하여 블루프린트에서 델리게이트 바인딩 등 초기화 작업을 수행할 수 있습니다.

MVC 패턴의 Controller 역할을 담당하는 UObject입니다. GAS 어빌리티 태스크에서 상호작용 상태가 변경될 때 BroadcastInteractionMessage()를 호출하면, OnInteractionMessageReceived 델리게이트를 통해 위젯에 메시지를 전달합니다. HideInteraction()은 빈 메시지를 브로드캐스트하여 위젯을 초기 상태로 되돌립니다.
변수명타입지정자설명
OnInteractionMessageReceived FOnInteractionMessageReceived BPAssignable 상호작용 메시지가 수신될 때 호출되는 다이내믹 멀티캐스트 델리게이트입니다. 위젯에서 바인딩하여 UI를 갱신합니다.
BroadcastInteractionMessage(FAO_InteractionMessage) BlueprintCallable ▶
하는 일
전달받은 FAO_InteractionMessage를 OnInteractionMessageReceived 델리게이트로 브로드캐스트합니다. GAS 어빌리티 태스크에서 상호작용 타겟 발견/해제 시 호출됩니다.
HideInteraction() BlueprintCallable ▶
하는 일
빈 FAO_InteractionMessage를 생성하여 브로드캐스트합니다. 상호작용 UI를 기본 상태로 숨깁니다.
방향대상 클래스관계 설명
생성자UAO_InteractionComponentInitializeInteractionUI()에서 NewObject로 생성됩니다.
구독자UAO_InteractionWidget (블루프린트)OnInteractionMessageReceived 델리게이트를 위젯이 바인딩하여 UI를 갱신합니다.
호출자GAS 어빌리티 태스크AO_AbilityTask_WaitFor* 류 태스크에서 BroadcastInteractionMessage()를 호출합니다.
UAO_UIStackManager
부모: UGameInstanceSubsystem
역할
위젯 스택 기반 UI 관리자 — Push/Pop으로 입력 모드와 커서를 자동 전환하는 GameInstance 서브시스템
부모 클래스
UGameInstanceSubsystem (맵 전환에도 영속)
실행 위치
로컬 클라이언트에서만 동작 (UI는 서버에서 관리하지 않음)
주요 책임
위젯 스택 Push/Pop, 입력 모드 정책 적용, Pause 메뉴 토글, 입력 상태 베이스라인 캐시/복구

GameInstance 서브시스템으로 맵 전환에도 유지되는 UI 스택 관리자입니다. PushWidgetInstance()로 위젯을 스택에 쌓으면 해당 위젯의 FAOUIStackPolicy에 따라 입력 모드, 커서, 포커스가 자동 전환됩니다. PopTop/PopAll/PopByClass로 위젯을 제거하면 남은 스택의 Top 정책을 재적용하거나, 비어있으면 베이스라인 입력 상태로 복구합니다. TryTogglePauseMenu()는 ESC/P 키 하나로 Pause 열기/닫기/다른 UI 닫기를 통합 처리합니다. 사망 상태에서는 GameAndUI 모드로 전환하되 이동/시점 입력을 차단하여 관전 UI를 지원합니다.
변수명타입지정자설명
DelegateManagerCached UAO_DelegateManager* UPROPERTY 설정창 열기/닫기 델리게이트를 바인딩하기 위해 캐시한 DelegateManager입니다.
Stack TArray<FAOUIStackEntry> UPROPERTY 위젯 스택 배열입니다. Last()가 현재 최상단 위젯입니다.
StackBaseZOrder int32 (constexpr) 스택 위젯의 기본 Z 순서입니다. 값은 100이며, 스택 인덱스가 더해집니다.
BaselineInputByPC TMap<TWeakObjectPtr<APlayerController>, FAOUIInputSnapshot> UPROPERTY UI 스택 사용 이전의 원래 입력 상태를 PC별로 캐시합니다.
LastAppliedInputByPC TMap<TWeakObjectPtr<APlayerController>, FAOUIInputSnapshot> UPROPERTY UI 스택이 마지막으로 적용한 입력 상태를 PC별로 저장합니다.
PauseMenuLockCount int32 UPROPERTY Pause 메뉴 열기 잠금 카운터입니다. 0 이하일 때만 Pause 메뉴를 열 수 있습니다.

UAO_UIActionKeySubsystem
부모: UGameInstanceSubsystem
역할
UI 열기/닫기 키를 InputMappingContext에서 읽어 캐시하고, 특정 키가 UI 키인지 판별하는 서브시스템
부모 클래스
UGameInstanceSubsystem
실행 위치
로컬 클라이언트에서만 동작 (UI 키바인드 추상화)
주요 책임
IMC에서 IA_UI_Open/Close 매핑 키 추출, IsUIOpenKey/CloseKey 판별 함수 제공
GameInstance 서브시스템으로 초기화 시 IMC_Player 에셋을 LoadObject로 로드하고, IA_UI_Open/IA_UI_Close InputAction에 매핑된 물리 키를 추출하여 CachedOpenKeys/CachedCloseKeys에 캐시합니다. 이후 IsUIOpenKey()/IsUICloseKey()를 통해 특정 FKey가 UI 열기/닫기 키인지 빠르게 판별할 수 있습니다. 이를 통해 UI 코드에서 하드코딩된 키 비교 없이 InputAction 기반으로 키바인드를 추상화합니다.
변수명타입지정자설명
CachedIMC UInputMappingContext* UPROPERTY LoadObject로 로드한 IMC_Player 에셋입니다. 매핑 정보를 순회하는 데 사용합니다.
CachedOpenAction UInputAction* UPROPERTY LoadObject로 로드한 IA_UI_Open InputAction 에셋입니다.
CachedCloseAction UInputAction* UPROPERTY LoadObject로 로드한 IA_UI_Close InputAction 에셋입니다.
CachedOpenKeys TArray<FKey> UPROPERTY IA_UI_Open에 매핑된 물리 키 배열입니다. IsUIOpenKey()에서 Contains로 검색합니다.
CachedCloseKeys TArray<FKey> UPROPERTY IA_UI_Close에 매핑된 물리 키 배열입니다. IsUICloseKey()에서 Contains로 검색합니다.

IsUICloseKey(const FKey&) ▶

전달된 FKey가 CachedCloseKeys에 포함되어 있는지 확인합니다. UI 닫기 키인지 판별하는 데 사용됩니다.

흐름

⑤ Mapping.Action이 Open이면 AddUniqueKey(CachedOpenKeys), Close이면 AddUniqueKey(CachedCloseKeys)
