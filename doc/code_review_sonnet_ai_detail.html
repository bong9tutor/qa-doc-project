AAO_BullController
부모: AAO_AggressiveAICtrl → AAO_AIControllerBase → AAIController
역할
Bull AI 전용 컨트롤러. 돌진(Charge) 및 근접 공격 판단 로직을 담당
상속 계층
AggressiveAICtrl 기반 — 추격/수색/대상 선택 로직 재사용
실행 위치
서버에서만 AI 로직 실행 (Authority)
StateTree 연동
CanChargeAttack, CanMeleeAttack으로 공격 조건 제공
AAO_AggressiveAICtrl을 상속하여 선공형 AI의 추격/수색/대상 선택 로직을 그대로 활용합니다. Bull 고유 기능으로 돌진 공격(MinChargeDistance~MaxChargeDistance 범위)과 근접 공격(MinChargeDistance 미만 거리) 판정을 제공합니다. OnPossess에서 Pawn을 AAO_Bull로 캐스팅하며, StateTree의 공격 Condition이 CanChargeAttack/CanMeleeAttack을 호출하여 공격 전이를 결정합니다.
변수명타입지정자설명
MinChargeDistance float EditDefaultsOnly 돌진 최소 거리 (기본값 500). 이 거리보다 가까우면 근접 공격으로 전환합니다.
MaxChargeDistance float EditDefaultsOnly 돌진 최대 거리 (기본값 2000). 이 범위를 초과하면 돌진 불가로 판정합니다.
GetBull() BlueprintCallable ▶
하는 일
현재 소유한 Pawn을 AAO_Bull*로 캐스팅하여 반환합니다. Cast<AAO_Bull>(GetPawn())을 수행합니다.
CanChargeAttack() BlueprintCallable ▶
하는 일
돌진 공격 가능 여부를 판정합니다. 기절/돌진 중/쿨다운 상태이면 false를 반환합니다.
판정 흐름
① Bull이 유효하고 기절/돌진/쿨다운 중이 아닌지 확인 → ② ChaseTarget이 존재하는지 확인 → ③ 대상과의 거리가 MinChargeDistance~MaxChargeDistance 범위인지 확인 → ④ HasPlayerInSight()로 시야 내인지 확인
거리 비교 방식
FVector::DistSquared로 제곱 거리를 비교하여 Sqrt 비용을 절약합니다.
CanMeleeAttack() BlueprintCallable ▶
하는 일
근접 공격 가능 여부를 판정합니다. 대상이 MinChargeDistance 미만으로 가까이 있을 때 true를 반환합니다.
돌진과의 관계
돌진은 일정 거리 이상이 필요하므로, 너무 가까운 경우 근접 공격으로 대체합니다.

AAO_CrabController
부모: AAO_AIControllerBase → AAIController
역할
비선공형 Crab AI 컨트롤러. 플레이어 감지 시 도망, 아이템 감지 및 기억
상속 계층
AIControllerBase 직접 상속 (AggressiveAICtrl 아님)
Perception 설정
시야 1200, 소실 1500, 시야각 120도, 청각 1500
고유 기능
도망 위치 계산 (12방향 샘플링 + NavMesh 프로젝션)
AAO_AIControllerBase를 직접 상속하며, AggressiveAICtrl과 달리 추격 로직 없이 도망 전용으로 동작합니다. 플레이어를 감지하면 ShouldFlee()가 true를 반환하여 StateTree가 도망 상태로 전이합니다. CalculateFleeLocation()은 위협 위치로부터 반대 방향으로 12방향을 샘플링하고 NavMesh 프로젝션 + 기억된 회피 위치와의 거리를 종합 채점하여 최적 도망 위치를 결정합니다. 아이템 감지 시 DiscoveredItems에 기억하며, 시야에서 사라져도 목록을 유지합니다.
변수명타입지정자설명
CachedCrab TWeakObjectPtr<AAO_Crab> (UPROPERTY) 소유한 Crab 캐릭터 약한 참조. OnPossess에서 캐스팅하여 캐시합니다.
FleeSearchRadius float EditDefaultsOnlyBPReadWrite 도망 위치 탐색 반경 (기본값 1500). 샘플링 지점까지의 최대 거리입니다.
PlayerProximityThreshold float EditDefaultsOnlyBPReadWrite 플레이어 근접 감지 거리 (기본값 500). 이 범위 안에 오면 시야 밖이어도 도망합니다.
ItemSearchRadius float EditDefaultsOnlyBPReadWrite 아이템 탐색 반경 (기본값 1500).
DiscoveredItems TArray<TWeakObjectPtr<AAO_MasterItem>> (UPROPERTY) 발견한 아이템 목록. 시야에서 사라져도 기억을 유지합니다. TWeakObjectPtr로 Dangling 방지합니다.

우선순위

하는 일
위협으로부터 반대 방향으로 도망할 최적 위치를 계산합니다.
알고리즘
① 위협 방향의 반대 벡터를 기준으로 12방향 샘플링 (30도 간격) → ② NavMesh 프로젝션으로 유효한 위치만 필터 → ③ 각 위치에 점수 부여: 위협 거리 + 기억된 회피 위치로부터의 거리 x 0.3 → ④ 최고 점수 위치 반환

AAO_StalkerController
부모: AAO_AggressiveAICtrl → AAO_AIControllerBase
역할
Stalker 전용 컨트롤러. 은신 접근, 시야 분석, Hit & Run 후퇴 관리
고유 특성
엄폐 위치 탐색 (32방향 샘플링 + LineTrace), EQS 비동기 요청
Hysteresis
LookingPlayer 안정적 유지 (최소 유지 시간 + 거리 비율 전환)
타겟 유지
시야 소실 후 TargetPersistenceSeconds(8초) 동안 스토킹 지속
가장 복잡한 AI 컨트롤러입니다. AggressiveAICtrl을 상속하되 OnPlayerLost를 완전히 재정의하여 시야 소실 시 즉시 Search로 넘어가지 않고 TargetPersistenceSeconds 동안 타겟을 유지합니다. FindHideLocation은 32방향 샘플링 + LineTrace로 플레이어 시야에서 숨을 수 있는 위치를 찾으며, 다른 Stalker와의 최소 200유닛 분리 거리를 보장합니다. FindHideLocationFromMultiple은 다중 플레이어 시야를 모두 고려합니다. EQS 비동기 결과는 RequestHideLocationEQS/ConsumePendingHideLocation으로 Controller에 저장 후 StateTree Task가 Tick에서 소비하는 패턴을 사용합니다. Hysteresis 로직은 LookingPlayer를 안정적으로 유지하여 빈번한 타겟 전환을 방지합니다.
변수명타입지정자설명
RetreatDuration float EditDefaultsOnly 공격 후 후퇴 지속 시간 (기본값 5초). 타이머 만료 시 Retreat 모드를 해제합니다.
TargetPersistenceSeconds float EditDefaultsOnly 시야 소실 후 타겟 유지 시간 (기본값 8초). 엄폐 접근 중 잠깐 시야를 잃는 것은 정상입니다.
LookingPlayerMinHoldTime float EditDefaultsOnly LookingPlayer 최소 유지 시간 (기본값 1.5초). 빈번한 타겟 변경을 방지합니다.
LookingPlayerSwitchDistanceRatio float EditDefaultsOnly 타겟 전환 거리 비율 (기본값 0.7). 새 타겟이 현재 타겟보다 30% 이상 가까워야 전환합니다.
StableLookingPlayer TWeakObjectPtr<AActor> (private) Hysteresis가 적용된 안정적인 LookingPlayer 참조입니다.
PendingHideLocation / bHasPendingHideLocation FVector / bool (private) EQS 비동기 결과 저장용. Task가 ConsumePendingHideLocation으로 소비합니다.
