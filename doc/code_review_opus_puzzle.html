AAO_PuzzleConditionChecker
부모: AActor
역할
퍼즐 시스템의 중앙 조건 검사기. FGameplayTagContainer 기반으로 이벤트를 수집하고, FPuzzleCondition 배열의 AND/OR 조합을 평가합니다.
조건 모드
RequiredTags(필수), BlockingTags(차단), OrderedTags(순서 검증), TimeLimit(시간 제한) 4가지 축을 FPuzzleCondition 구조체로 조합
네트워크
ActiveEvents/bIsCompleted 복제. CompletionTag를 TargetActors의 ASC에 AddLooseGameplayTag로 부여. MulticastPuzzleFailed로 실패 브로드캐스트
연결 구조
LinkedElements를 IAO_Interface_PuzzleElement로 캐스팅하여 ResetToInitialState/SetInteractionEnabled 호출. bDisableElementsOnComplete=true 시 완료 후 잠금
퍼즐 시스템의 허브 역할입니다. PuzzleElement, TorchPuzzleElement, PinballTrigger 등 다양한 소스에서 OnPuzzleEvent(FGameplayTag)를 수신하면 ActiveEvents에 태그를 추가하고, CheckAllConditions()로 조건을 평가합니다.

조건 평가 로직 (CheckSingleCondition):
1) BlockingTags — ActiveEvents에 하나라도 있으면 즉시 false
2) bRequiresOrder & OrderedTags — ConditionOrderHistory[index]의 최근 N개 태그가 순서와 정확히 일치하는지 확인
3) RequiredTags — ActiveEvents.HasAll()로 모든 필수 태그 존재 확인

실패 판정 (ShouldFailPuzzle): AND 모드에서는 순서 조건의 모든 태그를 받았는데 순서가 틀린 경우 즉시 실패. OR 모드에서는 모든 조건이 필요한 태그를 모두 수신했으나 전부 불충족일 때 실패합니다.

완료 시: CompletionTag를 TargetActors의 ASC에 부여 → PuzzleReactionActor의 OnTriggerTagChanged 콜백이 발동하여 물리적 반응(문 열림 등)을 트리거합니다.
심층 분석: StartConditionTimer는 매 OnPuzzleEvent마다 호출되어, 이미 실행 중인 타이머를 덮어씁니다(SetTimer가 기존 핸들을 자동 클리어). 따라서 시간 제한은 "마지막 이벤트 수신 후 N초"가 아닌 "첫 이벤트 이후 매번 리셋"되는 구조입니다. 이는 의도적 설계일 수 있지만, 퍼즐 디자인 시 주의가 필요합니다.

상태 전환 로직 (OnInteractionSuccess):

AAO_PuzzleReactionActor
부모: AActor · IAbilitySystemInterface
역할
퍼즐 완료/진행에 물리적으로 반응하는 액터. 문 열림, 플랫폼 이동 등 Transform 변환을 담당
트리거 방식
ASC의 RegisterGameplayTagEvent로 TriggerTag 변경을 NewOrRemoved 이벤트로 감시. Checker가 CompletionTag를 부여하면 자동 발동
반응 모드
OneTime(한 번만, 비활성화 불가), Toggle(태그 추가/제거에 따라 양방향), HoldActive(0~1 진행도에 비례하여 실시간 보간)
Transform 보간
16ms 타이머로 VInterpTo/RInterpTo 사용. InitialLocation→TargetRelativeLocation, InitialRotation→TargetRelativeRotation을 CurrentProgress(0~1)로 Lerp 후 보간
핵심 메커니즘: BeginPlay에서 ASC를 InitAbilityActorInfo(this, this)로 초기화하고, TriggerTag에 대해 RegisterGameplayTagEvent(NewOrRemoved)를 등록합니다. Checker가 TargetActors의 ASC에 CompletionTag를 부여하면 OnTriggerTagChanged 콜백이 호출됩니다.

AttachedActors 시스템: BeginPlay에서 각 AttachedActor의 MeshComponent 기준 상대 Transform을 AttachedActorOffsets에 저장합니다. UpdateTransform마다 UpdateAttachedActors()로 MeshComponent의 월드 Transform에 상대 오프셋을 적용하여 함께 이동시킵니다.

서버/클라이언트 분리: 서버에서 ActivateReaction/SetProgress로 상태 변경 → bIsActivated/TargetProgress 복제 → 클라이언트 OnRep에서 동일한 UpdateTransform 타이머 시작. 양쪽 모두 16ms 타이머로 부드러운 보간을 수행합니다.
심층 분석: HoldActive 모드에서 UpdateTransform은 목표 도달 시 타이머를 정지하지 않습니다(bReachedTarget 체크에서 HoldActive 제외). 이는 TargetProgress가 외부에서 계속 변경될 수 있기 때문에 타이머가 항상 활성 상태를 유지하는 설계입니다. SetProgress()에서도 이미 실행 중인 타이머는 시작하지 않습니다.
