AAO_Train
부모: AActor · IAbilitySystemInterface
역할
연료를 받아 움직이는 기차 액터 — GAS로 연료 수치 관리
연료 공급 방식
플레이어가 인벤토리의 연료 아이템을 선택한 채로 기차에 상호작용
실행 위치

연료 누수
FuelLeakSkillOn/Out으로 주기적 연료 감소 어빌리티 제어
AActor와 IAbilitySystemInterface를 함께 상속하여 비-캐릭터 액터에도 GAS를 적용한 사례입니다. 연료 수치는 UAO_Fuel_AttributeSet에 FGameplayAttributeData로 저장되며, 증가(AddEnergyAbilityClass)·감소(LeakEnergyAbilityClass) 두 가지 GameplayAbility로 제어합니다. InteractableComponent를 통해 플레이어의 상호작용을 받고, HandleInteractionSuccess에서 AddEnergy 어빌리티를 실행해 연료를 충전합니다. 연료가 변경될 때마다 OnFuelChangedDelegate를 브로드캐스트하여 UI와 Blueprint를 갱신합니다.

UCLASS(Abstract)로 선언되어 직접 배치는 불가하고 반드시 상속받아 사용합니다. 상호작용 흐름은 GAS AbilityTask(WaitForInteraction)가 주도하며, 플레이어가 범위 내에서 입력하면 CanInteraction() → 조건 통과 시 OnInteractionSuccess() 순서로 실행됩니다. 성공 시 MulticastNotifyInteractionSuccess RPC로 모든 클라이언트에 알리고, VFX/SFX 이펙트도 MulticastPlayInteractionEffect로 동기화합니다. AAO_PuzzleConditionChecker
부모: AActor (Blueprintable)
역할
GameplayTag 기반 퍼즐 조건을 검사하고 완료·실패를 판정
조건 유형
AND/OR 조합, 순서 조건, 시간 제한, 차단 태그
실행 위치
서버에서 조건 검사. ActiveEvents·bIsCompleted 복제로 클라이언트 동기화
완료 처리
CompletionTag를 TargetActors에 부여하고 OnPuzzleCompleted 브로드캐스트
레벨에 배치하는 범용 퍼즐 판정 액터입니다. PuzzleElement(버튼·레버 등)가 OnPuzzleEvent(tag)를 호출하면 ActiveEvents에 태그를 추가하고 CheckAllConditions()를 실행합니다. bRequireAllConditions이 true이면 모든 FPuzzleCondition을 AND로 검사하고, false이면 하나라도 만족하면 완료(OR)됩니다. 완료 시 TargetActors(문·엘리베이터 등)에 CompletionTag를 GameplayTag로 부여하여 해당 액터들이 자체 반응(문 열기, 플랫폼 이동)을 수행하게 합니다.
FPuzzleCondition (USTRUCT, BlueprintType)
RequiredTags 이 조건을 만족하려면 ActiveEvents에 반드시 포함되어야 할 GameplayTag 집합.
BlockingTags ActiveEvents에 이 태그 중 하나라도 있으면 조건을 실패시키는 차단 태그 집합. 잘못된 버튼 조합 감지에 사용합니다.
bRequiresOrder true이면 OrderedTags에 정의된 순서 그대로 이벤트가 발생해야 합니다. false이면 순서 무관하게 모두 활성화되면 됩니다.
OrderedTags bRequiresOrder=true일 때 필요한 이벤트 순서 배열. ConditionOrderHistory와 비교하여 정확한 순서인지 확인합니다.
TimeLimit 조건 만족까지의 시간 제한(초). 0이면 무제한. StartConditionTimer()가 이 값으로 타이머를 설정합니다.
변수명타입지정자설명
bRequireAllConditions bool EditAnywhereBPReadWrite true: 모든 Conditions가 만족되어야 완료(AND). false: 하나라도 만족하면 완료(OR).
Conditions TArray<FPuzzleCondition> EditAnywhereBPReadWrite 검사할 조건 목록. 에디터에서 여러 FPuzzleCondition을 추가하여 복합 퍼즐을 구성합니다.

OnPuzzleEvent(FGameplayTag EventTag, AActor* EventInstigator) BlueprintCallable ▶
하는 일
퍼즐 요소(버튼, 레버 등)가 활성화될 때 호출합니다. EventTag를 ActiveEvents에 추가하고 조건 검사를 실행합니다.
흐름
① bIsCompleted 이미 완료 상태이면 무시
② ActiveEvents에 EventTag 추가
③ ShouldFailPuzzle() 확인 → BlockingTag 감지 시 FailPuzzle()
④ CheckAllConditions() 실행
⑤ 조건 만족 시 CompletePuzzle() / 미만족 시 OnPuzzleProgress 브로드캐스트
⑥ 해당 조건에 TimeLimit이 있으면 StartConditionTimer() 시작
RemovePuzzleEvent(FGameplayTag EventTag) BlueprintCallable ▶
하는 일
토글 가능한 버튼·레버가 비활성화될 때 호출합니다. ActiveEvents에서 EventTag를 제거하고 조건을 재검사합니다.
CheckAllConditions() ▶
하는 일
모든 FPuzzleCondition을 bRequireAllConditions에 따라 AND 또는 OR로 검사합니다.
CheckSingleCondition 흐름
① ActiveEvents가 RequiredTags를 모두 포함하는지 확인
② BlockingTags 중 하나라도 활성화됐으면 false
③ bRequiresOrder이면 ConditionOrderHistory와 OrderedTags 순서 비교
CompletePuzzle() ▶
하는 일
퍼즐 완료를 최종 처리합니다.
흐름
① bIsCompleted = true (복제로 클라이언트에 전파)
② TargetActors에게 CompletionTag 부여(GAS AddLooseGameplayTag)
③ bDisableElementsOnComplete이면 DisableLinkedElements()
④ ClearAllTimers()
⑤ OnPuzzleCompleted 브로드캐스트
MulticastPuzzleFailed() NetMulticast, Reliable ▶
하는 일
서버에서 FailPuzzle()이 호출되면 모든 클라이언트에서 OnPuzzleFailed 델리게이트를 브로드캐스트합니다. 실패 사운드·이펙트 재생에 사용됩니다.
StartConditionTimer(ConditionIndex, Duration) ▶
하는 일
특정 조건(ConditionIndex)에 시간 제한 타이머를 설정합니다. Duration 초 안에 해당 조건이 만족되지 않으면 OnConditionTimeout이 실행되어 FailPuzzle()을 호출합니다.
타이머 관리
ConditionTimerHandles TMap에 조건 인덱스 → FTimerHandle로 저장하여 개별 취소가 가능합니다.
GetCompletionProgress() BlueprintPure ▶
하는 일
