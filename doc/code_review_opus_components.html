UAO_PlayerCharacter_AnimInstance
부모: UAnimInstance · IAO_TraversalTransform
역할
플레이어 캐릭터의 애니메이션 상태 갱신 및 Motion Matching용 궤적(Trajectory) 생성
부모 클래스
UAnimInstance, IAO_TraversalTransform (트래버설 위치 인터페이스). CreateAnimInstanceProxy/DestroyAnimInstanceProxy를 오버라이드하여 자체 FAOAnimInstanceProxy를 사용
실행 위치
모든 클라이언트. NativeThreadSafeUpdateAnimation으로 워커 스레드에서 실행 가능. NativeUpdateAnimation도 오버라이드되어 있으나 내부는 Super 호출만 수행
주요 책임
속도/가속도/상태(Gait, Stance, MovementMode) 갱신, PoseSearch 궤적 계산, 이동 분석 함수(IsMoving/IsStarting/IsPivoting/ShouldTurnInPlace 등) 블루프린트 노출
프록시 구조
FAOAnimInstanceProxy가 FAnimInstanceProxy를 상속하여 Owner(APawn*), Character(AAO_PlayerCharacter*), MovementComponent(UCharacterMovementComponent*) 3개 참조를 스레드 안전하게 보관. InitializeObjects에서 캐스팅 및 checkf 검증 수행
ABP 연동
모든 상태 변수(Speed2D, Gait, MovementMode 등)와 이동 분석 함수(IsMoving, IsPivoting 등)에 BlueprintThreadSafe 메타 지정. ABP의 스테이트 머신/전이 판단에서 직접 호출 가능. CurrentSelectedDatabase/CurrentDatabaseTags는 ABP에서 BlueprintReadWrite로 양방향 통신
UAnimInstance를 상속하며, CreateAnimInstanceProxy를 오버라이드하여 자체 FAOAnimInstanceProxy 인스턴스의 포인터를 반환합니다 (DestroyAnimInstanceProxy는 빈 구현 — Proxy가 멤버 변수이므로 별도 해제 불필요). FAOAnimInstanceProxy::InitializeObjects에서 TryGetPawnOwner → Cast<AAO_PlayerCharacter> → GetCharacterMovement 순서로 참조를 획득하며, 프리뷰 월드(IsPreviewWorld)에서는 조기 반환합니다. PreUpdate와 Update는 Super만 호출하는 패스스루입니다. NativeThreadSafeUpdateAnimation에서 Proxy의 3개 참조가 모두 유효할 때만 UpdateTrajectory → UpdateEssentialValues → UpdateStates를 순서대로 실행합니다. PoseSearch(Motion Matching)를 위한 궤적은 Speed2D에 따라 Idle/Moving 파라미터를 선택하고, PoseSearchGenerateTransformTrajectory → HandleTransformTrajectoryWorldCollisions → GetTransformTrajectoryVelocity(3구간: 과거 -0.3~-0.2s, 현재 0~0.2s, 미래 0.4~0.5s) 순서로 생성됩니다. IAO_TraversalTransform 인터페이스를 구현하여 트래버설 어빌리티가 목표 트랜스폼을 Set하면 ABP에서 Get하여 MotionWarping 대상 위치로 사용합니다.
Essential Values
변수명타입지정자설명
Proxy FAOAnimInstanceProxy Transient 스레드 안전 프록시. Owner(APawn*), Character(AAO_PlayerCharacter*), MovementComponent(UCharacterMovementComponent*) 3개 참조를 보관합니다. CreateAnimInstanceProxy()에서 &Proxy를 반환하여 엔진이 이 인스턴스를 사용하도록 합니다.
CharacterTransform / CharacterTransformLastFrame FTransform BPReadOnly 현재 및 이전 프레임의 캐릭터 월드 트랜스폼. UpdateEssentialValues()에서 Proxy.Character->GetActorTransform()으로 갱신됩니다.
Velocity / VelocityLastFrame / VelocityAcceleration FVector BPReadOnly 현재 속도(MovementComponent->Velocity에서 직접 읽음), 이전 프레임 속도, 속도의 변화율(가속도). VelocityAcceleration = (Velocity - VelocityLastFrame) / GetDeltaSeconds()로 계산됩니다.
Speed2D / bHasVelocity float / bool BPReadOnly 2D 수평 속도 크기(Velocity.Size2D())와 유효 속도 존재 여부(5.f 이상일 때 true). 궤적 파라미터 선택(Idle vs Moving)과 ShouldSpinTransition 판정에 사용됩니다.
Acceleration / AccelerationLastFrame / AccelerationAmount / bHasAcceleration FVector / FVector / float / bool BPReadOnly 현재 가속도(MovementComponent->GetCurrentAcceleration()), 이전 프레임 가속도, 정규화된 가속량(Length / GetMaxAcceleration(), 0~1 범위), 가속 존재 여부(AccelerationAmount > 0). IsMoving()에서 Acceleration.IsNearlyZero() 판정에 사용됩니다.
HeavyLandSpeedThreshold float BPReadOnly 강한 착지(Heavy Land) 판정 속도 임계값. 기본값 700.f. JustLanded_Heavy/Light에서 abs(LandVelocity.Z)와 비교하여 착지 세기를 판별합니다.
CurrentSelectedDatabase / CurrentDatabaseTags UPoseSearchDatabase* / TArray<FName> BPReadWrite 현재 선택된 PoseSearch 데이터베이스와 해당 데이터베이스의 태그 목록. ABP에서 BlueprintReadWrite로 양방향 접근 가능. IsStarting()/IsPivoting()에서 "Pivots" 태그 포함 여부를 확인합니다.
TraversalTransform FTransform BPReadWrite 트래버설 어빌리티에서 설정한 목표 트랜스폼. IAO_TraversalTransform 인터페이스의 Set/Get으로 접근. ABP에서 MotionWarping 대상 위치로 사용됩니다.
States
변수명타입지정자설명
MovementMode / MovementModeLastFrame ECustomMovementMode BPReadOnly 이동 모드. MOVE_None/Walking/Flying → OnGround, MOVE_Falling → InAir. UpdateStates()에서 MovementComponent->MovementMode을 읽어 변환합니다. 기본값 OnGround.
MovementState / MovementStateLastFrame EMovementState BPReadOnly 이동 상태(Idle/Moving). IsMoving() 결과에 따라 결정됩니다. 기본값 Idle, LastFrame 기본값 Moving (첫 프레임에 상태 변화를 감지하기 위함).
Gait / GaitLastFrame EGait BPReadOnly 보행 상태(Walk/Run/Sprint). Proxy.Character->Gait(Replicated 변수)에서 직접 읽습니다. 기본값 Run.
Stance / StanceLastFrame EStance BPReadOnly 자세(Stand/Crouch). MovementComponent->IsCrouching()으로 판정합니다. 기본값 Stand.
Trajectory (PoseSearch)
변수명타입지정자설명
Trajectory FTransformTrajectory BPReadOnly PoseSearch용 변환 궤적 결과. PoseSearchGenerateTransformTrajectory의 입출력으로 동시에 사용되며(In-Out 파라미터), HandleTransformTrajectoryWorldCollisions로 보정됩니다.
TrajectoryData_Idle / TrajectoryData_Moving FPoseSearchTrajectoryData BPReadOnly 정지/이동 시 궤적 생성 파라미터. NativeInitializeAnimation에서 초기화: Idle=(RotateTowardsMovementSpeed=0, MaxControllerYawRate=100), Moving=(RotateTowardsMovementSpeed=0, MaxControllerYawRate=0). Speed2D > 0이면 Moving 사용.
PreviousDesiredControllerYaw float BPReadOnly 이전 프레임의 목표 컨트롤러 Yaw. PoseSearchGenerateTransformTrajectory의 In-Out 파라미터로 프레임 간 연속성을 유지합니다. 기본값 0.
Trajectory_Collision FPoseSearchTrajectory_WorldCollisionResults BPReadOnly HandleTransformTrajectoryWorldCollisions의 충돌 결과 출력. TraceTypeQuery1 채널, 충돌 허용 150.f 범위로 계산됩니다.
TrjPastVelocity / TrjCurrentVelocity / TrjFutureVelocity FVector BPReadOnly 궤적에서 추출한 3구간 속도: Past(-0.3~-0.2s), Current(0~0.2s), Future(0.4~0.5s). GetTransformTrajectoryVelocity로 추출. IsMoving()은 TrjFutureVelocity.IsNearlyZero(10.f)로 판정, IsStarting()은 TrjFutureVelocity.Size2D() >= Velocity.Size2D()로 가속 감지.
NativeInitializeAnimation() Override ▶
하는 일
애니메이션 인스턴스 초기화 시 호출됩니다. 정지/이동 상태의 PoseSearch 궤적 파라미터를 설정합니다.
내부 흐름
1. Super::NativeInitializeAnimation() 호출
2. TrajectoryData_Idle 설정: RotateTowardsMovementSpeed=0.f, MaxControllerYawRate=100.f (정지 시 컨트롤러 Yaw 회전 반영)
3. TrajectoryData_Moving 설정: RotateTowardsMovementSpeed=0.f, MaxControllerYawRate=0.f (이동 시 컨트롤러 Yaw 미반영, 이동 방향만 사용)
ABP 연동
이 함수 이전에 FAOAnimInstanceProxy::InitializeObjects가 호출되어 Character/MovementComponent 참조가 이미 유효합니다. ABP의 Event Graph에서 추가 초기화를 수행할 수 있습니다.

NativeThreadSafeUpdateAnimation(float DeltaSeconds) Override ▶
하는 일
워커 스레드에서 매 프레임 호출됩니다. Proxy의 3개 참조가 모두 유효한 경우에만 갱신을 실행합니다.
내부 흐름
1. Super::NativeThreadSafeUpdateAnimation(DeltaSeconds) 호출
2. Proxy.Owner / Proxy.Character / Proxy.MovementComponent 중 하나라도 null이면 조기 반환 (로딩 중 발생)
3. UpdateTrajectory(DeltaSeconds) — PoseSearch 궤적 생성
4. UpdateEssentialValues() — 트랜스폼/가속도/속도 갱신
5. UpdateStates() — MovementMode/State/Gait/Stance 갱신
스레드 안전성
FAOAnimInstanceProxy를 통해 캐릭터 참조에 접근하므로 게임 스레드와의 동시 접근 문제를 방지합니다. 모든 호출 대상 함수도 BlueprintThreadSafe 메타가 지정되어 있습니다.
UpdateEssentialValues() BlueprintCallable, ThreadSafe ▶
하는 일
캐릭터 트랜스폼, 가속도, 속도 등 핵심 값을 MovementComponent로부터 읽어 갱신합니다.
내부 흐름
1. CharacterTransformLastFrame = CharacterTransform 저장
2. CharacterTransform = Proxy.Character->GetActorTransform()
3. AccelerationLastFrame = Acceleration 저장
4. Acceleration = Proxy.MovementComponent->GetCurrentAcceleration()
5. AccelerationAmount = Acceleration.Length() / GetMaxAcceleration() (0~1 정규화)
6. bHasAcceleration = (AccelerationAmount > 0.f)
7. VelocityLastFrame = Velocity 저장
8. Velocity = Proxy.MovementComponent->Velocity (직접 멤버 접근)
9. Speed2D = Velocity.Size2D()
10. bHasVelocity = (Speed2D > 5.f)
11. VelocityAcceleration = (Velocity - VelocityLastFrame) / GetDeltaSeconds()
UpdateStates() BlueprintCallable, ThreadSafe ▶
하는 일
4개 상태 열거형을 갱신합니다. 각각 LastFrame에 현재 값을 저장한 뒤 새 값을 결정합니다.
내부 흐름
1. MovementMode: MOVE_None/Walking/Flying → OnGround, MOVE_Falling → InAir
2. MovementState: IsMoving() ? Moving : Idle
3. Gait: Proxy.Character->Gait 값을 그대로 복사 (Character의 Gait은 Replicated 변수)
4. Stance: MovementComponent->IsCrouching() ? Crouch : Stand
네트워크 동기화
Gait은 캐릭터의 Replicated 변수이므로 다른 클라이언트에서도 올바른 보행 상태를 읽을 수 있습니다. MovementMode/Stance는 CharacterMovementComponent의 로컬 상태에서 직접 읽습니다.
UpdateTrajectory(float DeltaSeconds) BlueprintCallable, ThreadSafe ▶
하는 일
PoseSearch 궤적을 생성합니다. 3단계 파이프라인으로 예측 궤적 생성, 월드 충돌 처리, 속도 추출을 수행합니다.
내부 흐름
1. GetOwningComponent()->GetAnimInstance()로 AnimInstance 참조 획득
2. Speed2D > 0이면 TrajectoryData_Moving, 아니면 TrajectoryData_Idle 선택
3. PoseSearchGenerateTransformTrajectory(AnimInstance, TrajectoryData, DeltaSeconds, Trajectory, PreviousDesiredControllerYaw, Trajectory) — In/Out 동일 변수 사용
4. HandleTransformTrajectoryWorldCollisions(this, this, Trajectory, true, 0.01f, Trajectory, Trajectory_Collision, TraceTypeQuery1, false, 빈 배열, None, true, 150.f) — 중력/충돌 보정
5. GetTransformTrajectoryVelocity(-0.3, -0.2, TrjPastVelocity) — 과거 속도
6. GetTransformTrajectoryVelocity(0.0, 0.2, TrjCurrentVelocity) — 현재 속도
7. GetTransformTrajectoryVelocity(0.4, 0.5, TrjFutureVelocity) — 미래 속도
IsMoving() / IsStarting() / IsPivoting() BlueprintPure, ThreadSafe ▶
IsMoving
TrjFutureVelocity.IsNearlyZero(10.f)가 false이고 Acceleration.IsNearlyZero()가 false일 때 true. 미래 속도와 현재 가속도가 모두 유효해야 이동 중으로 판정합니다.
IsStarting
IsMoving()이 true이면서 TrjFutureVelocity.Size2D() >= Velocity.Size2D() (가속 중)이고 CurrentDatabaseTags에 "Pivots" FName이 없을 때 true. 방향 전환 중이 아닌 가속 시작 상태를 감지합니다.
IsPivoting
IsMoving()이 true이면 무조건 true를 반환합니다. 이름과 달리 실질적으로 IsMoving과 동일한 결과를 반환합니다 — ABP에서 별도 조건과 결합하여 사용될 수 있습니다.
ShouldTurnInPlace() / ShouldSpinTransition() BlueprintPure, ThreadSafe ▶
ShouldTurnInPlace
MovementState == Idle && MovementState == Moving을 동시에 확인합니다. 하나의 열거형이 Idle과 Moving을 동시에 만족할 수 없으므로 항상 false를 반환합니다. 원래 의도는 MovementStateLastFrame과 MovementState를 비교하여 Idle에서 Moving으로 전환되는 순간을 감지하는 것으로 보입니다.
ShouldSpinTransition
Speed2D >= 150.f이고 CurrentDatabaseTags에 "Pivots"가 없을 때 true. 일정 속도 이상에서 방향 전환이 아닌 경우 스핀 전환 애니메이션을 재생하는 조건입니다.
JustTraversal() / JustLanded_Heavy() / JustLanded_Light() BlueprintPure, ThreadSafe ▶
JustTraversal
!IsSlotActive("DefaultSlot") && GetCurveValue("MovingTraversal") > 0.0f && abs(GetTrajectoryTurnAngle()) < 50.f일 때 true. DefaultSlot이 비활성이고 MovingTraversal 애니메이션 커브가 활성이며 궤적 회전이 50도 미만일 때 트래버설 직후 이동으로 판정합니다.
JustLanded_Heavy
Proxy.Character->bJustLanded(Replicated)가 true이고 abs(LandVelocity.Z) >= abs(HeavyLandSpeedThreshold=700.f)일 때 true. Proxy null 검사를 먼저 수행합니다.
JustLanded_Light
bJustLanded가 true이고 abs(LandVelocity.Z) < abs(HeavyLandSpeedThreshold)일 때 true. Heavy와 반대 조건입니다.
GetTrajectoryTurnAngle() BlueprintPure, ThreadSafe ▶
내부 흐름
1. CurrentRotation = MakeRotFromX(Velocity) — 현재 속도 방향의 회전
2. FutureRotation = MakeRotFromX(TrjFutureVelocity) — 미래 속도 방향의 회전
3. NormalizedDeltaRotator(FutureRotation, CurrentRotation).Yaw 반환 — -180~180도 범위의 Yaw 차이
사용처
JustTraversal()에서 회전 각도 50도 미만 조건 판정에 사용됩니다.
SetTraversalTransform_Implementation / GetTraversalTransform_Implementation Interface Override ▶
하는 일
IAO_TraversalTransform 인터페이스 구현. Set은 this->TraversalTransform = InTraversalTransform으로 단순 대입, Get은 TraversalTransform을 반환합니다.
사용 흐름
트래버설 어빌리티(AO_GameplayAbility_Traversal)가 벽/장애물 히트 시 Set을 호출하여 목표 위치를 설정하면, ABP에서 Get을 호출하여 MotionWarping 타겟 위치로 사용합니다.
방향대상 클래스관계 설명
참조AAO_PlayerCharacterProxy.Character를 통해 Gait(Replicated), bJustLanded(Replicated), LandVelocity(Replicated) 읽음. 다른 클라이언트에서도 올바른 값을 수신

UAO_PlayerSoundSubsystem
부모: UGameInstanceSubsystem
역할
플레이어 캐릭터의 사운드(발소리, 피격음 등)를 캐릭터 메시 타입에 따라 조회하는 전역 서브시스템
부모 클래스
UGameInstanceSubsystem (GameInstance 수명 동안 유지 — 맵 전환 시에도 파괴되지 않음)
실행 위치
모든 클라이언트 및 서버. 사운드 재생 자체는 호출자(AnimNotify 등)가 로컬에서 수행
주요 책임
DeveloperSettings에서 SoundDataAsset 동기 로드, ECharacterMesh + GameplayTag 조합으로 사운드 조회, DefaultSounds 맵 기반 폴백 제공
복제
서브시스템 자체는 복제되지 않음. 각 클라이언트/서버가 독립적으로 동일한 DataAsset을 로드하여 동일한 결과를 반환
데이터 소스
UAO_PlayerSoundSettings(UDeveloperSettings 상속) → CharacterSoundDataAsset(TSoftObjectPtr) + DefaultSounds(TArray<FDefaultSoundEntry>) 구조. Settings는 프로젝트 설정 에디터에서 편집 가능

내부 흐름

2. Requests 전체 순회: Speed = bActive ? (1/BlendIn) : (1/BlendOut), Dir = bActive ? 1 : -1

UAO_TraversableComponent
부모: UActorComponent
역할
트래버설(벽 넘기, 장애물 점프) 가능한 액터에 부착되어 스플라인 기반 렛지(Ledge) 정보를 제공하는 컴포넌트
부모 클래스
UActorComponent (BlueprintSpawnableComponent). bCanEverTick=false — 이벤트 기반으로만 동작
실행 위치
서버 및 모든 클라이언트. OnRegister(컴포넌트 등록 시)에서 스플라인 스캔. 트래버설 판정(GetLedgeTransforms)은 어빌리티 실행 시 쿼리
주요 책임
소유 액터의 모든 USplineComponent 수집, 2개씩 쌍으로 묶어 반대편 렛지 양방향 매핑, 히트 위치 기반 전면/후면 렛지 트랜스폼 계산(Local/World Space 변환 포함)
렛지 쌍 규칙
ScanSplines에서 GetComponents<USplineComponent>로 수집한 배열을 인덱스 0-1, 2-3, ... 순서로 2개씩 쌍을 맺어 OppositeLedges에 양방향 등록. 홀수 개이면 마지막 스플라인은 반대편 없음
좌표 공간
FindLocationClosestToWorldLocation은 Local Space, GetDistanceAlongSplineAtLocation도 Local Space, GetTransformAtDistanceAlongSpline은 World Space. FindTransformClosestToWorldLocation은 World Space. 스플라인 거리 계산은 반드시 Local Space에서 수행해야 정확합니다
UActorComponent를 상속하며, 벽이나 장애물 액터에 부착됩니다. OnRegister에서 ScanSplines를 호출하여 소유 액터의 모든 USplineComponent를 수집하고, 인접한 2개씩 쌍을 맺어 양방향 OppositeLedges 맵에 등록합니다. 트래버설 어빌리티(AO_GameplayAbility_Traversal)가 GetLedgeTransforms를 호출하면: (1) FindLedgeClosestToActor로 가장 가까운 렛지 스플라인을 선택합니다 (UpVector*10을 더한 위치로 비교하여 렛지 윗면 기준 거리 계산). (2) 스플라인 길이가 MinLedgeWidth(60) 미만이면 실패 반환합니다. (3) HitLocation에서 스플라인 상의 가장 가까운 점을 Local Space에서 계산하고, 양쪽 여백(MinLedgeWidth/2)으로 클램핑하여 전면 렛지의 World Transform을 구합니다. (4) OppositeLedges 맵에서 반대편 렛지를 찾아 후면 렛지 Transform을 구합니다. 결과는 FTraversalCheckResult에 bHasFrontLedge/bHasBackLedge, Location, Normal(UpVector)로 반환됩니다.
변수명타입지정자설명
Ledges TArray<TObjectPtr<USplineComponent>> BPReadOnly ScanSplines에서 GetComponents(USplineComponent::StaticClass())로 수집한 소유 액터의 모든 스플라인 컴포넌트 배열. null 필터링 후 추가됩니다. 배열 순서가 쌍 매핑에 영향을 줍니다.
OppositeLedges TMap<TObjectPtr<USplineComponent>, TObjectPtr<USplineComponent>> BPReadOnly 렛지 쌍 양방향 매핑. ScanSplines에서 i와 i+1을 서로 반대편으로 등록(A→B, B→A). GetLedgeTransforms에서 *OppositeLedges.Find(ClosestLedge)로 반대편 렛지를 검색합니다. Find 실패 시 nullptr — bHasBackLedge=false.
MinLedgeWidth float BPReadOnly 렛지 최소 폭(기본 60.f). 두 가지 용도: (1) 스플라인 길이가 이 값 미만이면 트래버설 불가, (2) 가장 가까운 점을 클램핑할 때 양쪽 여백(MinLedgeWidth/2 = 30)으로 사용하여 렛지 끝부분에서의 불안정한 트래버설을 방지합니다.

NameplateWidgetClass TSubclassOf<UUserWidget> EditAnywhere 이름표 위젯 블루프린트 클래스. EnsureWidgetComponent에서 SetWidgetClass → InitWidget으로 적용됩니다. null이면 위젯이 생성되지 않습니다.
MinScaleDistance / MaxScaleDistance float EditAnywhere 거리 기반 스케일 범위(기본 200~1000). Alpha = Clamp((Distance - Min) / Max(1, Max - Min), 0, 1)로 계산됩니다.
MinScale / MaxScale float EditAnywhere 위젯 스케일 범위(기본 0.4~1.0). UIScale = Lerp(MaxScale, MinScale, Alpha) — 가까울수록 MaxScale, 멀수록 MinScale입니다.
HideDistance float EditAnywhere 이 거리(기본 1500) 이상이면 위젯을 SetVisibility(false, true)로 숨깁니다.
CapsuleHeight / BaseZOffset / ExtraZOffset float EditAnywhere 위젯 Z 위치 계산. 기본 높이 = CapsuleHeight(176) + BaseZOffset(5). 거리에 따라 Lerp(0, ExtraZOffset=10, Alpha)만큼 추가 상승. HandleCapsuleSizeChanged에서 CapsuleHeight가 동적으로 갱신됩니다.
bHideForSelf bool EditAnywhere true(기본)이면 로컬 플레이어(IsLocallyControlled)의 이름표를 숨깁니다.
복제 & 런타임
변수명타입지정자설명
DisplayName FText ReplicatedUsing=OnRep_DisplayName 서버에서 SetDisplayName_Server로 설정. EqualTo 비교로 동일 값 중복 설정을 방지합니다. DOREPLIFETIME(무조건 복제)으로 모든 클라이언트에 전달됩니다.
WidgetComponent TObjectPtr<UWidgetComponent> Transient EnsureWidgetComponent에서 런타임 생성. Screen Space 모드, DrawAtDesiredSize=true, Pivot(0.5, 0.5). 이미 존재하면 재생성하지 않습니다.
WidgetInstance TObjectPtr<UAO_NameTagWidget> Transient WidgetComponent->GetUserWidgetObject()를 Cast<UAO_NameTagWidget>한 인스턴스. SetPlayerName(), SetUIScale(), SetPlayerTalkingVisibility() 등을 호출합니다.
CachedCustomizingComponent TWeakObjectPtr<UAO_CustomizingComponent> (Private) BeginPlay에서 FindComponentByClass로 캐싱. OnCapsuleChangedDelegate 바인딩/해제에 사용. TWeakObjectPtr로 액터 파괴 시 자동 무효화됩니다.
CachedVOIPTalker TObjectPtr<UVOIPTalker> Transient SetVOIPTalker()로 설정. 위젯이 생성되기 전에 설정될 수 있으므로 별도 캐싱합니다. UpdateIsTalking에서 GetVoiceLevel()을 읽어 발화 상태를 판정합니다.
bIsTalking bool (Private) 현재 발화 중 여부. 히스테리시스 적용: 시작 조건 VoiceLevel > 0.02, 종료 조건 VoiceLevel <= 0.01. 깜빡임 방지 목적입니다. 초기값 false.

HandlePlayerStateChanged(APlayerState* NewPlayerState) ▶

EnsureWidgetComponent() ▶
내부 흐름
1. WidgetComponent 이미 존재하면 return (중복 생성 방지)
2. Owner = GetOwner(), checkf 검증
3. NewObject<UWidgetComponent>(Owner, "NameplateWidgetComponent")
4. RegisterComponent()
5. AttachToComponent(Root, KeepRelativeTransform)
6. SetWidgetSpace(Screen), SetDrawAtDesiredSize(true), SetPivot(0.5, 0.5)
7. NameplateWidgetClass 유효 시: SetWidgetClass → InitWidget
8. GetUserWidgetObject() → Cast<UAO_NameTagWidget> → WidgetInstance에 캐싱
ApplyDistanceVisuals() ▶
내부 흐름
1. !WidgetComponent → return
2. ShouldHideForSelf() → SetVisibility(false, true) 후 return
3. PlayerController[0]->PlayerCameraManager 획득. null이면 SetVisibility(true) 후 return
4. Distance = FVector::Distance(CameraLocation, OwnerLocation)
5. Distance > HideDistance(1500) → SetVisibility(false, true) 후 return
6. SetVisibility(true, true)
7. Alpha = Clamp((Distance - MinScaleDistance) / Max(1, MaxScaleDistance - MinScaleDistance), 0, 1)
8. UIScale = Lerp(MaxScale, MinScale, Alpha)
9. WidgetInstance->SetUIScale(UIScale)
10. ZOffset = CapsuleHeight + BaseZOffset + Lerp(0, ExtraZOffset, Alpha)
11. WidgetComponent->SetRelativeLocation(0, 0, ZOffset)
12. UpdateIsTalking()
UpdateIsTalking() ▶
내부 흐름
1. !CachedVOIPTalker → AO_LOG Warning 후 return
2. WidgetInstance 유효 시:
3. bIsTalking==true && VoiceLevel <= 0.01 → bIsTalking = false (종료 임계값)
4. bIsTalking==false && VoiceLevel > 0.02 → bIsTalking = true (시작 임계값)
5. WidgetInstance->SetPlayerTalkingVisibility(bIsTalking)
히스테리시스 설계
시작 임계값(0.02)과 종료 임계값(0.01)을 다르게 설정하여, VoiceLevel이 경계값 근처에서 진동할 때 스피커 아이콘이 빠르게 깜빡이는 것을 방지합니다.
HandleCapsuleSizeChanged(float NewHalfHeight) ▶
내부 흐름
1. !WidgetComponent → return
2. CapsuleHeight = NewHalfHeight — 멤버 변수 갱신
3. ZOffset = CapsuleHeight + BaseZOffset
4. WidgetComponent->SetRelativeLocation(0, 0, ZOffset)
트리거
UAO_CustomizingComponent->OnCapsuleChangedDelegate에서 호출. 캐릭터 외형 변경으로 캡슐 크기가 바뀔 때 위젯 높이를 맞춥니다.
ShouldHideForSelf() const ▶
내부 흐름
!bHideForSelf → return false. Cast<APawn>(GetOwner()) → Pawn && Pawn->IsLocallyControlled()이면 true. 자기 머리 위의 이름표를 숨기는 데 사용됩니다.
OnRep_DisplayName() / SetDisplayName_Server(const FText&) / ApplyDisplayNameToWidget() / TryInitNameFromOwner() ▶
OnRep_DisplayName
서버에서 DisplayName이 복제되면 클라이언트에서 자동 호출. ApplyDisplayNameToWidget() 위임.
SetDisplayName_Server
서버 전용. !HasAuthority() → return. DisplayName.EqualTo(InName) 검사로 동일 값 중복 설정 방지. 변경 시 DisplayName = InName 후 ApplyDisplayNameToWidget() 호출.
ApplyDisplayNameToWidget
EnsureWidgetComponent() 호출(위젯 미생성 방지) → WidgetInstance->SetPlayerName(DisplayName).
TryInitNameFromOwner
서버 전용. Cast<APawn>(Owner) → GetPlayerState() → 유효하면 HandlePlayerStateChanged(PS) 호출. BeginPlay 시 이름 초기화에 사용.

소유자AAO_PlayerCharacter캐릭터에 부착. PossessedBy에서 HandlePlayerStateChanged, OnRep_PlayerState에서도 HandlePlayerStateChanged 호출. VOIPTalker 생성 후 SetVOIPTalker 호출
읽음APlayerState / AAO_PlayerStateGetPlayerName()으로 이름 읽기. 비어있으면 AAO_PlayerState.OnPlayerNameReady 델리게이트 구독하여 비동기 수신

참조UAO_NameTagWidgetSetPlayerName(FText), SetUIScale(float), SetPlayerTalkingVisibility(bool) 호출하여 이름/스케일/스피커 아이콘 제어
