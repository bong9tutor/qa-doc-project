역할
로비 맵 전용 상호작용 오브젝트 — 레디, 게임 시작, 초대, 옷장
열거형
EAO_LobbyInteractType: ReadyToggle, StartGame, InviteFriends, Wardrobe
권한 체크
ReadyToggle은 게스트만, StartGame은 호스트만 — 익명 네임스페이스 IsLobbyHostController 헬퍼로 판별
네트워크
Server RPC를 PlayerController_Lobby를 통해 호출 — 로컬 선반영 후 서버 동기화 패턴
AAO_BaseInteractable을 상속하며 로비 맵에서의 네 가지 상호작용을 InteractType 열거형으로 분기합니다. .cpp에서 익명 네임스페이스에 IsLobbyHostController 유틸을 정의하여 PlayerState->IsLobbyHost()로 호스트/게스트를 판별합니다.

ReadyToggle 흐름 (심층): CanInteraction에서 !bIsHost 체크 → OnInteractionSuccess에서 PC->GetPlayerState<AAO_PlayerState>()로 PS 획득 → !PS->IsLobbyReady()로 토글값 계산 → IsLocalController()이면 PS->SetLobbyReady + OnRep_LobbyIsReady 로컬 선반영 → PC->Server_SetReady RPC. 주의: 로컬 선반영은 UI 즉시 반응을 위한 것으로, 서버 RPC 실패 시 상태 불일치 위험이 있습니다.

Wardrobe 흐름 (심층): Interactor를 AddDisabledPlayer()로 비활성화 목록에 추가하고, PC->CustomizingInteractable에 this를 저장한 뒤 Server_RequestWardrobe를 호출합니다. 이렇게 하면 커스터마이징 UI가 열린 동안 다른 플레이어가 같은 옷장에 접근하지 못합니다.

GetInteractionInfo 심층: Super 호출로 기본 Info를 받은 뒤, GetCurrentConditions로 bHasFuel/bHasClues를 확인합니다. 4가지 분기(모두 충족/둘 다 부족/연료만 부족/단서만 부족)에 따라 Title 문자열과 TitleTextColor(충족 시 회색, 미충족 시 빨강), HighlightStencilValue(충족 250, 미충족 252)를 동적으로 설정합니다. StencilValue의 차이는 커스텀 포스트프로세스 머티리얼에서 외곽선 색상을 다르게 렌더링하기 위한 것입니다.

Tutorial 레벨 이동 방식 차이: Tutorial 타입은 조건 충족 시 UGameplayStatics::OpenLevel로 로컬 레벨 전환합니다 (싱글플레이어 패턴). 반면 ExitToNextArea는 PC->Server_RequestStageExit() Server RPC를 통해 서버 트래블을 수행합니다 (멀티플레이어 패턴). 이 두 가지 전환 패턴의 차이는 중요한 네트워크 구조적 차이입니다.

GetCurrentConditions 연료 조회: TActorIterator<AAO_newTrain>로 월드에서 기차 액터를 찾아 ASC->GetNumericAttribute(UAO_Fuel_AttributeSet::GetFuelAttribute())로 연료 수치를 조회합니다. 첫 번째 기차를 찾으면 break하므로 월드에 기차가 여러 대 있을 경우 첫 번째만 검사됩니다.

BeginPlay에서 InitialInteractableLocation/Rotation을 저장하고, StartInteractionAnimation(bActivate)에서 목표를 계산합니다. 0.016초 간격 반복 타이머로 UpdateTransformAnimation을 호출하며, VInterpTo/RInterpTo로 보간합니다. 목표 도달 시(Dist < 0.1f && IsRotatorNearlyEqual 0.5도) 타이머를 클리어합니다. EndPlay에서도 타이머를 정리하여 메모리 누수를 방지합니다.

K2 이벤트 패턴: OnInteractActiveStarted/Ended/OnInteractionSuccess 각각에 대해 BlueprintImplementableEvent인 K2_ 변형을 호출하여 Blueprint에서 추가 로직을 구현할 수 있습니다.
