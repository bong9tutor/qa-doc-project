주요 책임
Enhanced Input 바인딩, Server RPC로 상호작용 트리거, 몽타주 멀티캐스트, 위젯 컨트롤러 MVC 계층 생성
플레이어 캐릭터에 부착되는 상호작용 전용 컴포넌트입니다. 생성자에서 PrimaryComponentTick.bCanEverTick = false로 Tick을 비활성화하고, SetIsReplicatedByDefault(true)로 네트워크 복제를 기본 활성화합니다.

BeginPlay에서 Owner의 ASC를 UAbilitySystemBlueprintLibrary::GetAbilitySystemComponent()로 조회하고, 서버 권한(HasAuthority)이면 GiveDefaultAbilities()를 호출하여 DefaultAbilities 배열의 어빌리티를 GAS에 부여합니다. 로컬 플레이어(IsLocalController)이면 InitializeInteractionUI(PC)를 호출하여 InteractionWidget과 WidgetController를 생성합니다.

Enhanced Input의 Started 이벤트에 OnInteractPressed, Completed 이벤트에 OnInteractReleased를 바인딩하여 상호작용 키 입력을 감지합니다. OnInteractPressed는 클라이언트 로컬에서 bIsHoldingInteract를 true로 설정하고, CurrentInteractTarget이 존재하면 ServerTriggerInteract RPC로 서버에 GameplayEvent를 전달합니다.

GetLifetimeReplicatedProps에서 DOREPLIFETIME 매크로로 bIsHoldingInteract를 복제 등록하여 서버-클라이언트 간 홀딩 상태가 동기화됩니다. MulticastPlayInteractionMontage를 통해 MotionWarpingComponent의 워프 타겟 설정과 몽타주 재생을 모든 클라이언트에서 동기화합니다.

OnSpawn 정책 상세
GA_Interact(주변 탐색) 어빌리티는 OnSpawn으로 설정되어 부여 즉시 활성화됩니다. 이 어빌리티 내부에서 WaitForInteractableTraceHit 태스크가 매 프레임 트레이스를 수행하여 상호작용 대상을 탐색하고, 발견 시 SetCurrentInteractTarget()으로 대상을 설정합니다. GA_Interact_Execute는 Manual 정책으로 ServerTriggerInteract의 GameplayEvent로만 활성화됩니다.

UAO_InteractableComponent
부모: UActorComponent · IAO_Interface_Interactable
역할
상호작용 가능한 액터에 부착되어 상호작용 정보 제공, 메시 콜리전 설정, 이펙트 재생을 담당하는 컴포넌트
부모 클래스
UActorComponent + IAO_Interface_Interactable 인터페이스 다중 상속
실행 위치
서버에서 NotifyInteractionSuccess 처리 및 멀티캐스트 이펙트 재생 트리거, 클라이언트에서 이펙트 스폰 실행
주요 책임
InteractionInfo 반환, 콜리전 채널 설정, VFX/SFX 스폰, 소켓 기반 워핑 Transform 제공
상호작용 대상 액터에 부착하는 컴포넌트입니다. UActorComponent와 IAO_Interface_Interactable을 다중 상속하여 GetInteractionInfo(), CanInteraction(), GetMeshComponents(), GetInteractionTransform()을 구현합니다.

생성자에서 PrimaryComponentTick.bCanEverTick = false, SetIsReplicatedByDefault(true)를 설정하고, InteractionTitle을 "Interact", InteractionContent를 ": Press F"로 기본값을 초기화합니다.

BeginPlay에서 SetupMeshCollisions()를 호출하여 Owner의 모든 유효한 메시 컴포넌트에 AO_TraceChannel_Interaction 콜리전 응답을 ECR_Block으로 설정합니다. 이를 통해 WaitForInteractableTraceHit 태스크의 라인 트레이스가 이 액터를 감지할 수 있습니다.

GetInteractionInfo()는 FAO_InteractionInfo 구조체에 Title, Content, Duration(0.0f), AbilityToGrant(UGA_Interact_Base::StaticClass()), 몽타주 2종, 소켓 Transform, WarpTargetName을 채워 반환합니다. AbilityToGrant에 기본 상호작용 어빌리티를 하드코딩하므로, 파생 클래스에서 오버라이드하지 않는 한 모든 상호작용은 GA_Interact_Base를 사용합니다.

상호작용 성공 시 NotifyInteractionSuccess()가 서버에서 호출되며, InteractionEffect 설정이 유효하면 MulticastPlayInteractionEffect를 통해 Niagara/Cascade VFX와 SFX를 모든 클라이언트에서 재생합니다. VFX/SFX 모두 지연 스폰(Delay)을 지원하며, FTimerHandle과 CreateWeakLambda를 사용하여 컴포넌트가 Destroy된 이후에도 안전하게 처리합니다.

MVC 패턴의 Controller 역할을 담당하는 UObject입니다. UCLASS(BlueprintType, Blueprintable)로 선언되어 블루프린트에서도 참조 가능합니다.

GAS 어빌리티 태스크(WaitForInteractableTraceHit, WaitForInvalidInteraction 등)에서 상호작용 대상이 발견/변경/해제될 때 BroadcastInteractionMessage()를 호출하면, OnInteractionMessageReceived 다이내믹 멀티캐스트 델리게이트를 통해 위젯에 메시지를 전달합니다.

HideInteraction()은 빈 FAO_InteractionMessage(기본 생성자 값)를 브로드캐스트하여 위젯을 초기 상태(숨김)로 되돌립니다. 이 패턴으로 "상호작용 불가능" 상태와 "아무 대상 없음" 상태를 모두 처리합니다.

NewObject<UAO_InteractionWidgetController>(this)로 생성되므로 Outer가 UAO_InteractionComponent이며, GC 소유 관계가 컴포넌트 수명에 바인딩됩니다. 컴포넌트가 파괴되면 컨트롤러도 GC 대상이 됩니다.

GameInstance 서브시스템으로 맵 전환에도 유지되는 UI 스택 관리자입니다. Initialize에서 UAO_DelegateManager의 OnSettingsOpen/Close 델리게이트를 바인딩합니다.

Push 메커니즘: PushWidgetInstance()로 위젯을 스택에 쌓으면, 먼저 CacheBaselineIfNeeded()로 UI 스택 사용 이전의 입력 상태를 스냅샷합니다. 같은 인스턴스가 이미 Top이면 정책만 재적용하고, 새로운 위젯이면 ZOrder를 계산하여 AddToViewport()하고 스택에 추가합니다. 마지막으로 ApplyTopPolicy()로 Top 위젯의 FAOUIStackPolicy를 PC에 적용합니다.

Pop 메커니즘: PopTop()은 Top 위젯을 Collapsed로 숨기고 스택에서 제거합니다. RemoveFromParent()가 아닌 Collapsed로 숨기므로 위젯 인스턴스가 보존됩니다. PopAll()은 모든 위젯을 RemoveFromParent()로 완전 제거합니다. PopByClass()는 뒤에서부터 검색하여 첫 번째 일치 항목만 제거합니다.

Fallback 정책: 스택이 비었을 때 ApplyFallbackPolicy()를 호출하여 베이스라인 복구를 시도합니다. 사망 상태(bIsAlive == false)에서는 GameAndUI + IgnoreLook/MoveInput으로 관전 UI를 지원하고, 생존 상태에서는 RestoreBaselineInput()으로 원래 입력 상태를 복구하거나 실패 시 GameOnly + 커서 숨김의 안전 기본값을 적용합니다.

ZOrder 계산
StackBaseZOrder(100) + Stack.Num()으로 계산됩니다. 첫 번째 위젯은 100, 두 번째는 101, 세 번째는 102... 나중에 Push된 위젯이 항상 위에 표시됩니다. 기존 UI 위젯(100 미만)을 가리지 않도록 베이스가 100입니다.
PopTop과의 비대칭
PushWidgetInstance는 AddToViewport를 호출하지만, PopTop은 RemoveFromParent가 아닌 SetVisibility(Collapsed)만 합니다. 이로 인해 Pop된 위젯도 뷰포트에 남아 있어 재Push 시 IsInViewport() 체크로 중복 추가를 방지합니다.

주요 책임
IMC에서 IA_UI_Open/Close 매핑 키 추출, IsUIOpenKey/CloseKey 판별 함수 제공, 캐시된 에셋 Getter
GameInstance 서브시스템으로 초기화 시 RefreshCache()를 호출하여 에셋을 로드합니다.

에셋 경로 (하드코딩):
• IMC: /Game/AVaOut/Character/Input/IMC_Player
• IA_UI_Open: /Game/AVaOut/UI/ActionKey/IA_UI_Open.IA_UI_Open
• IA_UI_Close: /Game/AVaOut/UI/ActionKey/IA_UI_Close.IA_UI_Close

LoadObject<UInputMappingContext>로 IMC를 로드하고, LoadObject<UInputAction>으로 Open/Close InputAction을 로드합니다. IMC의 GetMappings()를 순회하여 각 FEnhancedActionKeyMapping에서 Mapping.Action이 Open이면 해당 Key를 CachedOpenKeys에, Close이면 CachedCloseKeys에 추가합니다.

IsUIOpenKey()/IsUICloseKey()는 전달받은 FKey가 캐시된 배열에 Contains로 존재하는지 확인합니다. 이를 통해 UI 코드에서 EKeys::P, EKeys::Escape 등을 하드코딩하지 않고, InputAction 기반으로 키바인드를 추상화합니다. 키 리매핑 시 IMC만 변경하면 됩니다.

GetUIIMC(), GetUIOpenAction(), GetUICloseAction() getter는 캐시된 에셋 포인터를 반환하여 외부에서 IMC를 Enhanced Input에 추가하거나 Action을 바인딩할 때 사용합니다.
