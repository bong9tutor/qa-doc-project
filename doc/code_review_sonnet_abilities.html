UAO_GameplayAbility_Death
부모: UGameplayAbility
역할
플레이어 사망 처리 어빌리티 — 사망 몽타주 재생, 이동 차단, 래그돌 전환까지 관리
GAS 카테고리
Ability.State.Death · InstancedPerActor
실행 위치
ServerOnly — 서버에서만 활성화되어 권한 있는 상태에서 사망 로직 실행
주요 책임
HitReact 취소, 사망 사운드 재생, Death/Block 태그 적용, 래그돌 이벤트 대기 및 전환
체력이 0이 되면 서버에서 활성화되는 사망 전용 어빌리티입니다. 활성화 시 먼저 진행 중인 HitReact 어빌리티를 취소하고, 사망 GameplayCue를 실행하여 사운드를 재생합니다. DeathTagEffectClass와 BlockAbilitiesEffectClass를 적용하여 Status.Death 태그를 부여하고 다른 어빌리티를 차단합니다. DeathSpectateComponent를 통해 Multicast RPC로 사망 몽타주를 재생하며, 래그돌 이벤트(WaitGameplayEvent)를 수신하거나 폴백 타이머가 만료되면 FinalizeDeath()를 호출하여 몽타주를 중단하고 래그돌 상태로 전환합니다.
변수명타입지정자설명
DeathMontage TObjectPtr<UAnimMontage> EditDefaultsOnly 사망 시 재생할 애니메이션 몽타주. DeathSpectateComponent의 Multicast RPC로 모든 클라이언트에서 재생됩니다.
DeathTagEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly BPReadOnly Status.Death 태그를 부여하는 GameplayEffect 클래스. 다른 어빌리티가 이 태그를 확인하여 사망 상태를 감지합니다.
BlockAbilitiesEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly BPReadOnly 사망 중 다른 어빌리티 활성화를 차단하는 GameplayEffect 클래스.
RagdollEventTag FGameplayTag EditDefaultsOnly BPReadOnly 래그돌 전환 시점을 알리는 GameplayEvent의 태그. AnimNotify에서 이 태그로 이벤트를 발송합니다.
bDeathFinalizeCalled bool (private) FinalizeDeath() 중복 호출 방지 플래그. 래그돌 이벤트와 폴백 타이머 중 하나만 실행되도록 보장합니다.
DeathFinalizeTimerHandle FTimerHandle (private) 래그돌 이벤트를 수신하지 못했을 때 몽타주 길이 기반으로 FinalizeDeath()를 호출하는 폴백 타이머.

UAO_GameplayAbility_HitReact
부모: UGameplayAbility
역할
피격 리액션 전용 어빌리티 — 피격 방향에 따른 몽타주 재생과 무적/차단 효과 적용
GAS 카테고리
Ability.State.HitReact · InstancedPerActor · Event Trigger
실행 위치
ServerInitiated — 서버에서 시작하여 클라이언트로 복제
주요 책임
피격 방향 계산, 방향별 몽타주 선택/재생, 무적 효과 적용, 이동 모드 전환
Event.Combat.HitReact 이벤트를 트리거로 발동하는 피격 리액션 어빌리티입니다. Status.Death 태그가 있으면 활성화가 차단됩니다. 활성화 시 TriggerEventData에서 가해자 위치를 추출하고 GetHitDirectionSuffix()로 Front/Back/Left/Right 방향을 결정합니다. 방향 접미사가 포함된 태그로 HitReactMontageMap에서 몽타주를 검색하고, 없으면 기본 태그, 최종적으로 DefaultHitReactMontage를 사용합니다. 서버에서는 이동을 비활성화하고 InvulnerableEffectClass(무적)와 BlockAbilitiesEffectClass(차단)를 적용합니다. 몽타주 완료/취소 시 이동 모드를 Walking으로 복원하고 적용된 효과를 제거합니다.
변수명타입지정자설명
DefaultHitReactMontage TObjectPtr<UAnimMontage> EditDefaultsOnly BPReadOnly 방향별 몽타주를 찾지 못했을 때 사용하는 기본 피격 리액션 몽타주.
HitReactMontageMap TMap<FGameplayTag, UAnimMontage*> EditDefaultsOnly BPReadOnly GameplayTag를 키로 하는 방향별 피격 몽타주 맵. Event.Combat.HitReact.Heavy.Front 등의 태그로 매핑합니다.
InvulnerableEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly 피격 리액션 중 무적 상태를 부여하는 GameplayEffect. 리액션 종료 시 제거됩니다.
BlockAbilitiesEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly BPReadOnly 피격 리액션 중 다른 어빌리티 활성화를 차단하는 GameplayEffect.
InvulnerableEffectHandle FActiveGameplayEffectHandle (private) 적용된 무적 효과의 핸들. EndAbility에서 이 핸들로 효과를 제거합니다.
BlockAbilitiesEffectHandle FActiveGameplayEffectHandle (private) 적용된 차단 효과의 핸들. EndAbility에서 이 핸들로 효과를 제거합니다.

UAO_GameplayAbility_Sprint
부모: UGameplayAbility
역할
달리기 어빌리티 — 입력 유지 중 스프린트 상태를 활성화하고 스태미나를 소비
GAS 카테고리
Ability.Movement.Sprint · InstancedPerActor
실행 위치
LocalPredicted — 클라이언트에서 예측 실행, 서버에서 확인
주요 책임
스태미나 검사, 이동 속도 변경, 스프린트 Cost Effect 관리, 스프린트 후 재생 지연 적용
입력이 유지되는 동안 스프린트 상태를 관리하는 어빌리티입니다. CanActivateAbility에서 현재 스태미나가 StaminaCost 이상인지, 캐릭터가 이동 중인지, 앉기 상태가 아닌지, 스태미나 잠금 비율(StaminaLockoutPercent) 이상인지를 검사합니다. 활성화 시 캐릭터의 StartSprint_GAS(true)를 호출하여 이동 속도를 증가시키고, Status.Action.Sprint 태그를 ActivationOwnedTag로 부여합니다. 입력 해제(InputReleased) 시 어빌리티를 종료하며, EndAbility에서 스프린트 Cost Effect를 제거하고 PostSprintNoRegenEffectClass를 적용하여 스태미나 재생 지연을 부여합니다.
변수명타입지정자설명
PostSprintNoRegenEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly 스프린트 종료 후 스태미나 재생을 일시적으로 차단하는 GameplayEffect 클래스.
StaminaCost float EditDefaultsOnly 스프린트 활성화에 필요한 최소 스태미나 값. 기본값 2.0f.

UAO_GameplayAbility_Jump

역할
점프 어빌리티 — 캐릭터 점프 실행 및 착지 감지를 통한 종료 관리
GAS 카테고리
Ability.Movement.Jump · InstancedPerActor
실행 위치
LocalPredicted — 클라이언트에서 예측 실행, 서버에서 확인
주요 책임
스태미나 검사, Character::Jump() 호출, 착지(Walking) 감지 후 종료, 스태미나 변화 차단
점프 입력 시 활성화되는 어빌리티입니다. CanActivateAbility에서 스태미나가 StaminaCost 이상인지, 이미 활성 상태가 아닌지, Character::CanJump()가 true인지 검사합니다. 활성화 시 Character::Jump()를 호출하고, WaitMovementModeChange 태스크로 MOVE_Walking 전환(착지)을 대기합니다. 점프 중에는 Status.Action.Jump와 Status.Debuff.NoStaminaChange 태그를 ActivationOwnedTag로 부여하여 스태미나 변화를 차단합니다. Status.Action.Inspecting 태그가 있으면 활성화가 차단됩니다. EndAbility에서 서버 권한으로 PostSprintNoChangeEffectClass를 적용하여 착지 후 스태미나 변화 지연을 부여합니다.
변수명타입지정자설명
PostSprintNoChangeEffectClass TSubclassOf<UGameplayEffect> EditDefaultsOnly 착지 후 스태미나 변화를 일시적으로 차단하는 GameplayEffect 클래스.
StaminaCost float EditDefaultsOnly 점프 활성화에 필요한 최소 스태미나 값. 기본값 10.0f.

하는 일

근접 공격의 애니메이션 재생만 담당하는 기반 어빌리티입니다. 실제 히트 판정과 데미지 적용은 AnimNotify에서 발송하는 Event.Combat.Confirm 이벤트를 받는 AO_GameplayAbility_MeleeHitConfirm이 수행합니다. 이러한 분리를 통해 공격 애니메이션(클라이언트 예측)과 데미지 로직(서버 권한)의 실행 위치를 독립적으로 관리합니다. AttackMontage를 PlayMontageAndWait 태스크로 재생하며, 몽타주 완료 또는 취소 시 어빌리티를 종료합니다.

UAO_GameplayAbility_MeleeHitConfirm
부모: UGameplayAbility
역할
근접 히트 판정 어빌리티 — AnimNotify에서 보낸 이벤트를 받아 Sphere Trace로 적중 판정 및 데미지 적용
GAS 카테고리
Event.Combat.Confirm · InstancedPerActor · Event Trigger
실행 위치
ServerOnly — 서버에서만 데미지 판정을 수행하여 치트 방지
주요 책임
SphereTraceMulti 실행, 중복 대상 필터링, SetByCallerMagnitude 데미지 적용, HitReact 이벤트 전송
공격 몽타주의 AnimNotify가 전송하는 Event.Combat.Confirm 이벤트를 트리거로 발동합니다. TriggerEventData의 OptionalObject에 포함된 UAO_MeleeHitEventPayload에서 트레이스 파라미터(시작/끝 위치, 반경, DamageEffectClass, DamageAmount)를 추출합니다. SphereTraceMulti로 적중 액터를 수집하고, TSet으로 중복을 필터링합니다. 각 대상에 대해 TargetASC가 존재하고 Status.Invulnerable 태그가 없는 경우에만 SetByCallerMagnitude(Data.Damage)로 데미지를 적용하고, SendHitReactEvent()로 피격 리액션 이벤트를 전송합니다. 처리 완료 후 즉시 어빌리티를 종료합니다.
변수명타입지정자설명

bIgnoreInstigator bool EditDefaultsOnly BPReadOnly 트레이스에서 공격자 자신을 제외할지 여부. 기본값 true.

주요 책임
캡슐 트레이스로 장애물 감지, Chooser Table로 동작 평가, PoseSearch Motion Matching, MotionWarping 타겟 설정
장애물을 넘거나 기어오르는 파쿠르 동작을 처리하는 어빌리티입니다. CanActivateAbility에서 스태미나 검사 후 TryTraversal()을 호출하여 실제 트래버설 가능 여부를 판단합니다. SelectTraversal()에서 PoseSearch의 MotionMatch를 실행하여 현재 포즈와 레지 거리에 가장 적합한 몽타주와 시작 시간을 결정합니다. 활성화 시 MotionWarpingComponent에 FrontLedge/BackLedge/BackFloor 워프 타겟을 설정하고, 몽타주를 재생합니다.

Status.Death 태그가 추가되면 자동으로 어빌리티를 종료하며, EndAbility에서 하이라이트를 해제하고 타겟을 초기화합니다.

UAO_GameplayAbility_Inspect_Enter
부모: UAO_InteractionGameplayAbility
역할
검사 모드 진입 어빌리티 — 상호작용 대상의 InspectableComponent를 통해 검사 모드에 진입
GAS 카테고리
Interaction/Inspection · InstancedPerActor
실행 위치
ServerOnly — 서버에서만 실행되어 잠금 상태를 권한 있게 관리
주요 책임
다른 플레이어 잠금 확인, InspectableComponent 잠금 설정, InspectionComponent로 검사 모드 진입
GA_Interact_Execute에서 Finalize 이벤트 시 발동되는 검사 모드 진입 어빌리티입니다. 에디터에서 검사가 필요한 액터의 AbilityToGrant로 이 클래스를 설정합니다. 서버에서만 실행되며, TriggerEventData의 Target에서 InspectableComponent를 찾아 다른 플레이어가 이미 사용 중인지 확인합니다. 사용 가능하면 InspectableComponent를 잠금 처리하고, Instigator의 InspectionComponent로 검사 모드에 진입합니다. Status.Action.Inspecting 태그를 ActivationOwnedTag로 부여하고, 상호작용 어빌리티를 BlockAbilitiesWithTag로 차단합니다. 어빌리티가 활성화된 동안 검사 모드가 유지되며, 종료 시 잠금이 해제됩니다.
변수명타입지정자설명
이 클래스는 자체 멤버 변수를 선언하지 않습니다. 부모 UAO_InteractionGameplayAbility의 ActivationPolicy를 사용합니다.

ActivateAbility() ServerOnly ▶
하는 일
서버 권한을 check()로 확인하고 CommitAbility를 수행합니다. TriggerEventData에서 Target(검사 대상 액터)과 Instigator(요청 플레이어)를 추출합니다. Target의 InspectableComponent를 찾아 다른 플레이어가 잠금 중인지 확인하고, Instigator의 InspectionComponent를 찾아 잠금을 설정합니다. InspectEnterHandle을 저장하고 EnterInspectionMode()를 호출하여 검사 모드에 진입합니다.

GA_Interact_Execute의 SendFinalizeEvent()가 이 어빌리티를 트리거합니다.
