AAO_AIControllerBase
부모: AAIController
역할
모든 AI Controller의 공통 베이스 — Perception, StateTree 통합, 팀 시스템
부모 클래스
AAIController (IGenericTeamAgentInterface 포함)
실행 위치
서버에서만 실행 (HasAuthority 체크). 클라이언트에서는 BeginPlay/OnPossess가 조기 리턴
주요 책임
AIPerception(시야+청각) 설정, StateTree 시작/정지, 플레이어 감지 이벤트 분배
AAIController를 상속하여 시야(Sight)와 청각(Hearing) 두 가지 AIPerception 감각을 설정합니다. OnPossess에서 DefaultStateTree를 StateTreeAIComponent에 할당하고 StartLogic()으로 행동 트리를 가동합니다. 플레이어 감지/소실 이벤트를 OnPlayerDetected/OnPlayerLost 가상 함수로 분배하여 파생 컨트롤러가 오버라이드할 수 있습니다. NavMesh 도달 가능성 검증 함수(IsPlayerOnNavMesh, CanReachPlayer)를 제공하여 AI가 경계에서 멈추는 문제를 방지합니다. IGenericTeamAgentInterface를 통해 팀 시스템을 구현하며, AI는 팀 1번, 플레이어는 팀 0번으로 구분합니다.
변수명타입지정자설명

SightConfig / HearingConfig UAISenseConfig_Sight* / UAISenseConfig_Hearing* EditDefaultsOnlyBPReadOnly 시야와 청각 감각의 설정 객체. SetupPerceptionSystem()에서 NewObject로 생성하여 반경, 시야각 등을 적용합니다.
StateTreeComponent UStateTreeAIComponent* VisibleAnywhereBPReadOnly StateTree 실행 컴포넌트. OnPossess에서 DefaultStateTree를 할당받아 StartLogic()으로 가동합니다.
DefaultStateTree UStateTree* EditDefaultsOnlyBPReadOnly 기본 StateTree 에셋. 파생 클래스의 Blueprint에서 종별 StateTree를 지정합니다.
SightRadius / LoseSightRadius float EditDefaultsOnlyBPReadWrite 시야 감지 반경(기본 1500) 및 시야 유지 반경(기본 2000). LoseSightRadius가 더 커서 히스테리시스 효과를 줍니다.

bCanTargetDeadPlayer bool EditDefaultsOnlyBPReadWrite 죽은 플레이어도 타겟팅할 것인지 여부 (기본 false). Insect처럼 시체를 이용하는 AI는 true로 설정합니다.
bFilterUnreachablePlayers bool EditDefaultsOnlyBPReadWrite NavMesh 도달 불가능한 플레이어를 필터링할지 여부 (기본 true). AI가 경계에서 멈추는 문제를 방지합니다.
PlayersInSight TArray<TWeakObjectPtr<AAO_PlayerCharacter>> (UPROPERTY) 현재 시야 내에 있는 플레이어 목록. OnTargetPerceptionUpdated에서 자동으로 추가/제거됩니다.
TeamId FGenericTeamId EditDefaultsOnlyBPReadWrite 팀 ID (기본 1). AI는 1번 팀, 플레이어는 0번 팀으로 구분하여 적/아군을 판별합니다.

UAO_AISubsystem
부모: UWorldSubsystem
역할
AI 상태 관리 서브시스템 — 아이템 예약, 드롭 쿨다운, 납치 상태 관리
부모 클래스
UWorldSubsystem (월드 단위 싱글턴, 게임 월드에서만 생성)
실행 위치
서버에서 AI 로직이 실행되므로 서버 월드에서만 유효한 데이터 보유
주요 책임
아이템 중복 줍기 방지, Crab 드롭 쿨다운, Insect 납치 예약 관리
UWorldSubsystem을 상속하며 게임 월드에서만 생성됩니다 (ShouldCreateSubsystem에서 IsGameWorld() 확인). 아이템 예약 시스템으로 여러 AI가 같은 아이템을 동시에 줍는 것을 방지하고, 최근 드롭된 아이템에 쿨다운을 적용하여 반복적인 줍기/드롭을 막습니다. Insect의 납치 예약 시스템으로 한 플레이어가 동시에 여러 Insect에게 납치되는 것을 방지합니다. 5초마다 만료된 예약과 쿨다운을 자동 정리하는 타이머가 동작합니다. 모든 플레이어 위치 수집, 플레이어로부터 가장 먼 NavMesh 위치 탐색 등 유틸리티 함수도 제공합니다.
변수명타입지정자설명
ItemReservations TMap<TWeakObjectPtr<AAO_MasterItem>, FAO_ItemReservation> (UPROPERTY) 아이템별 예약 정보. 예약자(AI)와 예약 시간을 저장하며, ReservationTimeout(30초) 후 자동 만료됩니다.
RecentlyDroppedItems TMap<TWeakObjectPtr<AAO_MasterItem>, FAO_RecentlyDroppedItem> (UPROPERTY) 최근 드롭된 아이템 쿨다운 목록. CooldownDuration(기본 20초) 동안 같은 아이템을 다시 줍지 않습니다.
KidnappedPlayers TMap<TWeakObjectPtr<AAO_PlayerCharacter>, TWeakObjectPtr<AActor>> (UPROPERTY) 현재 납치 중인 플레이어와 납치자(Insect) 매핑. TryReservePlayerForKidnap으로 예약합니다.
RecentlyKidnappedPlayers TMap<TWeakObjectPtr<AAO_PlayerCharacter>, FAO_RecentlyDroppedItem> (UPROPERTY) 최근 납치된 플레이어 쿨다운. 같은 플레이어가 연속으로 납치되는 것을 방지합니다.
ReservationTimeout float (protected) 예약 만료 시간 (30초). 이 시간이 지나면 CleanupExpiredReservations에서 자동 해제됩니다.

UAO_AIAttributeSet
부모: UAttributeSet
역할
AI 캐릭터용 GAS Attribute Set — 이동 속도 어트리뷰트 정의
부모 클래스
UAttributeSet (GAS 프레임워크)
실행 위치
서버에서 값 변경, DOREPLIFETIME으로 클라이언트에 복제
주요 책임
MovementSpeed/MaxMovementSpeed 관리. AI는 체력이 없음 (요구사항)
UAttributeSet을 상속하며, AI 캐릭터의 이동 속도를 GAS 어트리뷰트로 관리합니다. MovementSpeed(기본 300)와 MaxMovementSpeed(기본 600) 두 가지 어트리뷰트를 제공합니다. PreAttributeChange에서 MovementSpeed가 0~MaxMovementSpeed 범위를 벗어나지 않도록 클램핑합니다. ReplicatedUsing으로 서버에서 변경된 값이 모든 클라이언트에 자동 복제됩니다. ATTRIBUTE_ACCESSORS_BASIC 매크로로 Getter/Setter/Init 함수를 자동 생성합니다.
변수명타입지정자설명
MovementSpeed FGameplayAttributeData BPReadOnlyReplicated 현재 이동 속도 (기본 300). GameplayEffect로 변경 가능하며, PreAttributeChange에서 0~MaxMovementSpeed로 클램핑됩니다.
MaxMovementSpeed FGameplayAttributeData BPReadOnlyReplicated 최대 이동 속도 (기본 600). MovementSpeed의 상한선으로 사용됩니다.
GetLifetimeReplicatedProps(OutLifetimeProps) Override ▶
하는 일
MovementSpeed와 MaxMovementSpeed를 DOREPLIFETIME_CONDITION_NOTIFY로 등록합니다. COND_None(항상 복제), REPNOTIFY_Always(변경 없어도 알림)로 설정됩니다.
PreAttributeChange(Attribute, NewValue) Override ▶
하는 일
어트리뷰트 값이 변경되기 전에 호출됩니다. MovementSpeed인 경우 0~MaxMovementSpeed로 클램핑합니다.
호출 시점
GameplayEffect가 MovementSpeed를 수정할 때 자동으로 호출됩니다.
OnRep_MovementSpeed / OnRep_MaxMovementSpeed UFUNCTION ▶
하는 일
클라이언트에서 복제된 값을 수신했을 때 호출되는 RepNotify 함수입니다. GAMEPLAYATTRIBUTE_REPNOTIFY 매크로로 GAS 시스템에 변경을 알립니다.
방향대상 클래스관계 설명
소유자AAO_AICharacterBaseAI 캐릭터의 AbilitySystemComponent에 등록됨

AAO_AggressiveAIBase
부모: AAO_AICharacterBase
역할
선공형 몬스터 공통 베이스 — 추격, 수색, 공격 범위 인터페이스
부모 클래스
AAO_AICharacterBase (GAS, Memory, Stun 지원)
실행 위치
서버에서 실행. AutoPossessAI = PlacedInWorldOrSpawned 설정
주요 책임
배회/추격/수색 상태 관리, 이동 속도 전환, 공격 범위 판단
AAO_AICharacterBase를 상속하며, 플레이어를 추격하고 공격하는 선공형 AI의 공통 로직을 정의합니다. 배회(Roam) → 추격(Chase) → 수색(Search) → 배회 사이클을 관리하며, 각 상태에 따라 이동 속도를 자동으로 전환합니다 (RoamSpeed → ChaseSpeed). CurrentTarget으로 현재 추격 대상을 관리하고, IsTargetInAttackRange()로 공격 가능 여부를 판단합니다. Bull, Insect, LavaMonster, Stalker, Troll, Werewolf이 이 클래스를 상속합니다.
변수명타입지정자설명
RoamSpeed float EditDefaultsOnlyBPReadWrite 배회 이동 속도 (기본 300). 파생 클래스에서 재설정합니다.
ChaseSpeed float EditDefaultsOnlyBPReadWrite 추격 이동 속도 (기본 500). 추격 모드에서 CharacterMovement에 적용됩니다.
AttackRange float EditDefaultsOnlyBPReadWrite 공격 범위 (기본 200). IsTargetInAttackRange()에서 거리 비교에 사용됩니다.
SearchDuration / SearchRadius float EditDefaultsOnlyBPReadWrite 수색 지속 시간 (기본 7초)과 수색 반경 (기본 500). StateTree의 수색 Task에서 참조합니다.
bIsChasing / bIsSearching bool BPReadOnly 현재 추격/수색 상태 플래그. SetChaseMode/SetSearchMode로 상호 배타적으로 전환됩니다.
CurrentTarget TWeakObjectPtr<AAO_PlayerCharacter> (UPROPERTY) 현재 추격 대상 플레이어. TWeakObjectPtr로 안전하게 참조합니다.
SetChaseMode(bChasing) / SetSearchMode(bSearching) BlueprintCallable ▶
하는 일
추격/수색 모드를 전환합니다. 두 모드는 상호 배타적이며, 전환 시 UpdateMovementSpeed()가 자동 호출됩니다.
상태 전이
SetChaseMode(true) → bIsChasing=true, bIsSearching=false, 속도=ChaseSpeed
SetSearchMode(true) → bIsSearching=true, bIsChasing=false, 속도=RoamSpeed
